    12장. 작은 이미지 예제로 확장하기¶

    최초 작성: 2026-05-23 01:43 KST 최종 업데이트: 2026-05-23 08:54 KST

    이 장의 질문¶

    2차원 toy example에서는 모든 것이 눈에 보였다. 이제 이미지를 생성하려면 무엇이 달라질까? 수식의 구조는 거의 그대로지만, 데이터 차원, 모델 구조, 시간 임베딩, normalization, sampling 비용이 모두 중요해진다.

    이 장은 MNIST, Fashion-MNIST, CIFAR-10 같은 작은 이미지 데이터셋을 염두에 둔다. 목적은 최고 품질 이미지를 만드는 것이 아니라, Flow Matching이 이미지 텐서에서 어떻게 작동하는지 이해하는 것이다.

    이미지 데이터의 차원¶

    이미지 배치는 다음 shape를 갖는다.

    \[ x_1\in\mathbb{R}^{B\times C\times H\times W}. \]

    예를 들어 MNIST는 \(C=1\), \(H=W=28\)이고, CIFAR-10은 \(C=3\), \(H=W=32\)다. 시작 노이즈도 같은 shape여야 한다.

    \[ x_0\sim\mathcal{N}(0,I), \qquad x_0\in\mathbb{R}^{B\times C\times H\times W}. \]

    직선 경로는 그대로 쓸 수 있다.

    \[ x_t=(1-t)x_0+tx_1, \]
    \[ u_t=x_1-x_0. \]

    다만 \(t\)는 broadcasting을 위해 다음 shape로 바꾸어야 한다.

    \[ t_{\mathrm{view}}\in\mathbb{R}^{B\times 1\times 1\times 1}. \]

    Normalization이 중요하다¶

    이미지 픽셀은 원래 \([0,255]\) 또는 \([0,1]\) 범위에 있다. 노이즈 \(x_0\)는 보통 표준정규분포에서 오므로 값의 범위가 다르다. 따라서 데이터 \(x_1\)을 어떤 범위로 normalize할지 결정해야 한다.

    많은 생성 모델 구현에서는 이미지를 \([-1,1]\) 범위로 변환한다.

    \[ x_1 = 2x_{\mathrm{raw}}-1. \]

    이렇게 하면 데이터와 노이즈의 scale이 지나치게 어긋나는 문제를 줄일 수 있다. normalization을 바꾸면 목표 속도 \(u_t=x_1-x_0\)의 scale도 바뀌므로 loss 크기와 학습 안정성에 영향을 준다.

    모델 구조: 작은 U-Net¶

    이미지에서는 MLP보다 convolution 기반 모델이 적합하다. 가장 흔한 선택은 U-Net 계열이다. U-Net은 downsampling과 upsampling 경로를 통해 local detail과 global context를 함께 다룬다.

    Flow Matching 모델은 다음 함수를 근사한다.

    \[ v_\theta:\mathbb{R}^{C\times H\times W}\times[0,1]\to\mathbb{R}^{C\times H\times W}. \]

    입력 이미지와 출력 velocity의 shape는 같다. 모델 내부에서는 시간 임베딩 \(\phi(t)\)를 각 block에 주입한다.

    \[ v_\theta(x_t,t)=\mathrm{UNet}_\theta(x_t,\phi(t)). \]

    입문 실험에서는 너무 큰 모델보다 작은 U-Net으로 시작하는 것이 좋다. 작은 데이터셋에서 모델이 너무 크면 학습은 잘 되는 것처럼 보여도 실험 시간이 길어지고 디버깅이 어려워진다.

    학습 루프의 차이¶

    수식은 toy example과 같지만 이미지에서는 다음 차이가 생긴다.

    • dataloader가 이미지 augmentation과 normalization을 담당한다.
    • 모델이 훨씬 크므로 GPU 메모리를 고려해야 한다.
    • batch size가 작아질 수 있다.
    • sample visualization을 주기적으로 저장해야 한다.
    • ODE sampling이 느릴 수 있으므로 step 수를 실험해야 한다.

    학습 step은 여전히 다음 구조다.

    x1 = batch
x0 = torch.randn_like(x1)
t = torch.rand(x1.shape[0], device=x1.device)
xt, ut = path.sample(x0, x1, t)
pred = model(xt, t)
loss = ((pred - ut) ** 2).mean()

    Sampling grid 만들기¶

    이미지 생성 결과는 일정한 grid로 저장하는 것이 좋다. 같은 노이즈 seed에서 step 수나 checkpoint를 바꾸어 비교하면 학습 진행을 보기 쉽다.

    예를 들어 다음 비교가 유용하다.

    • training step별 생성 이미지
    • ODE step 수 \(K=10,20,50,100\) 비교
    • Euler와 midpoint solver 비교
    • 같은 모델에서 서로 다른 random seed 비교

    샘플 이미지는 단일 장보다 grid로 보는 것이 좋다. 개별 샘플 하나는 우연히 좋아 보일 수 있기 때문이다.

    품질 평가를 어떻게 볼 것인가¶

    입문 단계에서는 FID 같은 정량 지표보다 먼저 시각적 품질과 실패 패턴을 관찰한다. 특히 작은 이미지 실험에서는 다음을 확인한다.

    • 이미지가 데이터셋의 대략적 구조를 갖는가?
    • 샘플 다양성이 있는가?
    • 지나치게 흐릿하거나 평균적인 이미지가 나오지 않는가?
    • 특정 클래스나 모드만 반복되지 않는가?
    • ODE step 수를 늘리면 결과가 안정적으로 좋아지는가?

    정량 지표는 중요하지만, 초기 디버깅에서는 오히려 문제의 원인을 가릴 수 있다. 먼저 그림을 보고 모델이 무엇을 하고 있는지 이해하는 편이 좋다.

    흔한 이미지 실험 실패¶

    이미지 Flow Matching 실험에서 자주 만나는 실패는 다음과 같다.

    첫째, normalization 오류다. 데이터가 \([0,1]\)인데 sampling 후 \([-1,1]\)로 복원한다고 가정하면 이미지가 이상하게 보인다.

    둘째, 시간 임베딩 오류다. \(t\)를 모델에 넣지 않거나 shape가 잘못되면 모델은 시간별 속도 차이를 배울 수 없다.

    셋째, solver step 부족이다. 학습은 된 것 같지만 sampling step이 너무 적으면 이미지가 덜 형성된다.

    넷째, 모델 용량 부족이다. toy example에서는 MLP로 충분했지만 이미지에서는 작은 U-Net도 데이터셋에 따라 부족할 수 있다.

    다섯째, 학습 target scale 문제다. 목표 속도의 norm이 너무 크거나 작으면 learning rate와 loss scale을 조정해야 한다.

    그림 후보¶

    • 이미지 텐서 경로 그림: 노이즈 이미지, 중간 이미지, 데이터 이미지가 시간 순서로 나열되는 도식.
    • 샘플 grid: 학습 step에 따라 생성 이미지가 어떻게 변하는지 보여 주는 grid.
    • ODE step 비교 grid: 같은 모델에서 step 수만 바꾸어 결과를 비교한다.
    • U-Net 구조 그림: 입력 \(x_t\)와 시간 임베딩이 U-Net block에 들어가고 velocity image가 출력되는 구조.

    요약¶

    • 작은 이미지 예제에서도 Flow Matching의 기본 수식은 toy example과 같지만 텐서 shape와 normalization이 더 중요해진다.
    • 이미지 데이터는 보통 \([-1,1]\) 범위로 normalize하고, 노이즈와 같은 shape를 유지한다.
    • 모델은 \(x_t\)와 시간 \(t\)를 입력받아 이미지와 같은 shape의 velocity를 출력한다.
    • 샘플 grid와 ODE step 비교는 이미지 실험의 핵심 디버깅 도구다.
    • 초기에는 정량 지표보다 normalization, 시간 임베딩, solver step, 샘플 다양성을 먼저 확인하는 것이 좋다.
