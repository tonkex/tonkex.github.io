    2장. 조건부 확률과 주변화¶

    하나의 데이터 샘플을 기준으로 만든 쉬운 길들을 모으면 전체 분포의 길이 될까?

    1장에서는 생성 모델을 분포와 샘플의 언어로 읽는 법을 정리했다. 이제 우리는 \(p_0\), \(p_1\), \(p_t\)가 각각 시작 분포, 목표 분포, 중간 분포를 뜻한다는 정도는 알고 있다. 하지만 Flow Matching을 제대로 이해하려면 한 단계가 더 필요하다. 바로 조건부 확률과 주변화다.

    Conditional Flow Matching이라는 이름 자체에 conditional이 들어간다. 이 말은 우연이 아니다. Flow Matching의 중요한 아이디어는 복잡한 전체 분포의 이동을 직접 다루기보다, 조건이 주어진 더 쉬운 이동을 먼저 정의하고, 그것을 평균내어 전체 이동으로 연결하는 데 있다.

    이 장에서는 \(p_t(x \mid z)\)와 \(p_t(x)\)의 차이를 천천히 살펴본다. 그리고 왜 조건부 path가 학습하기 쉬운 target을 만들고, 주변화가 그 target을 전체 분포의 velocity field와 연결하는지 직관적으로 준비한다.

    조건이 주어지면 불확실성이 달라진다¶

    조건부 확률은 “어떤 정보를 이미 알고 있을 때 확률이 어떻게 바뀌는가”를 말한다. 예를 들어 손글씨 숫자 이미지를 무작위로 하나 뽑는다고 하자. 아무 조건이 없다면 그 이미지는 0부터 9까지 어떤 숫자일 수도 있다. 하지만 “이 이미지는 숫자 3이다”라는 조건이 주어지면 가능한 이미지는 크게 줄어든다.

    이 차이를 확률로 쓰면 다음과 같다.

    \[ p(x) \]

    은 전체 이미지 분포를 뜻하고,

    \[ p(x \mid y = 3) \]

    은 label이 3이라는 조건 아래의 이미지 분포를 뜻한다. 조건이 주어졌기 때문에 분포의 모양이 달라진다. 숫자 3처럼 보이는 이미지들이 많이 모인 영역에 확률 질량이 집중될 것이다.

    Flow Matching에서 조건은 label만을 뜻하지 않는다. 조건은 데이터 샘플 하나일 수도 있고, noise 샘플과 data 샘플의 쌍일 수도 있으며, path를 정의하는 데 필요한 latent variable일 수도 있다. 이 책에서는 조건을 넓게 \(z\)라고 쓰겠다.

    \[ p(x \mid z) \]

    이 식은 “조건 \(z\)가 주어졌을 때 \(x\)의 분포”라고 읽는다.

    [그림 2-1: 조건이 바꾸는 분포]

    • 왼쪽: 전체 손글씨 이미지 분포 \(p(x)\)
    • 오른쪽: 숫자 3이라는 조건 아래의 분포 \(p(x \mid y=3)\)
    • 캡션: 조건이 주어지면 가능한 샘플의 범위와 분포의 모양이 달라진다.

    Joint, conditional, marginal¶

    조건부 확률을 이해하려면 세 가지 분포를 구분해야 한다.

    \[ p(x, z) \]

    은 joint distribution이다. \(x\)와 \(z\)가 함께 나타나는 방식을 설명한다. 예를 들어 \(z\)가 label이고 \(x\)가 이미지라면, \(p(x, z)\)는 어떤 label과 어떤 이미지가 함께 나올 확률 구조를 말한다.

    \[ p(x \mid z) \]

    은 conditional distribution이다. \(z\)가 주어졌을 때 \(x\)가 어떻게 분포하는지를 설명한다.

    \[ p(x) \]

    은 marginal distribution이다. \(z\)를 따로 보지 않고 \(x\)만 볼 때의 전체 분포다.

    이 셋은 서로 떨어진 개념이 아니다. 기본 관계는 다음과 같다.

    \[ p(x, z) = p(x \mid z)p(z) \]

    그리고 joint distribution에서 \(z\)를 모두 합치면 marginal distribution을 얻는다.

    \[ p(x) = \int p(x, z)\,dz \]

    위 두 식을 합치면 다음 식이 나온다.

    \[ p(x) = \int p(x \mid z)p(z)\,dz \]

    이 식이 주변화, 즉 marginalization이다. 말로 풀면 이렇다.

    조건 \(z\)별로 \(x\)의 분포를 구한 뒤, 각 조건이 얼마나 자주 나타나는지 \(p(z)\)로 가중 평균하면 전체 \(x\) 분포가 된다.

    적분 기호가 부담스럽다면 우선 “조건별 분포를 모두 섞는다”고 생각해도 된다. \(z\)가 연속값이면 적분이고, \(z\)가 이산 label이면 합으로 쓸 수 있다.

    \[ p(x) = \sum_z p(x \mid z)p(z) \]

    [그림 2-2: 조건별 분포를 섞어 전체 분포 만들기]

    • 왼쪽: 여러 조건 \(z_1, z_2, z_3\)에 대한 조건부 분포
    • 가운데: 각 조건의 비중 \(p(z)\)
    • 오른쪽: 모든 조건을 합친 marginal distribution \(p(x)\)
    • 캡션: marginalization은 조건부 분포들을 조건의 비중만큼 섞어 전체 분포를 만드는 과정이다.

    왜 주변화가 생성 모델에서 중요할까¶

    생성 모델은 전체 데이터 분포 \(p_{\text{data}}(x)\)를 배우고 싶어 한다. 하지만 전체 분포는 복잡하다. 이미지는 고차원이고, 같은 class 안에서도 다양한 스타일이 있으며, 전체 분포를 직접 계산하기는 거의 불가능하다.

    대신 조건을 하나 잡으면 문제가 쉬워지는 경우가 많다. 예를 들어 “특정 데이터 샘플 \(x_1\)을 목표로 삼는다”는 조건을 주면, noise 샘플 \(x_0\)에서 \(x_1\)로 가는 단순한 path를 만들 수 있다. 전체 데이터 분포로 가는 길은 복잡하지만, 특정 목표 샘플 하나로 가는 길은 상대적으로 단순하게 정의할 수 있다.

    이것이 Flow Matching에서 조건부 관점이 중요한 이유다. 우리는 전체 \(p_t(x)\)의 velocity field를 직접 계산하기 어렵다. 하지만 조건 \(z\)가 주어졌을 때의 path와 velocity는 더 쉽게 정의할 수 있다.

    예를 들어 조건 \(z\)가 목표 데이터 샘플 \(x_1\)이라고 하자. 그러면 시간 \(t\)에서의 중간 샘플을 단순하게 다음처럼 만들 수 있다.

    \[ x_t = (1-t)x_0 + tx_1 \]

    이 식은 noise 샘플 \(x_0\)와 data 샘플 \(x_1\) 사이를 직선으로 잇는다. 이 path의 속도는 간단하다.

    \[ \frac{d x_t}{dt} = x_1 - x_0 \]

    물론 실제 Flow Matching 논문에서는 path 설정이 더 일반적일 수 있다. 하지만 핵심 직관은 같다. 조건을 주면 path와 target velocity를 쉽게 만들 수 있다.

    [그림 2-3: 조건부 path의 직관]

    • 왼쪽: noise 샘플 \(x_0\)
    • 오른쪽: 특정 data 샘플 \(x_1\)
    • 가운데: \(x_t = (1-t)x_0 + tx_1\)로 이어지는 직선 path
    • 화살표: target velocity \(x_1 - x_0\)
    • 캡션: 특정 조건이 주어지면 하나의 샘플이 따라갈 쉬운 path를 정의할 수 있다.

    Conditional path와 marginal path¶

    이제 Flow Matching에서 자주 나오는 두 표현을 구분하자.

    Conditional path는 조건 \(z\)가 주어졌을 때의 중간 분포다.

    \[ p_t(x \mid z) \]

    Marginal path는 조건을 보지 않고 전체적으로 본 중간 분포다.

    \[ p_t(x) \]

    둘은 다음 관계로 연결된다.

    \[ p_t(x) = \int p_t(x \mid z)p(z)\,dz \]

    이 식은 1장에서 봤던 \(p_t\)를 더 자세히 설명한다. \(p_t\)는 그냥 하늘에서 떨어진 중간 분포가 아니다. 여러 조건부 path가 조건의 분포 \(p(z)\)에 따라 섞인 결과로 볼 수 있다.

    직관적으로는 이렇다. 각 데이터 샘플 또는 각 조건마다 noise에서 data로 가는 작은 길을 하나씩 만든다. 그 길들을 전체 데이터셋에 대해 많이 모으면, 전체 분포가 noise에서 data로 이동하는 큰 흐름이 된다. Conditional path는 작은 길이고, marginal path는 그 작은 길들이 모여 만든 전체 흐름이다.

    [그림 2-4: conditional path와 marginal path]

    • 여러 개의 얇은 선: 조건별 conditional path
    • 굵은 흐름 또는 density snapshot: 전체 marginal path
    • 캡션: 조건별 path를 모두 모으면 전체 분포의 시간별 이동인 marginal path를 볼 수 있다.

    조건부 target을 학습해도 전체 field를 배울 수 있는 이유¶

    Flow Matching에서 모델이 궁극적으로 필요한 것은 전체 분포를 이동시키는 velocity field다. 이를 \(v_t(x)\)라고 쓰자. Sampling할 때 모델은 특정 조건 \(z\)를 모르는 상태에서도 현재 위치 \(x_t\)와 시간 \(t\)를 보고 어느 방향으로 움직일지 예측해야 한다.

    그런데 학습할 때는 조건부 target velocity를 사용할 수 있다. 조건 \(z\)가 주어졌을 때의 target velocity를 \(u_t(x \mid z)\)라고 쓰자. Conditional Flow Matching은 모델 \(v_\theta(t, x)\)가 이 조건부 target을 평균적으로 맞추도록 학습한다.

    핵심 직관은 다음과 같다.

    같은 위치 \(x\)와 시간 \(t\)에서 가능한 조건부 velocity들을 조건에 대해 평균내면, 전체 marginal path를 움직이는 velocity가 된다.

    이를 매우 압축해서 쓰면 다음과 같은 형태로 생각할 수 있다.

    \[ v_t(x) = \mathbb{E}_{z \sim p(z \mid x)}[u_t(x \mid z)] \]

    이 식을 지금 완전히 증명할 필요는 없다. 중요한 것은 조건부 target이 단순한 편의 장치가 아니라, 전체 분포의 velocity field와 연결되어 있다는 점이다. 그래서 학습에서는 조건부 target을 사용하면서도, sampling에서는 전체 velocity model을 사용할 수 있다.

    이 구조 덕분에 Flow Matching은 복잡한 marginal velocity를 직접 계산하지 않고도 학습 가능한 objective를 만들 수 있다.

    Posterior처럼 보이는 식을 겁내지 않기¶

    방금 식에는 \(p(z \mid x)\)가 등장했다. 이런 식을 보면 갑자기 Bayesian posterior가 떠오르면서 어려워질 수 있다. 여기서는 너무 복잡하게 생각하지 않아도 된다.

    \(p(z \mid x)\)는 “현재 위치 \(x\)가 주어졌을 때 어떤 조건 \(z\)가 그 위치를 만들었을 가능성이 큰가”를 나타낸다. 예를 들어 여러 conditional path가 같은 위치 근처를 지나갈 수 있다. 그 위치에서 전체 velocity를 정하려면, 가능한 조건들의 velocity를 그냥 똑같이 평균내는 것이 아니라, 그 위치와 관련 있는 조건에 더 큰 비중을 주어 평균내야 한다.

    이 설명은 직관을 위한 것이다. 실제 학습에서는 \(p(z \mid x)\)를 직접 계산하지 않는다. Conditional Flow Matching의 강점은 바로 여기에 있다. 복잡한 posterior나 marginal velocity를 직접 구하지 않고도, 조건부 샘플링으로 loss를 구성할 수 있다.

    코드 관점에서 보면 무엇이 달라질까¶

    구현 관점에서 조건부 확률과 주변화는 다음 구조로 나타난다.

    x0 = sample_noise(batch_size)
x1 = sample_data(batch_size)
t = sample_time(batch_size)
xt = make_conditional_path(x0, x1, t)
target_velocity = compute_target_velocity(x0, x1, t)
pred_velocity = model(xt, t)
loss = ((pred_velocity - target_velocity) ** 2).mean()

    여기서 \(x_0\), \(x_1\), \(t\)를 뽑아 \(x_t\)와 target velocity를 만드는 부분이 조건부 path에 해당한다. 모델은 여러 batch를 통해 다양한 조건부 path를 보게 된다. 이 과정을 충분히 반복하면 모델은 특정 조건 하나에만 맞는 velocity가 아니라, 전체 분포의 평균적인 velocity field를 학습하게 된다.

    따라서 수식의 주변화는 코드에서 “많은 조건을 무작위로 샘플링하고 평균 loss를 줄이는 학습 과정”으로 나타난다. 이것이 논문 속 expectation과 실제 minibatch 학습이 연결되는 방식이다.

    [그림 2-5: CFM 학습에서 조건부 path가 쓰이는 위치]

    • 입력: noise batch \(x_0\), data batch \(x_1\), time batch \(t\)
    • 중간: conditional path로 \(x_t\)와 target velocity 생성
    • 출력: model prediction과 target velocity의 차이로 loss 계산
    • 캡션: 구현에서는 조건부 path를 샘플링해 target velocity를 만들고, 그 평균 loss로 모델을 학습한다.

    Flow Matching으로 이어지는 연결¶

    이 장에서 배운 조건부 확률과 주변화는 Flow Matching의 중심 구조로 바로 이어진다.

    \[ p_t(x) = \int p_t(x \mid z)p(z)\,dz \]

    이 식은 조건부 path들이 모여 marginal path를 만든다는 뜻이다. 그리고 조건부 target velocity를 사용해 학습해도 전체 velocity field를 배울 수 있다는 직관을 제공한다.

    다음 장에서는 “분포가 시간에 따라 움직인다”는 생각을 더 직접적으로 다룬다. 지금까지는 \(p_t\)를 중간 분포라고 불렀지만, 아직 그 분포가 어떻게 움직이는지는 자세히 말하지 않았다. 3장에서는 sample trajectory, distribution trajectory, vector field, ODE의 언어로 이 움직임을 설명한다.

    요약¶

    • 조건부 분포 \(p(x \mid z)\)는 조건 \(z\)가 주어졌을 때 \(x\)가 어떻게 분포하는지를 설명한다.
    • 주변화는 조건부 분포들을 조건의 비중만큼 섞어 전체 분포 \(p(x)\)를 얻는 과정이다.
    • Flow Matching에서는 조건부 path \(p_t(x \mid z)\)들이 모여 marginal path \(p_t(x)\)를 만든다고 볼 수 있다.
    • 조건부 target velocity는 직접 계산하기 쉬운 경우가 많고, 그 평균은 전체 velocity field와 연결된다.
    • 구현에서는 여러 조건부 path를 batch로 샘플링하고 평균 loss를 줄이는 방식으로 주변화의 효과가 나타난다.
