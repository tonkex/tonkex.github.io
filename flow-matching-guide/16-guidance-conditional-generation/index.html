    16장. Guidance와 Conditional Generation¶

    최초 작성: 2026-05-23 01:43 KST 최종 업데이트: 2026-05-23 08:54 KST

    이 장의 질문¶

    생성 모델은 단순히 그럴듯한 샘플을 만드는 것에서 끝나지 않는다. 사용자는 특정 숫자, 특정 클래스, 특정 텍스트 설명, 특정 스타일을 원한다. Flow Matching에서는 이런 조건부 생성을 어떻게 다룰 수 있을까? 그리고 diffusion에서 익숙한 guidance 개념은 어떤 식으로 연결될까?

    이 장은 조건부 Flow Matching의 실용적 구조를 다룬다. 조건 정보를 모델에 넣는 방법, classifier-free guidance의 직관, guidance scale의 trade-off, 구현에서 조심할 점을 정리한다.

    조건부 velocity model¶

    조건부 생성에서는 모델이 조건 \(c\)를 함께 입력받는다.

    \[ v_\theta(x_t,t,c). \]

    여기서 \(c\)는 다양한 형태일 수 있다.

    • 클래스 라벨
    • 텍스트 임베딩
    • 저해상도 이미지
    • segmentation map
    • 다른 modality의 feature

    학습 target은 여전히 velocity다.

    \[ \mathcal{L}(\theta)=\mathbb{E}\left[\left\|v_\theta(x_t,t,c)-u_t\right\|^2\right]. \]

    조건 \(c\)는 모델이 어떤 방향의 velocity를 선택해야 하는지 알려 주는 정보다. 같은 중간 상태 \(x_t\)라도 조건이 다르면 도착해야 할 데이터 영역이 달라질 수 있다.

    Class conditioning¶

    가장 단순한 조건은 클래스 라벨이다. 예를 들어 MNIST에서 \(c\)가 숫자 라벨이라면 모델은 특정 숫자 형태로 이동하는 velocity를 배운다.

    클래스 조건은 embedding으로 바꾸어 모델에 넣는다.

    \[ e_c=\mathrm{Embed}(c). \]

    모델은 시간 임베딩과 클래스 임베딩을 함께 사용할 수 있다.

    \[ v_\theta(x_t,t,c)=f_\theta(x_t,\phi(t),e_c). \]

    U-Net에서는 이 embedding을 residual block에 주입하거나 normalization layer를 조절하는 방식으로 사용할 수 있다.

    Text conditioning¶

    텍스트 조건은 클래스보다 복잡하다. 텍스트 encoder가 문장을 임베딩으로 바꾸고, 생성 모델은 그 임베딩을 cross-attention이나 conditioning block을 통해 사용한다.

    \[ e_c=\mathrm{TextEncoder}(c). \]

    그 다음 모델은

    \[ v_\theta(x_t,t,e_c) \]

    를 예측한다. 이 구조는 diffusion 기반 text-to-image 모델과 비슷한 면이 있다. 차이는 모델이 예측하는 값과 sampling dynamics가 Flow Matching의 velocity field라는 점이다.

    Classifier-free guidance의 직관¶

    Classifier-free guidance는 조건부 예측과 무조건 예측을 조합해 조건을 더 강하게 반영하는 방법이다. Flow Matching에서도 velocity 예측에 비슷한 아이디어를 적용할 수 있다.

    무조건 velocity를

    \[ v_{\mathrm{uncond}}=v_\theta(x_t,t,\varnothing) \]

    조건부 velocity를

    \[ v_{\mathrm{cond}}=v_\theta(x_t,t,c) \]

    라고 하자. guidance를 적용한 velocity는 다음처럼 쓸 수 있다.

    \[ v_{\mathrm{guided}}=v_{\mathrm{uncond}}+s\left(v_{\mathrm{cond}}-v_{\mathrm{uncond}}\right). \]

    여기서 \(s\)는 guidance scale이다. \(s=1\)이면 조건부 예측을 그대로 쓰고, \(s>1\)이면 조건 방향을 더 강하게 밀어 준다.

    Guidance scale의 trade-off¶

    Guidance scale을 키우면 조건 일치도가 좋아질 수 있다. 하지만 너무 크게 키우면 샘플 다양성이 줄고, 이미지가 부자연스러워질 수 있다.

    이 trade-off는 diffusion에서도 익숙하다. Flow Matching에서도 마찬가지로 velocity field를 guidance 방향으로 변형하기 때문에 trajectory가 원래 학습된 경로에서 벗어날 수 있다.

    따라서 guidance scale은 다음 기준으로 실험해야 한다.

    • 조건 일치도
    • 샘플 다양성
    • 시각적 자연스러움
    • ODE solver 안정성
    • step 수 변화에 따른 민감도

    특히 \(s\)를 크게 했을 때 velocity norm이 지나치게 커지는지 확인하는 것이 좋다.

    학습 시 조건 dropout¶

    Classifier-free guidance를 쓰려면 모델이 조건부와 무조건 예측을 모두 할 수 있어야 한다. 이를 위해 학습 중 일부 확률로 조건을 비운다.

    \[ c'=\begin{cases} \varnothing & \text{with probability } p_{\mathrm{drop}} \\ c & \text{otherwise} \end{cases} \]

    모델은 \(c'\)를 입력받아 velocity를 예측한다. 이렇게 하면 sampling 때 같은 모델로 조건부와 무조건 velocity를 모두 계산할 수 있다.

    구현에서는 null condition의 표현을 명확히 정해야 한다. 클래스 조건이라면 별도의 null class embedding을 둘 수 있고, 텍스트 조건이라면 empty prompt embedding을 사용할 수 있다.

    조건부 생성의 실패 패턴¶

    조건부 Flow Matching에서 흔한 실패는 다음과 같다.

    첫째, 조건을 무시한다. 모델이 \(x_t\)만 보고 평균적인 샘플을 만들고 조건 \(c\)를 충분히 사용하지 않는 경우다.

    둘째, guidance scale이 너무 커서 artifact가 생긴다. 조건은 맞지만 샘플이 어색해질 수 있다.

    셋째, null condition 학습이 부족하다. 조건 dropout 비율이 너무 낮거나 null embedding 처리가 불안정하면 guidance가 잘 작동하지 않는다.

    넷째, 조건과 데이터가 불균형하다. 특정 클래스나 텍스트 패턴이 적으면 해당 조건의 velocity를 제대로 배우기 어렵다.

    그림 후보¶

    • 조건부 velocity 모델 그림: \(x_t\), \(t\), 조건 \(c\)가 모델에 들어가 velocity가 출력되는 구조.
    • CFG 벡터 그림: 무조건 velocity와 조건부 velocity의 차이 방향을 guidance scale로 키우는 모습을 화살표로 표시한다.
    • guidance scale grid: \(s=1\), \(s=2\), \(s=4\), \(s=8\)에서 결과가 어떻게 달라지는지 비교한다.

    요약¶

    • 조건부 Flow Matching은 모델을 \(v_\theta(x_t,t,c)\) 형태로 확장해 조건에 맞는 velocity를 예측하게 한다.
    • 클래스 조건은 embedding으로, 텍스트 조건은 text encoder와 attention 구조로 모델에 주입할 수 있다.
    • Classifier-free guidance는 조건부 velocity와 무조건 velocity를 조합해 조건 반영을 강화한다.
    • Guidance scale이 커지면 조건 일치도는 높아질 수 있지만 다양성과 자연스러움이 떨어질 수 있다.
    • 조건 dropout, null condition 표현, velocity norm 모니터링은 guidance 구현에서 중요한 점검 항목이다.
