    부록 E. 시각 자료 제작 체크리스트¶

    최초 작성: 2026-05-23 02:18 KST 최종 업데이트: 2026-05-24 00:47 KST

    부록의 목적¶

    이 책은 텍스트와 수식만으로 읽히는 책이 아니라, 분포와 경로, 속도장, sampling 과정을 그림으로 함께 이해하는 책을 목표로 한다. 이 부록은 각 장에 어떤 시각 자료가 필요한지, 어떤 우선순위로 제작하면 좋은지 정리한다.

    1. 우선 제작할 핵심 그림¶

    대표 표지 이미지¶

    • 파일: docs/assets/cover-flow-matching-robotics.svg
    • 목적: noise prior, probability flow, structured data, robot action trajectory를 한 화면에 보여 준다.
    • 사용 위치: 머릿말 상단

    생성 모델 역사 지도¶

    • 사용 장: 0장
    • 내용: VAE, GAN, normalizing flow, diffusion, Flow Matching의 흐름
    • 목적: Flow Matching이 갑자기 등장한 개념이 아니라 생성 모델 문제의식의 연장임을 보여 준다.

    Probability path 그림¶

    • 사용 장: 3장, 6장
    • 내용: \(p_0\)에서 \(p_1\)로 이동하는 중간 분포 \(p_t\)
    • 목적: 분포가 시간에 따라 움직인다는 핵심 직관을 만든다.

    Conditional path와 marginal path 비교¶

    • 사용 장: 2장, 7장
    • 내용: 조건부 경로 여러 개가 전체 marginal flow를 만드는 모습
    • 목적: CFM의 핵심인 조건부 target과 marginal velocity의 관계를 시각화한다.

    CFM loss 파이프라인¶

    • 사용 장: 8장
    • 내용: \(x_0\), \(x_1\), \(t\)에서 \(x_t\)와 \(u_t\)를 만들고 모델 출력과 MSE(Mean Squared Error, 평균제곱오차)로 비교하는 흐름
    • 목적: 수식을 코드 구조로 연결한다.

    Diffusion vs Flow Matching 비교 그림¶

    • 사용 장: 9장
    • 내용: diffusion의 reverse denoising과 Flow Matching의 ODE velocity flow 비교
    • 목적: 두 방법의 공통점과 차이를 한 장에서 정리한다.

    2D velocity field와 trajectory¶

    • 사용 장: 10장
    • 내용: toy data에서 학습된 vector field와 샘플 trajectory
    • 목적: Flow Matching이 실제로 무엇을 학습하는지 눈으로 보여 준다.

    ODE solver step 비교¶

    • 사용 장: 14장
    • 내용: Euler step 수에 따른 trajectory 또는 샘플 품질 차이
    • 목적: NFE와 품질 trade-off를 설명한다.

    로보틱스 action chunk flow¶

    • 사용 장: 18장
    • 내용: 관측 \(o\)를 조건으로 noise action chunk \(A_0\)가 전문가 action chunk \(A_1\)로 흐르는 구조
    • 목적: 이미지 생성의 Flow Matching이 로봇 행동 생성으로 어떻게 바뀌는지 보여 준다.

    3. 시각 스타일 기준¶

    • 장식용 그림보다 개념 이해를 돕는 그림을 우선한다.
    • 한 그림에는 하나의 핵심 메시지만 담는다.
    • 색상은 noise, velocity, data, action을 구분하는 데 일관되게 사용한다.
    • 수식과 그림의 기호를 맞춘다. 예를 들어 그림의 중간 샘플은 본문과 동일하게 \(x_t\)로 표기한다.
    • 로보틱스 그림에서는 action, trajectory, observation, condition을 명확히 구분한다.
    • 작은 화면에서도 읽히도록 복잡한 텍스트를 그림 안에 과도하게 넣지 않는다.

    4. 제작 우선순위¶

    첫 번째 업데이트에서는 다음 5개를 우선 제작하면 책의 이해도가 크게 올라간다.

    1. 생성 모델 역사 지도
    2. Probability path 그림
    3. Conditional path와 marginal path 비교
    4. CFM loss pipeline
    5. 로보틱스 action chunk flow

    두 번째 업데이트에서는 toy example 실행 결과, ODE solver 비교, guidance scale 비교, VLA action expert 구조도를 추가한다.

    5. 그림 검수 체크리스트¶

    • 그림이 본문의 특정 문단과 직접 연결되는가?
    • 그림의 기호가 본문 수식과 일치하는가?
    • 캡션 없이도 대략의 의미가 보이는가?
    • 캡션을 읽으면 핵심 메시지가 명확한가?
    • 모바일 폭에서도 글자가 겹치지 않는가?
    • 색상만으로 정보를 구분하지 않고 위치, 선 종류, 라벨도 함께 쓰는가?
    • 생성형 이미지라면 개념 오류나 이상한 텍스트가 없는가?
    • 코드로 만든 그래프라면 데이터와 축 라벨이 재현 가능한가?
