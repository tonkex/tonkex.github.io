    부록의 목적¶

    이 부록은 본문에서 반복해서 등장하는 수식 기호와 핵심 용어를 한곳에 모은 것이다. Flow Matching과 diffusion 논문은 같은 대상을 서로 다른 표기로 설명하는 경우가 많다. 처음 읽을 때는 표기만 달라져도 완전히 다른 개념처럼 느껴질 수 있으므로, 이 부록을 참고해 기호의 역할을 먼저 확인하면 좋다.

    Flow Matching 용어¶

    Probability Path¶

    Probability path는 시간에 따라 변하는 분포들의 가족이다.

    \[ \{p_t\}_{t\in[0,1]}. \]

    Flow Matching에서는 시작 분포 \(p_0\)와 목표 분포 \(p_1\) 사이를 어떤 중간 분포들이 잇는지 먼저 생각한다. 이 경로를 어떻게 정하느냐가 target velocity와 sampling 효율에 영향을 준다.

    Velocity Field¶

    Velocity field는 위치 \(x\)와 시간 \(t\)에서 샘플이 어느 방향으로 움직여야 하는지 알려 주는 벡터장이다.

    \[ \frac{dx_t}{dt}=u_t(x_t). \]

    Flow Matching은 이 velocity field를 모델 \(v_\theta(x,t)\)가 맞추도록 학습한다.

    Conditional Flow Matching¶

    Conditional Flow Matching은 조건부 경로에서 계산하기 쉬운 target velocity를 샘플링해 학습하는 방식이다.

    \[ \mathcal{L}_{\mathrm{CFM}}(\theta)=\mathbb{E}\left[\left\|v_\theta(x_t,t)-u_t(x_t|z)\right\|^2\right]. \]

    조건부 target을 학습해도 MSE(Mean Squared Error, 평균제곱오차)의 조건부 평균 성질 때문에 marginal velocity를 배울 수 있다는 점이 핵심이다.

    Simulation-Free Training¶

    학습 중에 ODE(Ordinary Differential Equation, 상미분방정식) trajectory 전체를 매번 적분하지 않고, 임의의 시간 \(t\)에서 \(x_t\)와 target velocity를 직접 만들어 회귀 학습하는 방식을 말한다. 생성할 때는 여전히 ODE solver를 사용한다.

    Path Sampler¶

    Path sampler는 \(x_0\), \(x_1\), \(t\)를 받아 중간 샘플 \(x_t\)와 목표 속도 \(u_t\)를 반환하는 코드 단위다.

    xt, ut = path.sample(x0, x1, t)

    경로 실험을 쉽게 하려면 이 부분을 trainer와 분리하는 것이 좋다.

    Continuous Normalizing Flow¶

    Continuous Normalizing Flow는 normalizing flow의 변환을 연속 시간 ODE로 표현하는 모델 계열이다. 샘플은 velocity field를 따라 이동하고, density 변화는 divergence 항으로 계산한다. Flow Matching은 이 ODE/velocity field 언어를 공유하지만, 학습 목표를 likelihood 중심 simulation에서 velocity regression으로 바꾼다.

    Simulation-Based Training¶

    CNF처럼 학습 중 모델이 정의한 ODE trajectory를 직접 풀고, 그 결과를 통해 likelihood 또는 loss와 gradient를 계산하는 방식을 말한다. 모델 업데이트마다 solver가 개입하므로 비용, 메모리, 수치 안정성 문제가 커질 수 있다. Flow Matching의 simulation-free training은 이 부담을 줄이려는 문제의식에서 이해할 수 있다.

    Flux¶

    Flux \(J_t(x)\)는 확률 질량이 공간에서 흘러가는 양과 방향이다. Continuity equation은 \(\partial_t p_t(x)+\nabla\cdot J_t(x)=0\)처럼 쓸 수 있다. Flow Matching에서는 보통 \(J_t(x)=p_t(x)u_t(x)\)처럼 velocity field가 확률 질량을 운반한다고 본다.

    Liouville Equation¶

    확산항 없이 deterministic velocity field가 density를 이동시키는 경우의 continuity equation을 Liouville equation 관점으로 볼 수 있다. Flow Matching의 ODE 생성 과정을 이해할 때 유용한 배경이다.

    Fokker-Planck Equation¶

    SDE가 만드는 확률분포의 시간 변화를 설명하는 PDE다. Drift와 diffusion 항이 함께 등장한다. Diffusion model은 이 관점에서 자연스럽게 설명되고, Flow Matching과 diffusion을 같은 확률 질량 보존 언어에서 비교할 때도 유용하다.

    Diffusion 관련 용어¶

    Forward Noising Process¶

    데이터에 점진적으로 noise를 섞어 단순한 noise 분포로 보내는 과정이다. 보통 diffusion에서는 이 과정이 직접 설계된다.

    Reverse Process¶

    noise에서 시작해 data로 돌아오는 생성 과정이다. diffusion model은 이 역방향 denoising을 학습한다.

    Score¶

    Score는 log density의 입력에 대한 gradient다.

    \[ \nabla_x \log p_t(x). \]

    현재 위치에서 density가 증가하는 방향을 나타낸다. diffusion의 reverse dynamics에서 핵심 역할을 한다.

    Probability Flow ODE¶

    Score 기반 SDE(Stochastic Differential Equation, 확률미분방정식)와 같은 marginal distribution을 갖는 ODE 관점이다. diffusion과 Flow Matching이 ODE 언어로 만나는 지점 중 하나다.

    로보틱스 관련 용어¶

    Action Chunk¶

    한 번에 생성하는 여러 step의 action sequence다.

    \[ A=(a_1,a_2,\ldots,a_H). \]

    로봇 정책에서는 매 순간 단일 action만 예측하기보다 action chunk를 생성하고 일부를 실행한 뒤 다시 계획하는 방식이 많이 쓰인다.

    VLA¶

    Vision-Language-Action 모델의 약자다. 이미지와 언어 지시를 입력으로 받아 로봇 행동을 출력하는 모델 계열을 말한다.

    Action Expert¶

    VLA(Vision-Language-Action) 구조에서 시각/언어 backbone이 만든 표현을 받아 실제 연속 action을 생성하는 모듈이다. 최근 Flow Matching은 이 action expert로 자주 쓰인다.

    Dynamic Feasibility¶

    생성된 trajectory가 실제 로봇의 속도, 가속도, joint limit, 물리 제약 안에서 실행 가능한지를 뜻한다.

    Equivariance¶

    입력 장면이 회전하거나 이동할 때 출력 행동도 그 변화에 맞게 변하는 성질이다. 로봇 조작에서는 공간 일반화를 위해 중요하다.

    헷갈리기 쉬운 구분¶

    구분 설명
    score vs velocity score는 density가 증가하는 방향이고, velocity는 path를 따라 샘플이 움직이는 방향이다.
    path vs trajectory path는 분포 또는 조건부 생성 과정의 설계이고, trajectory는 특정 샘플이 시간에 따라 지나간 실제 경로다.
    training time vs sampling time 학습에서는 임의의 \(t\)에서 target velocity를 회귀하고, sampling에서는 ODE를 적분한다.
    noise prediction vs velocity prediction diffusion은 noise 또는 score를 예측할 수 있고, Flow Matching은 path의 velocity를 예측한다.
    action regression vs action generation 단일 평균 action을 예측하는 것과 가능한 action distribution에서 샘플링하는 것은 다르다.
