    부록 A. 참고문헌과 읽기 지도¶

    최초 작성: 2026-05-23 02:10 KST 최종 업데이트: 2026-05-23 23:20 KST

    부록의 목적¶

    이 부록은 본문에서 언급한 주요 자료를 독자가 다시 찾아 읽을 수 있도록 정리한 참고문헌 지도다. 단순한 논문 목록이 아니라, 각 자료가 이 책의 어느 질문에 답하는지 중심으로 묶었다.

    입문 독자는 모든 논문을 처음부터 끝까지 읽을 필요가 없다. 먼저 본문을 읽고, 막히는 개념이 있을 때 해당 묶음의 자료를 참고하는 방식이 좋다. 특히 수학이 부담된다면 abstract, introduction, figure, method overview를 먼저 읽고, theorem이나 긴 증명은 나중으로 미루어도 된다.

    1. Flow Matching 핵심 자료¶

    Flow Matching for Generative Modeling¶

    • 링크: arXiv:2210.02747
    • 저자: Yaron Lipman, Ricky T. Q. Chen, Heli Ben-Hamu, Maximilian Nickel, Matt Le
    • 본문 연결: 6장, 7장, 8장, 13장, 15장
    • 읽을 질문: Flow Matching은 어떤 의미에서 simulation-free CNF(Continuous Normalizing Flow) training인가?

    이 논문은 Flow Matching의 출발점으로 읽을 자료다. 핵심은 fixed conditional probability path의 vector field를 회귀해 Continuous Normalizing Flow를 학습한다는 관점이다. Gaussian probability path, diffusion path, optimal transport path를 같은 틀에서 비교할 수 있게 해 준다.

    처음 읽을 때는 다음을 확인하자.

    • probability path를 어떻게 정의하는가?
    • target vector field는 어떻게 만들어지는가?
    • diffusion path와 OT(Optimal Transport, 최적 수송) path를 어떤 기준으로 비교하는가?
    • 학습 때 ODE(Ordinary Differential Equation, 상미분방정식) simulation을 피한다는 말이 정확히 무엇을 뜻하는가?

    Flow Matching Guide and Code¶

    • 링크: Meta AI Research publication
    • 본문 연결: 전체 입문 흐름, 특히 6-10장
    • 읽을 질문: 수식과 코드 사이를 어떻게 연결할 수 있는가?

    가이드 성격의 자료로, 원 논문보다 입문자에게 접근성이 좋다. 본문에서 독자가 수식의 의미를 코드의 tensor와 연결하는 데 어려움을 느낄 때 참고하기 좋다.

    2. Diffusion과 score 기반 생성 모델¶

    Denoising Diffusion Probabilistic Models¶

    • 링크: arXiv:2006.11239
    • 저자: Jonathan Ho, Ajay Jain, Pieter Abbeel
    • 본문 연결: 0장, 4장, 5장, 9장, 15장
    • 읽을 질문: diffusion model은 왜 denoising objective로 학습되는가?

    DDPM(Denoising Diffusion Probabilistic Model)은 diffusion model의 대표적인 출발점이다. 이 책에서는 모든 변분 추론 세부를 따라가기보다, forward noising과 reverse denoising, noise prediction objective, 점진적 sampling이라는 아이디어를 이해하는 데 초점을 둔다.

    Score-Based Generative Modeling through Stochastic Differential Equations¶

    • 링크: arXiv:2011.13456
    • 저자: Yang Song, Jascha Sohl-Dickstein, Diederik P. Kingma, Abhishek Kumar, Stefano Ermon, Ben Poole
    • 본문 연결: 4장, 5장, 9장, 14장, 15장
    • 읽을 질문: score, reverse-time SDE(Stochastic Differential Equation, 확률미분방정식), probability flow ODE는 어떻게 연결되는가?

    Flow Matching과 diffusion을 비교하려면 이 자료가 중요하다. score 기반 모델이 SDE와 ODE 관점에서 어떻게 표현되는지 보여 주며, 9장에서 다룬 “두 방법이 어디서 만나고 어디서 갈라지는가”를 이해하는 데 도움이 된다.

    3. Rectified Flow와 경로 설계¶

    Flow Straight and Fast¶

    • 링크: arXiv:2209.03003
    • 본문 연결: 13장, 14장, 15장, 17장
    • 읽을 질문: 경로를 더 곧게 만들면 sampling 효율이 왜 좋아질 수 있는가?

    Rectified Flow는 trajectory를 더 straight하게 만들려는 문제의식과 연결된다. Flow Matching의 path parameterization을 공부한 뒤 읽으면, 경로 설계가 단순한 수식 선택이 아니라 sampling 비용과 품질을 좌우한다는 점이 잘 보인다.

    4. 구현과 실험을 위한 보조 자료¶

    MIT 6.S184 Generative AI with Stochastic Differential Equations¶

    • 로컬 자료: raw data/[DR4R_260516] MIT Class 6.S184_ Generative AI With Stochastic Differential Equations, 2026 An Introduction to Flow Matching and Diffusion Models.pdf
    • 본문 연결: 4장, 5장, 9장, 14장
    • 읽을 질문: SDE와 diffusion을 입문 강의 흐름으로 어떻게 이해할 수 있는가?

    이 자료는 수식 난이도가 있지만, diffusion과 Flow Matching을 같은 연속 시간 생성 모델 흐름에서 보는 데 도움이 된다. 수학이 약한 독자는 본문 1-5장을 읽은 뒤 필요한 부분만 골라 읽는 편이 좋다.

    FM가이드-Yaron@Meta¶

    • 로컬 자료: raw data/FM가이드-Yaron@Meta.pdf
    • 본문 연결: 6장, 7장, 8장, 13장
    • 읽을 질문: Flow Matching 핵심 수식을 어떤 직관으로 받아들일 수 있는가?

    이 자료는 Flow Matching의 핵심 설명과 코드 감각을 보완하는 데 사용한다. 원 논문보다 직관적 설명을 먼저 보고 싶을 때 참고한다.

    NF-CNF-FM 모델 진화 발표 자료¶

    • 로컬 자료: raw data/NF-CNF-FM모델진화-FMvsDIFFU관계-발표.html
    • 본문 연결: 0장, 5장, 6장, 9장, 13장, 15장
    • 읽을 질문: Normalizing Flow, Neural ODE, CNF, Flow Matching은 어떤 문제의식으로 이어지는가?
    • 읽을 질문: CNF의 simulation-based training 병목은 Flow Matching의 simulation-free training과 어떻게 대비되는가?
    • 읽을 질문: Fokker-Planck/continuity equation 관점에서 diffusion과 Flow Matching의 공통 뿌리를 어떻게 설명할 수 있는가?

    이 자료는 발표형 HTML이라 시각적 비유가 많다. 본문에서는 역사적 계보, flux와 확률 질량 보존, simulation-free training, diffusion/Flow Matching 비교를 보강하는 데 사용한다.

    5. 로보틱스와 AI 응용 자료¶

    Riemannian Flow Matching Policy for Robot Motion Learning¶

    • 링크: arXiv:2403.10672
    • 본문 연결: 17장, 18장
    • 읽을 질문: 로봇 상태가 manifold 위에 있을 때 Flow Matching policy는 어떻게 달라져야 하는가?

    RFMP는 로봇 상태 공간의 기하 구조를 정책에 반영하려는 흐름이다. 로보틱스에서는 manifold가 추상 수학이 아니라 pose, rotation, configuration space와 연결된 실제 문제라는 점을 보여 준다.

    Affordance-based Robot Manipulation with Flow Matching¶

    • 링크: arXiv:2409.01083
    • 본문 연결: 18장
    • 읽을 질문: perception이 찾은 affordance를 action trajectory 생성에 어떻게 연결할 수 있는가?

    이 논문은 affordance understanding과 Flow Matching 기반 action trajectory 생성을 결합한다. AI 로보틱스 독자에게는 perception module과 action generation module이 어떻게 함께 설계되는지 보는 사례로 유용하다.

    ActionFlow¶

    • 링크: arXiv:2409.04576
    • 본문 연결: 18장
    • 읽을 질문: 공간 대칭성과 SE(3) 관계를 action generation policy에 어떻게 넣을 수 있는가?

    ActionFlow는 Flow Matching과 spatial symmetry inductive bias를 결합한다. 로봇 조작에서 일반화를 위해 단순히 데이터를 늘리는 것뿐 아니라 구조적 bias를 넣는 방향을 보여 준다.

    \(\pi_0\): A Vision-Language-Action Flow Model for General Robot Control¶

    • 링크: arXiv:2410.24164
    • 본문 연결: 18장
    • 읽을 질문: VLM(Vision-Language Model) backbone과 Flow Matching action expert는 어떻게 결합되는가?

    \(\pi_0\)는 Flow Matching이 robot foundation model의 action 생성 모듈로 들어간 대표 사례다. 언어와 시각 이해를 담당하는 backbone 위에 연속 action을 생성하는 flow model을 붙이는 구조를 이해하는 데 중요하다.

    FlowMP¶

    • 링크: arXiv:2503.06135
    • 본문 연결: 14장, 18장
    • 읽을 질문: robot motion planning에서 acceleration-aware flow는 왜 중요한가?

    FlowMP는 motion planning에서 second-order trajectory dynamics를 고려하는 Conditional Flow Matching 접근이다. 로봇 trajectory는 단순히 목적지에 도달하면 끝이 아니라 smoothness와 physical executability가 중요하다는 점을 잘 보여 준다.

    Safe Flow Matching¶

    • 링크: arXiv:2504.08661
    • 본문 연결: 18장
    • 읽을 질문: Flow Matching planner에 safety constraint를 어떻게 결합할 수 있는가?

    Safe Flow Matching은 control barrier function을 통해 안전한 trajectory generation을 다룬다. 로봇 적용에서는 생성 모델의 다양성과 안전 보장이 함께 필요하다는 문제의식을 보여 준다.

    VFP: Variational Flow-Matching Policy for Multi-Modal Robot Manipulation¶

    • 링크: arXiv:2508.01622
    • 본문 연결: 18장
    • 읽을 질문: multi-modal manipulation policy에서 평균 행동으로 붕괴되는 문제를 어떻게 줄일 수 있는가?

    VFP는 variational latent prior, optimal transport alignment, mixture-of-experts decoder를 통해 multi-modal robot manipulation을 다룬다. 로봇 조작에서 가능한 성공 경로가 여러 개인 상황을 이해하는 데 좋다.

    ManiFlow¶

    • 링크: arXiv:2509.01819
    • 본문 연결: 14장, 18장
    • 읽을 질문: action generation을 1-2 inference step으로 줄이려면 어떤 training 구조가 필요한가?

    ManiFlow는 consistency flow training을 통해 dexterous action generation을 빠르게 하려는 흐름이다. 로봇 제어 주기에 맞는 latency와 high-dimensional action generation을 함께 고민할 때 참고할 만하다.

    AsyncVLA¶

    • 링크: arXiv:2511.14148
    • 본문 연결: 18장
    • 읽을 질문: VLA(Vision-Language-Action)의 Flow Matching action generation에서 uniform time schedule은 왜 문제가 될 수 있는가?

    AsyncVLA는 synchronous Flow Matching의 고정 시간 스케줄을 long-horizon task의 불안정성 원인으로 보고, action context-aware asynchronous schedule과 self-correction을 제안한다. VLA와 Flow Matching이 결합된 뒤 새로 생기는 문제를 보여 주는 자료다.

    Morphologically Equivariant Flow Matching for Bimanual Mobile Manipulation¶

    • 링크: papers.cool summary
    • 본문 연결: 18장
    • 읽을 질문: 양팔 로봇의 좌우 대칭을 policy 구조에 넣으면 어떤 일반화 이점이 생기는가?

    2026년 5월 기준 최신 흐름 중 하나다. 공개 초록 기준으로 \(\mathbb{C}_2\)-equivariant Flow Matching policy를 사용해 mirrored configuration에 대한 zero-shot generalization과 sample efficiency를 개선하려는 방향을 제안한다.

    SmolVLA¶

    • 링크: Hugging Face blog, arXiv:2506.01844
    • 본문 연결: 18장
    • 읽을 질문: Flow Matching action expert를 더 작고 접근 가능한 VLA(Vision-Language-Action) 시스템에 어떻게 넣을 수 있는가?

    SmolVLA는 compact VLA와 flow-matching decoder를 결합해 저비용 하드웨어와 공개 데이터 기반 로보틱스 연구를 지향한다. 로보틱스와 AI에 관심 있는 개발자에게는 재현 가능성과 접근성 측면에서 중요한 사례다.

    6. 읽기 순서 추천¶

    처음 읽는 독자에게는 다음 순서를 권한다.

    1. 본문 0-9장을 먼저 읽는다.
    2. Flow Matching for Generative Modeling의 abstract와 introduction을 읽는다.
    3. diffusion 배경이 약하면 DDPM(Denoising Diffusion Probabilistic Model)과 Score-Based SDE의 introduction만 먼저 본다.
    4. 본문 10-14장의 구현과 solver를 읽는다.
    5. path 설계가 궁금하면 Flow Straight and Fast를 읽는다.
    6. 로보틱스와 AI에 관심이 많다면 18장을 읽은 뒤 \(\pi_0\), RFMP, FlowMP, SafeFM 순서로 본다.
    7. VLA와 최신 action expert 흐름이 궁금하면 SmolVLA, ManiFlow, AsyncVLA를 이어서 본다.

    7. 참고문헌을 읽을 때의 기록 템플릿¶

    논문을 읽을 때는 다음 항목을 기록하면 본문과 연결하기 쉽다.

    항목 기록할 내용
    문제의식 기존 방법의 어떤 한계를 해결하려 하는가?
    데이터 공간 이미지, action chunk, trajectory, manifold, discrete state 중 무엇인가?
    조건 class, text, image, robot state, goal, obstacle 중 무엇을 조건으로 쓰는가?
    경로 \(x_t\) 또는 \(A_t\)를 어떻게 정의하는가?
    target noise, score, velocity, transition 중 무엇을 맞추는가?
    sampling ODE, SDE, discrete transition, NFE, latency는 어떻게 되는가?
    실험 어떤 benchmark와 real-world setting에서 검증했는가?
    한계 safety, compute, data scale, generalization, implementation complexity 중 무엇이 남는가?
