    부록 B. 최소 수식으로 보는 KV Cache¶

    KV cache 크기는 정확한 모델 구조에 따라 달라진다. 이 부록의 목적은 완벽한 계산기가 아니라, 어떤 변수가 memory를 키우는지 감각을 잡는 것이다.

    간단한 근사는 다음과 같이 생각할 수 있다.

    KV cache bytes
≈ batch_size
  x sequence_length
  x num_layers
  x 2
  x num_kv_heads
  x head_dim
  x bytes_per_element

    여기서 2는 Key와 Value 두 종류를 의미한다.

    B.1 KV cache 크기 근사¶

    B.1.1 Batch size¶

    동시에 처리하는 sequence 수다. Batch size가 2배가 되면 KV cache도 대체로 2배가 된다.

    B.1.2 Sequence length¶

    Context에 포함된 token 수다. Prompt와 생성된 output token이 모두 포함된다. Sequence length가 길어지면 KV cache가 선형으로 증가한다.

    B.1.3 Layer 수¶

    Transformer layer마다 Key와 Value를 저장한다. Layer 수가 많을수록 KV cache도 커진다.

    B.1.4 KV head 수¶

    MHA, GQA, MQA 구조에 따라 KV head 수가 달라진다. GQA나 MQA는 Query head 수보다 KV head 수를 줄여 KV cache memory를 줄일 수 있다.

    B.1.5 Precision¶

    KV cache element 하나가 몇 byte인지 결정한다. FP16/BF16은 보통 2 byte다. KV cache quantization을 적용하면 더 줄일 수 있지만 품질과 kernel 지원을 확인해야 한다.

    B.2 예제 계산¶

    B.2.1 7B 모델의 KV cache 감각¶

    예를 들어 layer 32개, KV head 32개, head dimension 128, FP16, batch size 1, sequence length 8K라고 가정해 보자.

    1 x 8192 x 32 x 2 x 32 x 128 x 2 bytes
≈ 4.29 GB

    실제 모델 구조가 GQA를 사용하면 KV head 수가 더 작아져 이보다 줄어들 수 있다. 중요한 것은 8K context 하나도 GB 단위 cache를 만들 수 있다는 감각이다.

    B.2.2 70B 모델의 KV cache 감각¶

    큰 모델은 layer 수와 dimension이 커지는 경향이 있다. Weight memory만 문제가 아니라 KV cache도 커진다. 70B 모델을 serving할 때는 quantization으로 weight를 줄여도 long-context batch에서 KV cache가 병목이 될 수 있다.

    B.2.3 128K context에서의 KV cache 감각¶

    8K에서 128K로 context가 늘면 sequence length가 16배다. 다른 조건이 같다면 KV cache도 16배가 된다. 8K에서 4GB였던 cache가 128K에서는 수십 GB가 될 수 있다.

    이 때문에 long-context serving에서는 max context length를 현실적으로 제한하고, prefix caching, KV cache quantization, CP/DCP 같은 전략을 검토한다.

    B.3 MHA, MQA, GQA, MLA 비교¶

    B.3.1 Multi-head Attention¶

    MHA는 여러 attention head가 각각 Query, Key, Value를 가진다. KV head 수가 많아 KV cache memory가 커질 수 있다.

    B.3.2 Multi-query Attention¶

    MQA는 여러 Query head가 더 적은 Key/Value head를 공유한다. KV cache memory를 줄일 수 있지만 모델 구조와 품질 trade-off가 있다.

    B.3.3 Grouped-query Attention¶

    GQA는 MHA와 MQA의 중간 형태로, Query head들을 여러 group으로 묶어 Key/Value를 공유한다. 많은 modern LLM에서 KV cache 효율을 위해 사용된다.

    B.3.4 Multi-head Latent Attention¶

    MLA는 Key/Value 표현을 더 압축된 latent 형태로 다루어 KV cache 부담을 줄이려는 구조다. 모델별 구현 차이가 크므로 serving 엔진 지원을 확인해야 한다.
