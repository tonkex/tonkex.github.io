    16мһҘ. LLM м„ңлІ„лҠ” м–ҙл–»кІҢ кө¬м„ұлҗҳлҠ”к°Җ¶

    LLM м„ңлІ„лҠ” лӢЁмҲңнһҲ лӘЁлҚё к°қмІҙм—җ generate()лҘј нҳём¶ңн•ҳлҠ” н”„лЎңк·ёлһЁмқҙ м•„лӢҲлӢӨ. мҡҙмҳҒ нҷҳкІҪм—җм„ңлҠ” API мҡ”мІӯмқ„ л°ӣкі , tokenizationмқ„ мҲҳн–үн•ҳкі , м—¬лҹ¬ мҡ”мІӯмқ„ schedulingн•ҳкі , GPU workerм—җм„ң prefillкіј decodeлҘј мӢӨн–үн•ҳкі , tokenмқ„ streamingмңјлЎң лҸҢл ӨмӨҖлӢӨ.

    мқҙ мһҘм—җм„ңлҠ” vLLM, SGLang, TensorRT-LLM к°ҷмқҖ serving мӢңмҠӨн…ңмқ„ мқҙн•ҙн•ҳкё° мң„н•ң кіөнҶө кө¬мЎ°лҘј мӮҙнҺҙліёлӢӨ.

    н•ҷмҠө лӘ©н‘ң¶

    • LLM serving мӢңмҠӨн…ңмқҳ мЈјмҡ” кө¬м„ұ мҡ”мҶҢлҘј мқҙн•ҙн•ңлӢӨ.
    • мҡ”мІӯмқҙ tokenize, prefill, decode, streamingмңјлЎң мқҙм–ҙм§ҖлҠ” нқҗлҰ„мқ„ м„ӨлӘ…н•ңлӢӨ.
    • к°ңл°ңмһҗк°Җ нҷ•мқён•ҙм•ј н•ҳлҠ” кё°ліё м„Өм •мқ„ нҢҢм•…н•ңлӢӨ.

    16.1 кё°ліё кө¬м„ұ мҡ”мҶҢ¶

    LLM м„ңлІ„лҠ” м—¬лҹ¬ м»ҙнҸ¬л„ҢнҠёк°Җ нҳ‘л Ҙн•ҙ лҸҷмһ‘н•ңлӢӨ. мһ‘мқҖ мӢӨмҠөм—җм„ңлҠ” н•ҳлӮҳмқҳ н”„лЎңм„ёмҠӨ м•Ҳм—җ лӘЁл‘җ л“Өм–ҙ мһҲмқ„ мҲҳ мһҲм§Җл§Ң, к°ңл…җм ҒмңјлЎңлҠ” API server, tokenizer, scheduler, workerлҘј лӮҳлҲ„м–ҙ ліј мҲҳ мһҲлӢӨ.

    16.1.1 API server¶

    API serverлҠ” мҷёл¶Җ мҡ”мІӯмқ„ л°ӣлҠ” мһ…кө¬лӢӨ. OpenAI-compatible APIлҘј м ңкіөн•ҳлҠ” кІҪмҡ° /v1/chat/completions к°ҷмқҖ endpointлҘј нҶөн•ҙ мҡ”мІӯмқ„ л°ӣлҠ”лӢӨ. API serverлҠ” мқёмҰқ, rate limit, request validation, streaming connection кҙҖлҰ¬лҘј лӢҙлӢ№н•  мҲҳ мһҲлӢӨ.

    к°ңл°ңмһҗ мһ…мһҘм—җм„ңлҠ” API serverк°Җ л°ӣлҠ” parameterк°Җ мӢӨм ң м—”м§„ м„Өм •кіј м–ҙл–»кІҢ м—°кІ°лҗҳлҠ”м§Җ мқҙн•ҙн•ҙм•ј н•ңлӢӨ. max_tokens, temperature, stream, stop, response_format к°ҷмқҖ к°’мқҙ decode лҸҷмһ‘кіј latencyм—җ мҳҒн–Ҙмқ„ мӨҖлӢӨ.

    16.1.2 Scheduler¶

    SchedulerлҠ” м–ҙл–Ө мҡ”мІӯмқ„ м–ём ң GPUм—җ л„Јмқ„м§Җ кІ°м •н•ңлӢӨ. LLM servingм—җм„ң schedulerлҠ” л§Өмҡ° мӨ‘мҡ”н•ҳлӢӨ. мҡ”мІӯл§ҲлӢӨ prompt кёёмқҙ, output кёёмқҙ, KV cache нҒ¬кё°, priorityк°Җ лӢӨлҘҙкё° л•Ңл¬ёмқҙлӢӨ.

    мўӢмқҖ schedulerлҠ” GPUлҘј лҶҖлҰ¬м§Җ м•Ҡмңјл©ҙм„ңлҸ„ latency SLOлҘј м§ҖнӮӨл Өкі  н•ңлӢӨ. Continuous batching, chunked prefill, KV block allocation к°ҷмқҖ кё°лҠҘмқҙ schedulerмҷҖ м—°кІ°лҗңлӢӨ.

    16.1.3 Worker¶

    WorkerлҠ” мӢӨм ң лӘЁлҚёмқ„ GPUм—җ мҳ¬лҰ¬кі  prefillкіј decodeлҘј мҲҳн–үн•ңлӢӨ. лӢЁмқј GPU workerмқј мҲҳлҸ„ мһҲкі , Tensor Parallel лҳҗлҠ” Pipeline ParallelлЎң л¬¶мқё м—¬лҹ¬ GPU workerмқј мҲҳлҸ„ мһҲлӢӨ.

    WorkerлҠ” model weights, KV cache, temporary bufferлҘј кҙҖлҰ¬н•ңлӢӨ. GPU memory utilization м„Өм •мқҙ л„Ҳл¬ҙ лҶ’мңјл©ҙ лҚ” л§ҺмқҖ мҡ”мІӯмқ„ мІҳлҰ¬н•  мҲҳ мһҲм§Җл§Ң OOM мң„н—ҳлҸ„ м»Өм§„лӢӨ.

    16.1.4 Tokenizer¶

    TokenizerлҠ” н…ҚмҠӨнҠёлҘј token idлЎң л°”кҫјлӢӨ. Chat templateмқ„ м Ғмҡ©н•ҳкі , system/user/assistant л©”мӢңм§ҖлҘј лӘЁлҚёмқҙ кё°лҢҖн•ҳлҠ” нҳ•мӢқмңјлЎң ліҖнҷҳн•ңлӢӨ. TokenizationмқҖ CPUм—җм„ң мҲҳн–үлҗҳлҠ” кІҪмҡ°к°Җ л§Һм•„, кёҙ promptмҷҖ лҶ’мқҖ мҡ”мІӯлҹүм—җм„ңлҠ” CPU лі‘лӘ©мқҙ лҗ  мҲҳ мһҲлӢӨ.

    Tokenizer м„Өм •мқҙ лӘЁлҚёкіј л§һм§Җ м•Ҡмңјл©ҙ н’Ҳм§Ҳ л¬ём ңлҝҗ м•„лӢҲлқј prefix caching мӢӨнҢЁлҸ„ мғқкёё мҲҳ мһҲлӢӨ. к°ҷмқҖ н…ҚмҠӨнҠёлқјлҸ„ templateмқҙ лӢӨлҘҙл©ҙ token sequenceк°Җ лӢ¬лқјм§Ҳ мҲҳ мһҲлӢӨ.

    16.2 мҡ”мІӯ мІҳлҰ¬ нқҗлҰ„¶

    LLM мҡ”мІӯмқҖ лҢҖлһө request мҲҳмӢ , tokenization, scheduling, prefill, decode, streaming мҲңм„ңлЎң мІҳлҰ¬лҗңлӢӨ. к°Ғ лӢЁкі„лҠ” м„ңлЎң лӢӨлҘё лі‘лӘ©мқ„ л§Ңл“Ө мҲҳ мһҲлӢӨ.

    16.2.1 Requestк°Җ л“Өм–ҙмҳЁлӢӨ¶

    мӮ¬мҡ©мһҗмқҳ requestм—җлҠ” prompt лҳҗлҠ” chat messages, generation parameter, stream м—¬л¶Җ, output м ңн•ң л“ұмқҙ нҸ¬н•ЁлҗңлӢӨ. API serverлҠ” мқҙ мҡ”мІӯмқ„ кІҖмҰқн•ҳкі  лӮҙл¶Җ request к°қмІҙлЎң ліҖнҷҳн•ңлӢӨ.

    мқҙ лӢЁкі„м—җм„ң rate limit, мқёмҰқ, tenantлі„ quotaлҸ„ м Ғмҡ©лҗ  мҲҳ мһҲлӢӨ. мҡҙмҳҒ м„ңл№„мҠӨм—җм„ңлҠ” лӘЁлҚё м„ұлҠҘл§ҢнҒјмқҙлӮҳ мҡ”мІӯ м ңм–ҙк°Җ мӨ‘мҡ”н•ҳлӢӨ.

    16.2.2 Tokenizeн•ңлӢӨ¶

    н…ҚмҠӨнҠёлҠ” token id sequenceлЎң ліҖнҷҳлҗңлӢӨ. мқҙл•Ң chat template, tool schema, system promptк°Җ н•Ёк»ҳ л“Өм–ҙк°Ҳ мҲҳ мһҲлӢӨ. Token мҲҳк°Җ кі„мӮ°лҗҳкі , max model lengthлҘј мҙҲкіјн•ҳлҠ”м§Җ нҷ•мқён•ңлӢӨ.

    Token мҲҳлҠ” мқҙнӣ„ schedulingкіј л№„мҡ© кі„мӮ°мқҳ кё°мӨҖмқҙ лҗңлӢӨ.

    16.2.3 Prefillмқ„ мҲҳн–үн•ңлӢӨ¶

    WorkerлҠ” prompt tokenмқ„ мІҳлҰ¬н•ҳкі  KV cacheлҘј мғқм„ұн•ңлӢӨ. Prefillмқҙ кёҙ мҡ”мІӯмқҖ TTFTлҘј мҰқк°ҖмӢңнӮ¬ мҲҳ мһҲлӢӨ. SchedulerлҠ” кёҙ prefillмқҙ decode мӨ‘мқё мҡ”мІӯмқ„ кіјлҸ„н•ҳкІҢ л°©н•ҙн•ҳм§Җ м•ҠлҸ„лЎқ chunked prefill к°ҷмқҖ м „лһөмқ„ мӮ¬мҡ©н•  мҲҳ мһҲлӢӨ.

    16.2.4 DecodeлҘј л°ҳліөн•ңлӢӨ¶

    мІ« token мқҙнӣ„м—җлҠ” decode loopк°Җ л°ҳліөлҗңлӢӨ. к°Ғ stepм—җм„ң лӢӨмқҢ token logitsлҘј кі„мӮ°н•ҳкі , sampling лҳҗлҠ” constrained decodingмңјлЎң tokenмқ„ м„ нғқн•ңлӢӨ. мғҲ tokenмқҳ KV cacheлҸ„ м¶”к°ҖлҗңлӢӨ.

    DecodeлҠ” TPOTмҷҖ streaming мІҙк°җ мҶҚлҸ„м—җ м§Ғм ‘ м—°кІ°лҗңлӢӨ.

    16.2.5 StreamingмңјлЎң мқ‘лӢөн•ңлӢӨ¶

    Streamingмқҙ мјңм ё мһҲмңјл©ҙ м„ңлІ„лҠ” мғқм„ұлҗң token лҳҗлҠ” text chunkлҘј нҒҙлқјмқҙм–ёнҠём—җ мҲңм°Ём ҒмңјлЎң ліҙлӮёлӢӨ. StreamingмқҖ м „мІҙ мғқм„ұ мӢңк°„мқ„ мӨ„мқҙм§ҖлҠ” м•Ҡм§Җл§Ң, мӮ¬мҡ©мһҗк°Җ мІ« кІ°кіјлҘј л№ЁлҰ¬ ліј мҲҳ мһҲкІҢ н•ҙ мӨҖлӢӨ.

    Streaming кө¬нҳ„м—җм„ңлҠ” backpressure, client disconnect, partial response loggingлҸ„ кі л Өн•ҙм•ј н•ңлӢӨ.

    16.3 к°ңл°ңмһҗк°Җ нҷ•мқён•ҙм•ј н•  м„Өм •¶

    LLM м„ңлІ„лҘј лқ„мҡё л•Ң мһҗмЈј л§ҢлӮҳлҠ” м„Өм •мқҖ м„ұлҠҘкіј м•Ҳм •м„ұм—җ м§Ғм ‘ мҳҒн–Ҙмқ„ мӨҖлӢӨ. кё°ліёк°’мқ„ к·ёлҢҖлЎң м“°кё°ліҙлӢӨ workloadм—җ л§һкІҢ мқҙн•ҙн•ҳкі  мЎ°м •н•ҙм•ј н•ңлӢӨ.

    16.3.1 Model path¶

    Model pathлҠ” мӮ¬мҡ©н•  model weightsмҷҖ tokenizerлҘј к°ҖлҰ¬нӮЁлӢӨ. Hugging Face repository мқҙлҰ„мқј мҲҳлҸ„ мһҲкі , лЎңм»¬ л””л үн„°лҰ¬мқј мҲҳлҸ„ мһҲлӢӨ. лӘЁлҚёкіј tokenizerк°Җ л§һм§Җ м•Ҡмңјл©ҙ н’Ҳм§Ҳкіј tokenization л¬ём ңк°Җ мғқкёҙлӢӨ.

    16.3.2 Max model length¶

    Max model lengthлҠ” н—Ҳмҡ©н•  мөңлҢҖ context lengthлӢӨ. л„Ҳл¬ҙ лӮ®мңјл©ҙ кёҙ мҡ”мІӯмқ„ мІҳлҰ¬н•  мҲҳ м—Ҷкі , л„Ҳл¬ҙ лҶ’мңјл©ҙ KV cache memory мҳҲм•Ҫкіј OOM мң„н—ҳмқҙ м»Өм§Ҳ мҲҳ мһҲлӢӨ. мӢӨм ң workloadмқҳ token л¶„нҸ¬лҘј ліҙкі  м„Өм •н•ҙм•ј н•ңлӢӨ.

    16.3.3 GPU memory utilization¶

    GPU memory utilizationмқҖ м—”м§„мқҙ GPU memoryлҘј м–јл§ҲлӮҳ кіөкІ©м ҒмңјлЎң мӮ¬мҡ©н• м§Җ м •н•ҳлҠ” м„Өм •мқҙлӢӨ. к°’мқ„ лҶ’мқҙл©ҙ лҚ” л§ҺмқҖ KV cache кіөк°„мқ„ нҷ•ліҙн•  мҲҳ мһҲм§Җл§Ң, temporary bufferмҷҖ fragmentation м—¬мң к°Җ мӨ„м–ҙ OOM мң„н—ҳмқҙ м»Өм§Ҳ мҲҳ мһҲлӢӨ.

    16.3.4 Tensor parallel size¶

    Tensor parallel sizeлҠ” лӘЁлҚё layer лӮҙл¶Җ кі„мӮ°мқ„ лӘҮ GPUм—җ лӮҳлҲҢм§Җ м •н•ңлӢӨ. лӘЁлҚёмқҙ GPU н•ң мһҘм—җ л“Өм–ҙк°Җм§Җ м•Ҡкұ°лӮҳ latencyлҘј мӨ„мқҙкі  мӢ¶мқ„ л•Ң мЎ°м •н•ңлӢӨ. н•ҳм§Җл§Ң TP sizeк°Җ м»Өм§ҲмҲҳлЎқ нҶөмӢ  л№„мҡ©мқҙ лҠҳ мҲҳ мһҲмңјлҜҖлЎң interconnectмҷҖ лӘЁлҚё нҒ¬кё°лҘј н•Ёк»ҳ лҙҗм•ј н•ңлӢӨ.
