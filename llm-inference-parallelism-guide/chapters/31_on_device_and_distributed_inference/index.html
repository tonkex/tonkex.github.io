    31장. On-device와 분산 추론¶

    LLM 추론은 cloud GPU에서만 일어나는 것이 아니다. 작은 모델과 quantization이 발전하면서 노트북, 스마트폰, edge device에서도 LLM을 실행할 수 있게 되었다. 동시에 여러 장치를 묶어 추론하려는 실험도 있다.

    이 장에서는 on-device, 개인용 분산 추론, edge-cloud hybrid 구조를 입문자 관점에서 정리한다.

    학습 목표¶

    • On-device 추론의 장점과 제약을 이해한다.
    • 개인용 분산 추론과 production 분산 추론을 구분한다.
    • Edge와 cloud의 역할 분담을 설명한다.

    31.1 On-device 추론¶

    On-device 추론은 사용자의 장치에서 모델을 직접 실행하는 방식이다. 개인정보 보호, offline 사용, 낮은 network latency 측면에서 장점이 있다.

    31.1.1 개인정보 보호와 latency 장점¶

    입력이 외부 서버로 나가지 않으므로 민감한 데이터를 다루기 쉽다. 네트워크 왕복이 줄어 특정 작업에서는 응답성이 좋아질 수 있다. Offline 환경에서도 사용할 수 있다.

    31.1.2 메모리와 연산 성능 제약¶

    장치의 memory와 compute는 cloud GPU보다 제한적이다. 큰 모델을 실행하기 어렵고, 긴 context나 높은 throughput을 기대하기 어렵다. Battery와 발열도 고려해야 한다.

    31.1.3 작은 모델과 quantization이 중요하다¶

    On-device에서는 작은 모델과 quantization이 핵심이다. INT4 같은 낮은 precision 모델을 사용하고, context length를 제한하며, task를 좁히면 실용성이 올라간다.

    31.2 개인용 분산 추론¶

    개인용 분산 추론은 여러 장치를 묶어 하나의 모델을 실행하려는 아이디어다. 예를 들어 여러 PC나 노트북의 GPU/CPU를 연결해 큰 모델을 나누어 실행하는 방식이다.

    31.2.1 여러 장치를 묶는 아이디어¶

    모델 weight나 layer를 여러 장치에 나누면 단일 장치보다 큰 모델을 실행할 수 있을 것처럼 보인다. 개념적으로는 TP나 PP와 비슷한 부분이 있다.

    31.2.2 네트워크가 병목이 되기 쉽다¶

    문제는 네트워크다. LLM 병렬화는 GPU 간 빠른 통신을 요구하는 경우가 많다. 일반 가정용 네트워크나 Wi-Fi는 latency와 bandwidth가 부족할 수 있다. 결과적으로 계산보다 데이터 이동이 더 큰 병목이 된다.

    31.2.3 실험과 production은 구분해야 한다¶

    개인용 분산 추론은 학습과 실험에는 흥미롭지만, 안정적인 production serving과는 다르다. 장애 처리, 재현성, 통신 성능, 보안, monitoring이 부족할 수 있다.

    31.3 Edge와 cloud의 역할 분담¶

    실무에서는 모든 것을 on-device로 하거나 모든 것을 cloud로 보내는 이분법보다 hybrid 구조가 현실적일 수 있다.

    31.3.1 간단한 작업은 edge에서 처리¶

    짧은 분류, 간단한 요약, 개인정보가 민감한 전처리, UI 보조 기능은 edge에서 처리할 수 있다. 작은 모델로 충분한 task를 cloud로 보내지 않으면 latency와 비용을 줄일 수 있다.

    31.3.2 복잡한 작업은 cloud로 위임¶

    긴 문서 분석, 큰 모델이 필요한 reasoning, 고품질 코드 생성, 대규모 batch 처리는 cloud GPU가 적합하다. Cloud는 더 큰 모델, 더 긴 context, 더 높은 throughput을 제공할 수 있다.

    31.3.3 Hybrid inference 구조¶

    Hybrid 구조에서는 edge model이 요청을 분류하거나 초안을 만들고, 필요할 때 cloud model로 위임한다. 개인정보가 필요한 부분은 local에서 처리하고, 일반화된 reasoning은 cloud에서 처리할 수 있다.

    이 구조에서는 routing policy, fallback, 데이터 보안, latency budget을 설계해야 한다.

    정리¶

    On-device 추론은 privacy와 offline 사용성에서 장점이 있지만 memory와 compute 제약이 크다. 개인용 분산 추론은 실험적으로 흥미롭지만 네트워크가 병목이 되기 쉽다. 실무에서는 edge와 cloud가 역할을 나누는 hybrid 구조가 현실적인 선택이 될 수 있다.
