    6мһҘ. мІ« LLM server: OpenAI-compatible endpoint л§Ңл“Өкё°¶

    Draft v0.1 Updated: 2026-07-08 Part 3 - лӢЁмқј GPUм—җм„ң servingмқ„ мёЎм •н•ҳкё°

    лӘЁлҚёмқ„ APIлЎң лқ„мҡ°лҠ” мІ« мӢӨмҠөм—җм„ң к°ҖмһҘ нқ”н•ң м°©к°ҒмқҖ "мқ‘лӢөмқҙ мҳӨл©ҙ м„ұкіө"мқҙлқјкі  ліҙлҠ” кІғмқҙлӢӨ. л¬јлЎ  мІ« мқ‘лӢөмқҖ мӨ‘мҡ”н•ҳлӢӨ. н•ҳм§Җл§Ң AI мқён”„лқј м—”м§ҖлӢҲм–ҙм—җкІҢ мІ« serverмқҳ лӘ©н‘ңлҠ” "лҗңлӢӨ"к°Җ м•„лӢҲлқј "мёЎм •лҗңлӢӨ"мқҙлӢӨ.

    мқҙ мһҘм—җм„ңлҠ” vLLMмқҳ OpenAI-compatible serverлҘј кё°мӨҖмңјлЎң мөңмҶҢ serving stackмқ„ л§Ңл“ лӢӨ. н•өмӢ¬мқҖ лӘ…л №м–ҙ н•ҳлӮҳлҘј мҷёмҡ°лҠ” кІғмқҙ м•„лӢҲлӢӨ. model artifact, tokenizer, runtime, scheduler, GPU memory, HTTP endpoint, client SDKк°Җ м–ҙл–Ө кҙҖкі„мқём§Җ мқҙн•ҙн•ҳлҠ” кІғмқҙлӢӨ. к°ҷмқҖ /v1/chat/completions endpointлҘј м—ҙм–ҙлҸ„ лӮҙл¶Җ runtime м„Өм •мқҙ лӢӨлҘҙл©ҙ capacityмҷҖ latencyлҠ” мҷ„м „нһҲ лӢ¬лқјм§„лӢӨ.

    мқҙ мһҘмқҳ лҸ…мһҗ м§Ҳл¬ёмқҖ лӢӨмқҢкіј к°ҷлӢӨ.

    лӘЁлҚёмқ„ лЎңм»¬ лҳҗлҠ” м„ңлІ„ GPUм—җм„ң APIлЎң лқ„мҡ°л Өл©ҙ мөңмҶҢ кө¬м„ұмқҖ л¬ҙм—Үмқҙкі , мІ« мӢӨн—ҳм—җм„ң л¬ҙм—Үмқ„ кё°лЎқн•ҙм•ј н•ҳлҠ”к°Җ?

    OpenAI-compatible endpointк°Җ мҲЁкё°лҠ” кІғ¶

    OpenAI-compatible endpointлҠ” application к°ңл°ңмһҗм—җкІҢ нҺён•ҳлӢӨ. кё°мЎҙ OpenAI SDKлҘј кұ°мқҳ к·ёлҢҖлЎң м“°кі , base_urlл§Ң л°”кҫёл©ҙ local serverлӮҳ private GPU serverлҘј нҳём¶ңн•  мҲҳ мһҲлӢӨ. vLLM quickstartлҸ„ OpenAI API protocolмқ„ кө¬нҳ„н•ҳлҠ” serverлЎң л°°нҸ¬н•  мҲҳ мһҲкі , кё°ліём ҒмңјлЎң http://localhost:8000м—җм„ң мӢңмһ‘н•ҳл©° --host, --portлЎң мЈјмҶҢлҘј м§Җм •н•  мҲҳ мһҲлӢӨкі  м„ӨлӘ…н•ңлӢӨ.

    н•ҳм§Җл§Ң мқҙ нҺёлҰ¬н•ЁмқҖ лҸҷмӢңм—җ мң„н—ҳн•ҳлӢӨ. API лӘЁм–‘мқҙ к°ҷлӢӨкі  лӮҙл¶Җ systemмқҙ к°ҷмқҖ кІғмқҖ м•„лӢҲлӢӨ.

    кІүмңјлЎң ліҙмқҙлҠ” кІғ мӢӨм ңлЎң кІ°м •н•ҙм•ј н•ҳлҠ” кІғ
    /v1/chat/completions tokenizer, chat template, model config
    model л¬ёмһҗм—ҙ weight мң„м№ҳ, precision, trust policy
    messages л°°м—ҙ prompt serialization, input token count
    max_tokens decode кёёмқҙ, KV cache мҲҳлӘ…
    temperature sampling policy, мһ¬нҳ„м„ұ
    HTTP мқ‘лӢө scheduler, GPU memory, queueing, metrics

    Application к°ңл°ңмһҗлҠ” endpoint contractлҘј ліёлӢӨ. AI мқён”„лқј м—”м§ҖлӢҲм–ҙлҠ” endpoint м•„лһҳмқҳ capacity contractлҘј лҙҗм•ј н•ңлӢӨ.

    мІ« serverмқҳ кө¬м„ұ мҡ”мҶҢ¶

    мІ« LLM serverлҘј лқ„мҡё л•Ң н•„мҡ”н•ң кө¬м„ұ мҡ”мҶҢлҠ” м—¬м„Ҝ к°Җм§ҖлӢӨ.

    Model artifact¶

    Model artifactлҠ” лӢЁмҲңнһҲ weight нҢҢмқј н•ҳлӮҳк°Җ м•„лӢҲлӢӨ. ліҙнҶө лӢӨмқҢмқ„ нҸ¬н•Ён•ңлӢӨ.

    • model weights
    • model config
    • tokenizer
    • tokenizer config
    • chat template
    • generation config
    • special token м •мқҳ

    Serving runtimeмқҖ мқҙ л¬¶мқҢмқ„ мқҪм–ҙ modelмқ„ мҙҲкё°нҷ”н•ңлӢӨ. tokenizerмҷҖ chat templateмқҙ л§һм§Җ м•Ҡмңјл©ҙ к°ҷмқҖ messagesлҘј ліҙлӮҙлҸ„ мҷ„м „нһҲ лӢӨлҘё prompt token sequenceк°Җ л§Ңл“Өм–ҙм§Ҳ мҲҳ мһҲлӢӨ. л”°лқјм„ң model artifactлҠ” "лӘЁлҚё нҢҢмқј"мқҙ м•„лӢҲлқј "мһ…л Ҙмқ„ м–ҙл–Ө token sequenceлЎң л°”кҝҖм§Җк№Ңм§Җ нҸ¬н•Ён•ң мӢӨн–ү лӢЁмң„"лЎң лҙҗм•ј н•ңлӢӨ.

    Runtime¶

    RuntimeмқҖ modelмқ„ GPUм—җм„ң мӢӨн–үн•ҳлҠ” м—”м§„мқҙлӢӨ. мқҙ мһҘм—җм„ңлҠ” vLLMмқ„ кё°мӨҖмңјлЎң м„ӨлӘ…н•ңлӢӨ. vLLMмқҖ OpenAI-compatible serverлҘј м ңкіөн•ҳкі , PagedAttention, scheduler, batching, KV cache кҙҖлҰ¬, metrics к°ҷмқҖ serving кё°лҠҘмқ„ нҸ¬н•Ён•ңлӢӨ.

    Runtimeмқ„ м„ нғқн•ҳл©ҙ лӢӨмқҢ м§Ҳл¬ёлҸ„ л”°лқјмҳЁлӢӨ.

    • м–ҙл–Ө attention backendлҘј м“ё кІғмқёк°Җ?
    • GPU memory utilizationмқ„ м–ҙлҠҗ м •лҸ„к№Ңм§Җ н—Ҳмҡ©н•  кІғмқёк°Җ?
    • max model lengthлҘј м–ҙл–»кІҢ м ңн•ңн•  кІғмқёк°Җ?
    • prefix caching, quantization, speculative decoding к°ҷмқҖ кё°лҠҘмқ„ мјӨ кІғмқёк°Җ?
    • metricsлҘј м–ҙл””лЎң л…ём¶ңн•ҳкі  м–ҙл–»кІҢ мҲҳм§‘н•  кІғмқёк°Җ?

    мІ« мӢӨмҠөм—җм„ңлҠ” кё°лҠҘмқ„ л§Һмқҙ мјңм§Җ м•ҠлҠ”лӢӨ. baselineмқ„ лЁјм Җ л§Ңл“ лӢӨ.

    HTTP endpoint¶

    HTTP endpointлҠ” applicationкіј runtime мӮ¬мқҙмқҳ кі„м•ҪмқҙлӢӨ. vLLM quickstart кё°мӨҖмңјлЎң serverлҠ” OpenAI API нҳ•мӢқмңјлЎң model list, chat completion, completion endpointлҘј м ңкіөн•ңлӢӨ. л”°лқјм„ң OpenAI SDKлҘј м“°лҠ” applicationмқҖ base_urlмқ„ local serverлЎң л°”кҝ” нҳём¶ңн•  мҲҳ мһҲлӢӨ.

    н•ҳм§Җл§Ң endpointлҘј м—ҙм—ҲлӢӨкі  production serverк°Җ лҗң кІғмқҖ м•„лӢҲлӢӨ. мқёмҰқ, rate limit, timeout, logging, tracing, load balancing, model rolloutмқҖ м•„м§Ғ м—ҶлӢӨ. мІ« serverлҠ” "мӢӨн—ҳ к°ҖлҠҘн•ң лӢЁмқј runtime"мқҙлӢӨ.

    Client¶

    ClientлҠ” curlмқј мҲҳлҸ„ мһҲкі  Python OpenAI SDKмқј мҲҳлҸ„ мһҲлӢӨ. мІ« мӢӨн—ҳм—җм„ңлҠ” л‘ҳ лӢӨ мҚЁ ліҙлҠ” кІғмқҙ мўӢлӢӨ.

    • curlмқҖ HTTP payloadмҷҖ мқ‘лӢөмқ„ м§Ғм ‘ нҷ•мқён•ҳкё° мўӢлӢӨ.
    • SDKлҠ” мӢӨм ң application integrationкіј к°Җк№Ңмҡҙ нҳ•нғңлҘј нҷ•мқён•ҳкё° мўӢлӢӨ.

    Clientм—җм„ң л°ҳл“ңмӢң кё°лЎқн•ҙм•ј н•ҳлҠ” кІғмқҖ мқ‘лӢө textк°Җ м•„лӢҲлӢӨ. request payload, latency, token count, error м—¬л¶ҖлӢӨ.

    GPU memory¶

    5мһҘм—җм„ң ліё кІғмІҳлҹј GPU memoryм—җлҠ” weightл§Ң мҳ¬лқјк°Җм§Җ м•ҠлҠ”лӢӨ. KV cache, runtime overhead, temporary buffer, allocator м—¬мң л¶„мқҙ н•Ёк»ҳ л“Өм–ҙк°„лӢӨ. мІ« serverм—җм„ң nvidia-smiл§Ң ліҙлҠ” кІғмқҖ л¶ҖмЎұн•ҳм§Җл§Ң, к·ёлһҳлҸ„ мӢңмһ‘м җмңјлЎңлҠ” мң мҡ©н•ҳлӢӨ.

    мІ« мӢӨн—ҳм—җм„ңлҠ” лӢӨмқҢмқ„ кё°лЎқн•ңлӢӨ.

    • server мӢңмһ‘ м§Ғнӣ„ GPU memory
    • мІ« мҡ”мІӯ нӣ„ GPU memory
    • кёҙ prompt мҡ”мІӯ нӣ„ GPU memory
    • лҸҷмӢң мҡ”мІӯ лҳҗлҠ” л°ҳліө мҡ”мІӯ нӣ„ GPU memory

    м •нҷ•н•ң мӣҗмқё л¶„м„қмқҖ runtime metricsмҷҖ н•Ёк»ҳ н•ҙм•ј н•ҳм§Җл§Ң, GPU memory ліҖнҷ” к°җк°ҒмқҖ мҙҲл°ҳм—җ кјӯ мһЎм•„м•ј н•ңлӢӨ.

    Metrics¶

    vLLM production metrics л¬ём„ңлҠ” OpenAI-compatible API serverмқҳ /metrics endpointлҘј нҶөн•ҙ system healthлҘј лӘЁлӢҲн„°л§Ғн•  мҲҳ мһҲлӢӨкі  м„ӨлӘ…н•ңлӢӨ. мқҙ metricsм—җлҠ” request success, KV cache usage, running/waiting requests, e2e latency, inter-token latency, prefill/decode time, TTFT л“ұмқҙ нҸ¬н•ЁлҗңлӢӨ.

    мІ« server мӢӨн—ҳм—җм„ң metricsлҘј ліҙлҠ” мқҙмң лҠ” к°„лӢЁн•ҳлӢӨ. "лҠҗлҰ¬лӢӨ"лҘј "queueм—җм„ң кё°лӢӨл ёлӢӨ", "prefillмқҙ кёём—ҲлӢӨ", "decodeк°Җ лҠҗл ёлӢӨ", "KV cacheк°Җ м°јлӢӨ"лЎң лӮҳлҲ„кё° мң„н•ҙм„ңлӢӨ.

    мөңмҶҢ мӢӨн–ү нқҗлҰ„¶

    vLLM quickstartмқҳ online serving мҳҲмӢңлҠ” лӢӨмқҢ нҳ•нғңлЎң serverлҘј мӢңмһ‘н•ңлӢӨ.

    vllm serve Qwen/Qwen2.5-1.5B-Instruct

    мқҙ лӘ…л №мқҖ Hugging Face model idлҘј л°ӣм•„ serverлҘј мӢңмһ‘н•ңлӢӨ. кіөмӢқ quickstartлҠ” мқҙ мҳҲмӢңм—җм„ң Qwen/Qwen2.5-1.5B-InstructлҘј мӮ¬мҡ©н•ңлӢӨ. мӢӨм ң мӢӨмҠөм—җм„ңлҠ” GPU VRAM, network access, license, мЎ°м§Ғ м •мұ…м—җ л§һлҠ” мһ‘мқҖ instruct modelмқ„ м„ нғқн•ҙм•ј н•ңлӢӨ.

    мІ« мӢӨн–үм—җм„ңлҠ” лӢӨмқҢмІҳлҹј мҳөм…ҳмқ„ лӘ…мӢңн•ҳлҠ” нҺёмқҙ мўӢлӢӨ.

    vllm serve Qwen/Qwen2.5-1.5B-Instruct \
  --host 0.0.0.0 \
  --port 8000 \
  --generation-config vllm

    --generation-config vllmмқ„ л„ЈлҠ” мқҙмң лҠ” baselineмқ„ л¶„лӘ…н•ҳкІҢ л§Ңл“Өкё° мң„н•ҙм„ңлӢӨ. vLLM quickstartлҠ” Hugging Face repositoryм—җ generation_config.jsonмқҙ мһҲмңјл©ҙ кё°ліё sampling parameterк°Җ model creator к¶ҢмһҘк°’мңјлЎң overrideлҗ  мҲҳ мһҲкі , мқҙлҘј л№„нҷңм„ұнҷ”н•ҳл Өл©ҙ --generation-config vllmмқ„ л„ҳкё°лқјкі  м„ӨлӘ…н•ңлӢӨ. мІ« мёЎм •м—җм„ңлҠ” мҲЁмқҖ кё°ліёк°’мқ„ мӨ„мқҙлҠ” нҺёмқҙ мўӢлӢӨ.

    мқҙ лӘ…л №мқҖ production к¶ҢмһҘ м„Өм •мқҙ м•„лӢҲлӢӨ. мІ« baselineмқ„ л§Ңл“Өкё° мң„н•ң мөңмҶҢ мӢңмһ‘м җмқҙлӢӨ.

    Serverк°Җ л–ҙлҠ”м§Җ нҷ•мқён•ҳкё°¶

    м„ңлІ„к°Җ лңЁл©ҙ лЁјм Җ model list endpointлҘј нҷ•мқён•ңлӢӨ.

    curl http://localhost:8000/v1/models

    мқҙ мҡ”мІӯмқҖ modelмқҙ мӢӨм ңлЎң API serverм—җ л“ұлЎқлҗҗлҠ”м§Җ нҷ•мқён•ңлӢӨ. м—¬кё°м„ң мӢӨнҢЁн•ҳл©ҙ chat completionмқ„ ліј н•„мҡ”к°Җ м—ҶлӢӨ. лЁјм Җ server log, model download, GPU memory, port binding, Python environmentлҘј нҷ•мқён•ҙм•ј н•ңлӢӨ.

    лӢӨмқҢмңјлЎң chat completionмқ„ ліҙлӮёлӢӨ.

    curl http://localhost:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "Qwen/Qwen2.5-1.5B-Instruct",
    "messages": [
      {"role": "system", "content": "You are a concise assistant."},
      {"role": "user", "content": "KV cacheлҘј н•ң л¬ёмһҘмңјлЎң м„ӨлӘ…н•ҙ мӨҳ."}
    ],
    "max_tokens": 128,
    "temperature": 0
  }'

    мІ« мқ‘лӢөм—җм„ң нҷ•мқён•  кІғмқҖ м„ё к°Җм§ҖлӢӨ.

    1. HTTP statusк°Җ м •мғҒмқёк°Җ?
    2. мқ‘лӢө schemaк°Җ applicationмқҙ кё°лҢҖн•ҳлҠ” нҳ•нғңмқёк°Җ?
    3. usage лҳҗлҠ” token countлҘј кё°лЎқн•  мҲҳ мһҲлҠ”к°Җ?

    лӮҙмҡ© н’Ҳм§Ҳ нҸүк°ҖлҠ” лӮҳмӨ‘ л¬ём ңлӢӨ. м§ҖкёҲмқҖ server contractмҷҖ мёЎм • к°ҖлҠҘм„ұмқҙ лЁјм ҖлӢӨ.

    Python SDKлЎң нҳём¶ңн•ҳкё°¶

    Applicationкіј к°Җк№Ңмҡҙ нҳ•нғңлҠ” Python OpenAI SDKлӢӨ. vLLM quickstartлҸ„ base_urlмқ„ local vLLM serverлЎң л°”кҫёлҠ” мҳҲмӢңлҘј м ңкіөн•ңлӢӨ.

    from openai import OpenAI

client = OpenAI(
    api_key="EMPTY",
    base_url="http://localhost:8000/v1",
)

response = client.chat.completions.create(
    model="Qwen/Qwen2.5-1.5B-Instruct",
    messages=[
        {"role": "system", "content": "You are a concise assistant."},
        {"role": "user", "content": "KV cacheлҘј н•ң л¬ёмһҘмңјлЎң м„ӨлӘ…н•ҙ мӨҳ."},

    м—¬кё°м„ң api_key="EMPTY"лҠ” local testм—җм„ң нқ”нһҲ м“°лҠ” placeholderлӢӨ. мӢӨм ң shared serverм—җм„ңлҠ” key м—Ҷмқҙ м—ҙм–ҙ л‘җл©ҙ м•Ҳ лҗңлӢӨ. vLLM quickstartлҠ” --api-key лҳҗлҠ” VLLM_API_KEYлЎң API key кІҖмӮ¬лҘј мјӨ мҲҳ мһҲлӢӨкі  м„ӨлӘ…н•ңлӢӨ. нҳјмһҗ м“°лҠ” localhostмҷҖ нҢҖмқҙ м“°лҠ” serverлҠ” security contractк°Җ лӢӨлҘҙлӢӨ.

    мІ« мӢӨн—ҳм—җм„ң кё°лЎқн•  н‘ң¶

    мІ« server мӢӨн—ҳм—җм„ң к°ҖмһҘ мӨ‘мҡ”н•ң мӮ°м¶ңл¬јмқҖ мӢӨн–ү лӘ…л №мқҙ м•„лӢҲлқј кё°лЎқн‘ңлӢӨ.

    лӢӨмқҢ н‘ңлҘј л§Ңл“ лӢӨ.

    н•ӯлӘ© кё°лЎқ мҳҲмӢң мҷң н•„мҡ”н•ңк°Җ
    model id Qwen/Qwen2.5-1.5B-Instruct model configмҷҖ tokenizer м¶”м Ғ
    server command vllm serve ... мһ¬нҳ„м„ұ
    GPU A10, L4, A100 л“ұ hardware л№„көҗ
    dtype/quantization BF16, FP16, AWQ л“ұ memoryмҷҖ kernel мҳҒн–Ҙ
    max model length runtime м„Өм •к°’ KV cache budget
    prompt type short chat, long RAG л“ұ workload л¶„лҰ¬
    input tokens prompt token count prefill/KV cache
    output tokens generated token count decode мӢңк°„
    TTFT мІ« tokenк№Ңм§Җ мӢңк°„ мӮ¬мҡ©мһҗ мІҙк°җ мӢңмһ‘м җ
    TPOT output tokenлӢ№ мӢңк°„ decode лі‘лӘ©
    total latency м „мІҙ мқ‘лӢө мӢңк°„ product SLA
    server metrics snapshot /metrics мқјл¶Җ queue, cache, request state

    мқҙ н‘ңк°Җ мһҲм–ҙм•ј 7мһҘм—җм„ң benchmarkлҘј мқҳлҜё мһҲкІҢ мқҪмқ„ мҲҳ мһҲлӢӨ.

    TTFTлҘј к°„лӢЁнһҲ мһ¬лҠ” streaming client¶

    curlл§ҢмңјлЎңлҸ„ лҢҖлһөм Ғмқё м „мІҙ latencyлҠ” ліј мҲҳ мһҲлӢӨ. н•ҳм§Җл§Ң TTFTлҘј ліҙл Өл©ҙ streaming мқ‘лӢөмқҳ мІ« chunk мӢңк°„мқ„ кё°лЎқн•ҳлҠ” нҺёмқҙ мўӢлӢӨ. м•„лһҳ мҪ”л“ңлҠ” мӢӨмҠөмҡ© skeletonмқҙлӢӨ.

    import time
from openai import OpenAI

client = OpenAI(api_key="EMPTY", base_url="http://localhost:8000/v1")

started = time.perf_counter()
first_token_at = None
output_chars = 0

stream = client.chat.completions.create(
    model="Qwen/Qwen2.5-1.5B-Instruct",
    messages=[
        {"role": "system", "content": "You are a concise assistant."},
        {"role": "user", "content": "KV cacheмҷҖ prefill/decodeмқҳ кҙҖкі„лҘј м„ӨлӘ…н•ҙ мӨҳ."},
    ],
    max_tokens=256,
    temperature=0,
    stream=True,
)

for chunk in stream:
    delta = chunk.choices[0].delta.content or ""
    if delta and first_token_at is None:
        first_token_at = time.perf_counter()
    output_chars += len(delta)

ended = time.perf_counter()

print({
    "ttft_seconds": None if first_token_at is None else first_token_at - started,
    "total_latency_seconds": ended - started,
    "output_chars": output_chars,
})

    мқҙ мҪ”л“ңлҠ” мҷ„м „н•ң benchmarkк°Җ м•„лӢҲлӢӨ. token count лҢҖмӢ  character countлҘј мһ„мӢңлЎң кё°лЎқн•ҳкі , network overheadмҷҖ client overheadлҸ„ м„һм—¬ мһҲлӢӨ. н•ҳм§Җл§Ң мІ« мӢӨн—ҳм—җм„ңлҠ” м¶©л¶„нһҲ мўӢмқҖ м¶ңл°ңм җмқҙлӢӨ. лӢӨмқҢ мһҘм—җм„ң лҚ” м—„л°Җн•ң benchmark м„Өкі„лЎң л„ҳм–ҙк°„лӢӨ.

    /metricsлҘј к°ҷмқҙ лҙҗм•ј н•ҳлҠ” мқҙмң ¶

    Clientм—җм„ң ліҙмқҙлҠ” latencyл§ҢмңјлЎңлҠ” лі‘лӘ©мқ„ м•Ң мҲҳ м—ҶлӢӨ. vLLM metrics endpointлҘј к°ҷмқҙ нҷ•мқён•ңлӢӨ.

    curl http://localhost:8000/metrics

    мІ« мӢӨн—ҳм—җм„ң нҠ№нһҲ ліј л§Ңн•ң н•ӯлӘ©мқҖ лӢӨмқҢкіј к°ҷлӢӨ.

    metric н•ҙм„қ
    vllm:time_to_first_token_seconds TTFT histogram
    vllm:request_time_per_output_token_seconds requestлі„ TPOT
    vllm:e2e_request_latency_seconds end-to-end latency
    vllm:request_prefill_time_seconds prefill лӢЁкі„ мӢңк°„
    vllm:request_decode_time_seconds decode лӢЁкі„ мӢңк°„
    vllm:kv_cache_usage_perc KV cache мӮ¬мҡ©лҘ 
    vllm:num_requests_running мӢӨн–ү batch м•Ҳмқҳ request мҲҳ
    vllm:num_requests_waiting лҢҖкё° мӨ‘мқё request мҲҳ
    vllm:request_prompt_tokens prompt token л¶„нҸ¬
    vllm:request_generation_tokens generation token л¶„нҸ¬

    мқҙ metricsлҠ” 5мһҘм—җм„ң л§Ңл“  mental modelмқ„ мӢӨм ң serverм—җ л¶ҷм—¬ мӨҖлӢӨ. TTFTк°Җ лӮҳмҒңм§Җ, TPOTк°Җ лӮҳмҒңм§Җ, queueк°Җ мҢ“мқҙлҠ”м§Җ, KV cacheк°Җ м°јлҠ”м§Җ л¶„лҰ¬н•  мҲҳ мһҲлӢӨ.

    Serving configлҠ” codeк°Җ м•„лӢҲлқј capacity contractлӢӨ¶

    мІ« serverм—җм„ң м„Өм •к°’мқ„ л¬ҙмӢ¬мҪ” л„ҳкё°л©ҙ лӮҳмӨ‘м—җ мһ¬нҳ„мқҙ м–ҙл өлӢӨ. лӢӨмқҢ м„Өм •мқҖ л°ҳл“ңмӢң кё°лЎқн•ҙм•ј н•ңлӢӨ.

    model¶

    Model idлҠ” лӢЁмҲң мқҙлҰ„мқҙ м•„лӢҲлӢӨ. weight, config, tokenizer, chat template, generation configлҘј н•Ёк»ҳ к°ҖлҰ¬нӮЁлӢӨ. к°ҷмқҖ model familyлқјлҸ„ instruct/base, quantized/non-quantized, revisionм—җ л”°лқј serving кІ°кіјк°Җ лӢ¬лқјм§„лӢӨ.

    max_model_len¶

    мөңлҢҖ context lengthлҠ” product кё°лҠҘмІҳлҹј ліҙмқҙм§Җл§Ң мӮ¬мӢӨмқҖ KV cache budgetмқҙлӢӨ. л„Ҳл¬ҙ нҒ¬кІҢ мһЎмңјл©ҙ кёҙ requestлҘј л°ӣмқ„ мҲҳлҠ” мһҲм§Җл§Ң лҸҷмӢңм„ұмқҙ мӨ„кі  tail latencyк°Җ м•…нҷ”лҗ  мҲҳ мһҲлӢӨ.

    gpu_memory_utilization¶

    GPU memoryлҘј м–јл§ҲлӮҳ кіөкІ©м ҒмңјлЎң м“ём§Җ м •н•ңлӢӨ. лҶ’кІҢ мһЎмңјл©ҙ capacityлҠ” лҠҳ мҲҳ мһҲм§Җл§Ң OOM, fragmentation, лӢӨлҘё processмҷҖмқҳ м¶©лҸҢ мң„н—ҳлҸ„ м»Өм§„лӢӨ.

    dtypeкіј quantization¶

    Weight dtypeкіј quantizationмқҖ memory footprintмҷҖ kernel pathлҘј л°”кҫјлӢӨ. 5мһҘм—җм„ң лҙӨл“Ҝ KV cache dtypeлҸ„ лі„лҸ„ м¶•мқҙлӢӨ. "мһ‘м•„м§„лӢӨ"мҷҖ "л№Ёлқјм§„лӢӨ"лҠ” к°ҷмқҖ л§җмқҙ м•„лӢҲлӢӨ.

    generation_config¶

    Sampling кё°ліёк°’мқҙ model repositoryмқҳ generation_config.jsonм—җм„ң мҳ¬ мҲҳ мһҲлӢӨ. baseline мёЎм •м—җм„ңлҠ” мқҙ м°ЁмқҙлҘј лӘ…нҷ•нһҲ н•ҙм•ј н•ңлӢӨ. мӢӨн—ҳ кІ°кіјлҘј л№„көҗн•  л•Ң temperature, top_p, max_tokensк°Җ лӢ¬лқјм§Җл©ҙ latencyмҷҖ output lengthлҸ„ лӢ¬лқјм§„лӢӨ.

    API keyмҷҖ binding¶

    localhostм—җл§Ң лқ„мҡ°лҠ” мӢӨн—ҳкіј 0.0.0.0мңјлЎң нҢҖ л„ӨнҠёмӣҢнҒ¬м—җ м—¬лҠ” serverлҠ” ліҙм•Ҳ мқҳлҜёк°Җ лӢӨлҘҙлӢӨ. shared serverлқјл©ҙ API key, firewall, reverse proxy, TLS, access logлҘј лі„лҸ„ кіјм ңлЎң мһЎм•„м•ј н•ңлӢӨ.

    мІ« мӢӨн—ҳ workload м„ёнҠё¶

    мІ« serverлҘј лқ„мӣ лӢӨл©ҙ н•ҳлӮҳмқҳ promptл§Ң ліҙлӮҙм§Җ м•ҠлҠ”лӢӨ. мөңмҶҢ м„ё workloadлҘј ліҙлӮёлӢӨ.

    workload лӘ©м Ғ мҳҲмӢң
    short chat кё°ліё endpoint latency нҷ•мқё 200-500 input tokens, 100 output tokens
    long prompt prefillкіј KV cache pressure нҷ•мқё 4k-16k input tokens, м§§мқҖ лӢөліҖ
    repeated prefix prefix caching нӣ„ліҙ нҷ•мқё к°ҷмқҖ system/document prefix, лӢӨлҘё м§Ҳл¬ё

    м•„м§Ғ load testлҘј н•ҳм§Җ м•Ҡм•„лҸ„ лҗңлӢӨ. к°Ғ workloadлҘј 5-10нҡҢ л°ҳліөн•ҙ кё°лЎқн•ҳл©ҙ м¶©л¶„н•ҳлӢӨ. мқҙ мһ‘мқҖ н‘ңл§Ң мһҲм–ҙлҸ„ лӢӨмқҢ мһҘм—җм„ң benchmark кіЎм„ мқ„ мқҪмқ„ мӨҖл№„к°Җ лҗңлӢӨ.

    мӢӨнҢЁн•ҳл©ҙ м–ҙл””лҘј ліј кІғмқёк°Җ¶

    мІ« server мӢӨмҠөмқҖ мһҗмЈј мӢӨнҢЁн•ңлӢӨ. мӢӨнҢЁ мһҗмІҙлҠ” л¬ём ңк°Җ м•„лӢҲлӢӨ. л¬ём ңлҠ” мӢӨнҢЁк°Җ м–ҙлҠҗ кі„мёөм—җм„ң лӮ¬лҠ”м§Җ лӘЁлҘҙлҠ” кІғмқҙлӢӨ.

    мҰқмғҒ лЁјм Җ ліј кіі нқ”н•ң мӣҗмқё
    serverк°Җ мӢңмһ‘н•ҳм§Җ м•ҠмқҢ install, CUDA, model download Python/CUDA mismatch, network, disk
    model load мӨ‘ OOM GPU memory, dtype, model size weightк°Җ GPUм—җ мҳ¬лқјк°Җм§Җ м•ҠмқҢ
    /v1/models мӢӨнҢЁ server log, port binding server лҜёкё°лҸҷ, port м¶©лҸҢ
    chat template error tokenizer, model type template лҲ„лқҪ лҳҗлҠ” л©”мӢңм§Җ нҳ•мӢқ л¶Ҳмқјм№ҳ
    мІ« tokenмқҙ л„Ҳл¬ҙ лҠҗлҰј input token, queue, prefill кёҙ prompt, cold start
    token мғқм„ұмқҙ лҠҗлҰј TPOT, GPU utilization, backend decode лі‘лӘ©, attention backend
    к°ҷмқҖ prompt кІ°кіјк°Җ лӢӨлҰ„ sampling config temperature/top_p/default generation config

    мқҙ н‘ңлҠ” troubleshootingмқ„ л№ лҘҙкІҢ л§Ңл“ лӢӨ. "vLLMмқҙ м•Ҳ лҗңлӢӨ"ліҙлӢӨ "model load OOMмқҙлӢӨ", "chat template л¬ём ңлӢӨ", "decode backend л¬ём ңлӢӨ"лқјкі  л§җн•  мҲҳ мһҲм–ҙм•ј н•ңлӢӨ.

    llama.cppмҷҖ л№„көҗн•ҳл©ҙ л¬ҙм—Үмқҙ ліҙмқҙлҠ”к°Җ¶

    мқҙ мһҘмқҳ baselineмқҖ vLLMмқҙлӢӨ. н•ҳм§Җл§Ң local inferenceм—җм„ң llama.cpp к°ҷмқҖ runtimeмқ„ л§ҢлӮҳлҠ” кІҪмҡ°лҸ„ л§ҺлӢӨ. л‘җ лҸ„кө¬лҘј лӢЁмҲңнһҲ "м–ҙлҠҗ мӘҪмқҙ л№ лҘёк°Җ"лЎң л№„көҗн•ҳл©ҙ м•Ҳ лҗңлӢӨ.

    vLLMмқҖ GPU serverмҷҖ online serving baselineмңјлЎң мқҙн•ҙн•ҳлҠ” кІғмқҙ мўӢлӢӨ. OpenAI-compatible API, batching, scheduler, KV cache кҙҖлҰ¬, metrics, multi-GPU нҷ•мһҘ к°ҷмқҖ production serving мҡ”мҶҢлҘј лӢӨлЈ¬лӢӨ.

    llama.cppлҠ” local/edge inferenceмҷҖ GGUF ecosystemм—җм„ң к°•н•ҳлӢӨ. CPU, Apple Silicon, partial offload, local quantized model мӢӨн–ү к°ҷмқҖ л§ҘлқҪм—җм„ң мһҗмЈј м“°мқёлӢӨ.

    л№„көҗ м§Ҳл¬ёмқҖ лӢӨмқҢмІҳлҹј мһЎм•„м•ј н•ңлӢӨ.

    • лӮҙ лӘ©н‘ңлҠ” developer laptop local inferenceмқёк°Җ, shared GPU API serverмқёк°Җ?
    • лҸҷмӢң мҡ”мІӯмқ„ batchлЎң мІҳлҰ¬н•ҙм•ј н•ҳлҠ”к°Җ?
    • OpenAI-compatible APIл§Ң н•„мҡ”н•ңк°Җ, production metricsмҷҖ schedulerлҸ„ н•„мҡ”н•ңк°Җ?
    • model formatкіј quantization workflowлҠ” л¬ҙм—Үмқёк°Җ?
    • мҡҙмҳҒ лҢҖмғҒмқҙ лӢЁмқј мӮ¬мҡ©мһҗ desktopмқёк°Җ, м—¬лҹ¬ tenantк°Җ м“°лҠ” serverмқёк°Җ?

    мқҙ мұ…м—җм„ңлҠ” vLLMмқ„ лЁјм Җ baselineмңјлЎң л‘”лӢӨ. мқҙмң лҠ” production LLM servingмқҳ лі‘лӘ©мқ„ scheduler, KV cache, metricsмҷҖ н•Ёк»ҳ м„ӨлӘ…н•ҳкё° мўӢкё° л•Ңл¬ёмқҙлӢӨ.

    мқҙ мһҘмқҳ кІ°лЎ ¶

    мІ« LLM serverлҠ” лӘЁлҚёмқ„ "лқ„мҡ°лҠ”" лӢЁкі„к°Җ м•„лӢҲлқј serving systemмқ„ "мёЎм • к°ҖлҠҘн•ң мғҒнғңлЎң л§Ңл“ңлҠ”" лӢЁкі„лӢӨ.

    • OpenAI-compatible endpointлҠ” application integrationмқ„ мүҪкІҢ н•ҳм§Җл§Ң runtime м°ЁмқҙлҘј мҲЁкёҙлӢӨ.
    • Model artifactлҠ” weightлҝҗ м•„лӢҲлқј config, tokenizer, chat template, generation configлҘј нҸ¬н•Ён•ңлӢӨ.
    • мІ« baselineм—җм„ңлҠ” server command, model id, GPU, dtype, max context, prompt shape, latency, token countлҘј кё°лЎқн•ҙм•ј н•ңлӢӨ.
    • TTFTмҷҖ TPOTлҘј л¶„лҰ¬н•ҳл Өл©ҙ streaming clientмҷҖ /metricsлҘј н•Ёк»ҳ ліҙлҠ” кІғмқҙ мўӢлӢӨ.
    • Serving configлҠ” code snippetмқҙ м•„лӢҲлқј capacity contractлӢӨ.
    • мІ« workloadлҠ” short chat, long prompt, repeated prefixлЎң лӮҳлҲ„м–ҙм•ј н•ңлӢӨ.

    лӢӨмқҢ мһҘм—җм„ңлҠ” мқҙ мёЎм •к°’мқ„ лҚ” мІҙкі„м ҒмңјлЎң лӢӨлЈ¬лӢӨ. TTFT, TPOT, throughput, p95/p99 latencyлҘј л¶„лҰ¬н•ҳкі , benchmark кІ°кіјлҘј м–ҙл–»кІҢ мқҪм–ҙм•ј н•ҳлҠ”м§Җ м •лҰ¬н•ңлӢӨ.
