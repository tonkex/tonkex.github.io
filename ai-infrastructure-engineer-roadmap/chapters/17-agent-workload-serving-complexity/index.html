    17мһҘ. Agent workloadлҠ” мҷң servingмқ„ лҚ” м–ҙл өкІҢ л§Ңл“ңлҠ”к°Җ¶

    Draft v0.1 Updated: 2026-07-10 Part 7 - Application infrastructure

    16мһҘм—җм„ңлҠ” RAGлҘј vector DB н•ҳлӮҳк°Җ м•„лӢҲлқј ingestion, retrieval, rerank, prompt assembly, evaluationмңјлЎң мқҙм–ҙм§ҖлҠ” pipelineмңјлЎң лҙӨлӢӨ. Agent workloadлҠ” м—¬кё°м„ң н•ң лӢЁкі„ лҚ” ліөмһЎн•ҳлӢӨ.

    RAG requestлҠ” ліҙнҶө "кІҖмғүн•ҳкі  лӢөн•ңлӢӨ"лЎң лҒқлӮңлӢӨ. Agent requestлҠ” "мғқк°Ғн•ҳкі , лҸ„кө¬лҘј кі лҘҙкі , мӢӨн–үн•ҳкі , кІ°кіјлҘј ліҙкі , лӢӨмӢң кі„нҡҚн•ҳкі , н•„мҡ”н•ҳл©ҙ лӢӨлҘё agentлӮҳ мӮ¬лһҢм—җкІҢ л„ҳкё°кі , лӢӨмӢң лӘЁлҚёмқ„ нҳём¶ңн•ҳлҠ”" trajectoryк°Җ лҗңлӢӨ.

    мҡҙмҳҒм—җм„ң мӨ‘мҡ”н•ң м§Ҳл¬ёмқҖ мқҙкІғмқҙлӢӨ.

    мӮ¬мҡ©мһҗлҠ” мҡ”мІӯ н•ҳлӮҳлҘј ліҙлғҲлҠ”лҚ°, лӮҙл¶Җм—җм„ңлҠ” лӘҮ лІҲмқҳ model call, tool call, retry, handoff, memory read/write, approvalмқҙ л°ңмғқн–ҲлҠ”к°Җ?

    мқҙ м§Ҳл¬ём—җ лӢөн•ҳм§Җ лӘ»н•ҳл©ҙ agentлҠ” latency, cost, KV cache, ліҙм•Ҳ кІҪкі„лҘј мҳҲмёЎн•ҳкё° м–ҙл Өмҡҙ workloadк°Җ лҗңлӢӨ.

    мқҙ мһҘмқҳ лҸ…мһҗ м§Ҳл¬ёмқҖ лӢӨмқҢкіј к°ҷлӢӨ.

    Agent workloadлҘј productionм—җм„ң мҡҙмҳҒн•ҳл Өл©ҙ request лӢЁмң„к°Җ м•„лӢҲлқј trajectory лӢЁмң„лЎң л¬ҙм—Үмқ„ м ңн•ңн•ҳкі  кҙҖмёЎн•ҙм•ј н•ҳлҠ”к°Җ?

    н•өмӢ¬мқҖ agentлҘј "лҳ‘лҳ‘н•ң prompt"к°Җ м•„лӢҲлқј stateful distributed workflowлЎң ліҙлҠ” кІғмқҙлӢӨ.

    AgentлҠ” лӢЁмқј requestк°Җ м•„лӢҲлқј trajectoryлӢӨ¶

    OpenAI Agents SDK л¬ём„ңлҠ” agentлҘј plan, tool call, specialist collaboration, state мң м§ҖлЎң multi-step workлҘј мҷ„лЈҢн•ҳлҠ” applicationмқҙлқјкі  м„ӨлӘ…н•ңлӢӨ. LangGraph л¬ём„ңлҸ„ long-running, stateful agentлҘј кө¬м¶•н•ҳкі  мҡҙмҳҒн•ҳкё° мң„н•ң orchestration runtimeмқ„ к°•мЎ°н•ңлӢӨ.

    мқҙ л§җмқҖ agent мҡҙмҳҒ лӢЁмң„к°Җ API request н•ҳлӮҳк°Җ м•„лӢҲлқјлҠ” лң»мқҙлӢӨ.

    Tool call loopк°Җ latencyмҷҖ costлҘј кіұн•ңлӢӨ¶

    Prefix reuseлҠ” agent servingм—җм„ң лҚ” мӨ‘мҡ”н•ҙм§„лӢӨ¶

    Agent workloadлҠ” л°ҳліө prefixлҘј л§Һмқҙ л§Ңл“ лӢӨ.

    • к°ҷмқҖ system prompt
    • к°ҷмқҖ tool schema
    • к°ҷмқҖ policy text
    • к°ҷмқҖ RAG instruction
    • к°ҷмқҖ conversation prefix
    • к°ҷмқҖ few-shot examples

    мқҙ кө¬мЎ°лҠ” KV cacheмҷҖ prefix cachingм—җ м§Ғм ‘ м—°кІ°лҗңлӢӨ. 5мһҘм—җм„ң ліё кІғмІҳлҹј кіјкұ° tokenмқҳ K/VлҘј мһ¬мӮ¬мҡ©н•  мҲҳ мһҲмңјл©ҙ prefill л№„мҡ©мқ„ мӨ„мқј мҲҳ мһҲлӢӨ. SGLang л…јл¬ёмқҖ ліөмһЎн•ң language model programм—җм„ң л°ҳліөлҗҳлҠ” promptмҷҖ structured generationмқ„ нҡЁмңЁм ҒмңјлЎң мӢӨн–үн•ҳкё° мң„н•ҙ RadixAttention к°ҷмқҖ KV cache reuse мөңм Ғнҷ”лҘј м ңмӢңн•ңлӢӨ.

    Prefix ліҖкІҪ л№ҲлҸ„ мөңм Ғнҷ”

    Agent promptлҘј л§Ө step мғҲлЎң л§Ңл“ңлҠ” л°©мӢқмқҖ кө¬нҳ„мқҖ мүҪм§Җл§Ң cache reuseк°Җ м–ҙл өлӢӨ. л°ҳлҢҖлЎң stable prefixмҷҖ volatile observationмқ„ л¶„лҰ¬н•ҳл©ҙ TTFTмҷҖ prefill costлҘј мӨ„мқј мҲҳ мһҲлӢӨ.

    HandoffмҷҖ multi-agentлҠ” ownership л¬ём ңлӢӨ¶

    Agentк°Җ лӢӨлҘё agentм—җкІҢ л„ҳкё°кұ°лӮҳ specialistлҘј нҳём¶ңн•ҳл©ҙ workflowлҠ” лҚ” мң м—°н•ҙм§„лӢӨ. лҸҷмӢңм—җ ownershipмқҙ нқҗл Өм§„лӢӨ.

    нҢЁн„ҙ мһҘм җ мң„н—ҳ

    Specialist agent м—ӯн• /лҸ„кө¬ л¶„лҰ¬ handoff мҳӨлҘҳ
    Agent as tool мһ¬мӮ¬мҡ© мү¬мӣҖ nested latency/cost
    Planner-executor нҶөм ңмҷҖ audit мү¬мӣҖ planner н’Ҳм§Ҳ мқҳмЎҙ

    GuardrailмқҖ final answerл§Ң ліҙм§Җ м•ҠлҠ”лӢӨ¶

    Agents SDK guardrails л¬ём„ңлҠ” input guardrails, output guardrails, tool guardrailsлҘј кө¬л¶„н•ҳкі , tool guardrailsк°Җ custom function-tool invocationл§ҲлӢӨ мӢӨн–үлҗ  мҲҳ мһҲлӢӨкі  м„ӨлӘ…н•ңлӢӨ. Agentм—җм„ңлҠ” final answer guardrailл§ҢмңјлЎң л¶ҖмЎұн•ҳлӢӨ. мң„н—ҳмқҖ мӨ‘к°„ tool callм—җм„ң мқҙлҜё л°ңмғқн•  мҲҳ мһҲкё° л•Ңл¬ёмқҙлӢӨ.

    Guardrail мң„м№ҳлҘј л¶„лҰ¬н•ңлӢӨ.

    мң„м№ҳ л§үлҠ” л¬ём ң
    Input guardrail м•…м„ұ мҡ”мІӯ, scope л°– мҡ”мІӯ
    Plan guardrail мң„н—ҳн•ң plan, л¶Ҳн•„мҡ”н•ң tool chain
    Tool input guardrail secret м „лӢ¬, к¶Ңн•ң л°– action
    Tool output guardrail secret leak, prompt injection payload
    Final output guardrail policy мң„л°ҳ лӢөліҖ

    Tool outputлҸ„ untrusted inputмқҙлӢӨ. Web search, RAG л¬ём„ң, MCP tool, SaaS API responseлҠ” лӘЁл‘җ agentлҘј мҶҚмқј мҲҳ мһҲлӢӨ. Tool outputмқ„ к·ёлҢҖлЎң next promptм—җ л„Јмңјл©ҙ indirect prompt injectionм—җ м·Ём•Ҫн•ҙм§„лӢӨ.

    Failure pattern¶

    Request лӢЁмң„ latencyл§Ң ліёлӢӨ¶

    Agentм—җм„ңлҠ” request н•ҳлӮҳк°Җ м—¬лҹ¬ stepмңјлЎң лҠҳм–ҙлӮңлӢӨ. E2E latencyлҘј model call, tool call, approval wait, retryлЎң л¶„н•ҙн•ҙм•ј н•ңлӢӨ.

    Toolмқ„ л„Ҳл¬ҙ л§Һмқҙ л…ём¶ңн•ңлӢӨ¶

    Tool schemaлҠ” contextмҷҖ л№„мҡ©мқ„ м°Ём§Җн•ңлӢӨ. н•„мҡ” м—ҶлҠ” toolмқҖ лӘЁлҚё м„ нғқ мҳӨлҘҳмҷҖ кіөкІ©л©ҙмқ„ нӮӨмҡҙлӢӨ.

    Retryк°Җ side effectлҘј л°ҳліөн•ңлӢӨ¶

    Timeout нӣ„ к°ҷмқҖ mutating toolмқ„ лӢӨмӢң нҳём¶ңн•ҳл©ҙ мӨ‘ліө мӢӨн–үмқҙ мғқкёҙлӢӨ. Idempotency keyмҷҖ side effect IDк°Җ н•„мҡ”н•ҳлӢӨ.

    MemoryлҘј л¬ҙм ңн•ң contextлЎң м“ҙлӢӨ¶

    Conversation historyлҘј кі„мҶҚ л¶ҷмқҙл©ҙ context costк°Җ нҸӯл°ңн•ҳкі  мҳӨлһҳлҗң м •ліҙк°Җ нҳ„мһ¬ кІ°м •мқ„ мҳӨм—јмӢңнӮЁлӢӨ.

    Final answerл§Ң нҸүк°Җн•ңлӢӨ¶

    AgentлҠ” мӨ‘к°„ tool callкіј side effectк°Җ м ңн’Ҳ лҸҷмһ‘мқҙлӢӨ. Tool pathмҷҖ budget, approvalмқ„ н•Ёк»ҳ нҸүк°Җн•ҙм•ј н•ңлӢӨ.
