    8мһҘ. QuantizationмқҖ м–ём ң м„ұлҠҘ мөңм Ғнҷ”мқҙкі  м–ём ң мң„н—ҳмқёк°Җ¶

    Draft v0.1 Updated: 2026-07-10 Part 3 - лӢЁмқј GPUм—җм„ң servingмқ„ мёЎм •н•ҳкё°

    7мһҘм—җм„ң мҡ°лҰ¬лҠ” LLM serverмқҳ м„ұлҠҘмқ„ TTFT, TPOT, throughput, p95/p99, queue, KV cache usageлЎң лӮҳлҲ„м–ҙ мқҪм—ҲлӢӨ. мқҙм ң мһҗм—°мҠӨлҹҪкІҢ лӢӨмқҢ м§Ҳл¬ёмқҙ лӮҳмҳЁлӢӨ.

    4-bitлӮҳ FP8лЎң мӨ„мқҙл©ҙ лҚ” л№ лҘҙкі  мӢём§ҖлҠ”к°Җ?

    лӢөмқҖ "к·ёлҹҙ мҲҳ мһҲм§Җл§Ң, мһҗлҸҷмңјлЎң к·ёл Үм§ҖлҠ” м•ҠлӢӨ"мқҙлӢӨ. QuantizationмқҖ precisionмқ„ лӮ®м¶° memory footprintмҷҖ memory trafficмқ„ мӨ„мқҙлҠ” кё°мҲ мқҙлӢӨ. н•ҳм§Җл§Ң лӮ®мқҖ precisionмқҖ accuracy, kernel support, dequantization overhead, calibration, runtime compatibilityлҘј н•Ёк»ҳ лҒҢкі  мҳЁлӢӨ.

    мқҙ мһҘмқҳ лҸ…мһҗ м§Ҳл¬ёмқҖ лӢӨмқҢкіј к°ҷлӢӨ.

    QuantizationмқҖ м–ём ң м„ұлҠҘ мөңм Ғнҷ”мқҙкі , м–ём ң production riskк°Җ лҗҳлҠ”к°Җ?

    н•өмӢ¬мқҖ н•ҳлӮҳлӢӨ. QuantizationмқҖ "мһ‘кІҢ л§Ңл“ңлҠ” лІ„нҠј"мқҙ м•„лӢҲлқј serving systemмқҳ лі‘лӘ©мқ„ м–ҙлҠҗ м¶•м—җм„ң мӨ„мқјм§Җ м„ нғқн•ҳлҠ” мҡҙмҳҒ кІ°м •мқҙлӢӨ.

    лЁјм Җ л¬ҙм—Үмқ„ quantizeн•ҳлҠ”м§Җ л¶„лҰ¬н•ңлӢӨ¶

    Quantizationмқ„ л§җн•  л•Ң к°ҖмһҘ лЁјм Җ н•ҙм•ј н•  мқјмқҖ лҢҖмғҒ л¶„лҰ¬лӢӨ. Weight quantization, activation quantization, KV cache quantizationмқҖ м„ңлЎң лӢӨлҘё л¬ём ңлӢӨ.

    лҢҖмғҒ мӨ„мқҙлҠ” кІғ мЈјлЎң мҳҒн–Ҙ л°ӣлҠ” м§Җн‘ң лҢҖн‘ң мң„н—ҳ
    weights model weight memory model load memory, throughput, TPOT kernel лҜём§Җмӣҗ, accuracy м Җн•ҳ
    activations layer мӨ‘к°„ кі„мӮ° к°’ compute/memory traffic calibration лҜјк°җлҸ„, kernel path
    KV cache requestлі„ context state max concurrency, long context, TTFT/p95 long-context н’Ҳм§Ҳ, scale calibration

    мқҙ м„ё к°Җм§ҖлҘј м„һм–ҙ л§җн•ҳл©ҙ нҢҗлӢЁмқҙ нӢҖм–ҙм§„лӢӨ. мҳҲлҘј л“Өм–ҙ INT4 weight quantizationмқҖ model weight memoryлҘј нҒ¬кІҢ мӨ„мқј мҲҳ мһҲм§Җл§Ң, кёҙ contextм—җм„ң KV cacheк°Җ лі‘лӘ©мқҙлқјл©ҙ лҸҷмӢңм„ұ л¬ём ңлҘј м¶©л¶„нһҲ н•ҙкІ°н•ҳм§Җ лӘ»н•  мҲҳ мһҲлӢӨ. л°ҳлҢҖлЎң FP8 KV cacheлҠ” long-context capacityм—җлҠ” м§Ғм ‘ лҸ„мӣҖмқҙ лҗҳм§Җл§Ң, weight memoryк°Җ мқҙлҜё GPUлҘј кұ°мқҳ мұ„мҡҙ мғҒнғңлқјл©ҙ model мһҗмІҙлҘј мҳ¬лҰ¬лҠ” л¬ём ңлҠ” н•ҙкІ°н•ҳм§Җ лӘ»н•ңлӢӨ.

    "мһ‘м•„м§„лӢӨ"мҷҖ "л№Ёлқјм§„лӢӨ"лҠ” лӢӨлҘҙлӢӨ¶

    Quantizationмқҙ memoryлҘј мӨ„мқҙлҠ” мқҙмң лҠ” м§ҒкҙҖм ҒмқҙлӢӨ. к°ҷмқҖ мҲ«мһҗ к°ңмҲҳлҘј лҚ” м ҒмқҖ bitлЎң м ҖмһҘн•ҳкё° л•Ңл¬ёмқҙлӢӨ.

    лҢҖлһөм Ғмқё weight memoryлҠ” лӢӨмқҢмІҳлҹј ліј мҲҳ мһҲлӢӨ.

    \[ Memory_{weights} \approx Parameters \times Bytes\ per\ parameter \]

    л”°лқјм„ң 7B modelмқ„ лӢЁмҲңнҷ”н•ҙм„ң ліҙл©ҙ лӢӨмқҢкіј к°ҷмқҖ к°җк°Ғмқ„ м–»мқ„ мҲҳ мһҲлӢӨ.

    dtype parameterлӢ№ bytes 7B weight rough memory

    FP16/BF16 2 м•Ҫ 14 GB

    мқҙ н‘ңлҠ” optimizer state, metadata, quantization scale, padding, runtime overheadлҘј ләҖ rough estimateлӢӨ. мӢӨм ң serving memoryмҷҖлҠ” лӢӨлҘҙлӢӨ. н•ҳм§Җл§Ң мҷң quantizationмқҙ мһ‘мқҖ GPUм—җм„ң нҒ° modelмқ„ мҳ¬лҰ¬лҠ” лҚ° лҸ„мӣҖмқҙ лҗҳлҠ”м§ҖлҠ” ліҙм—¬мӨҖлӢӨ.

    л¬ём ңлҠ” мҶҚлҸ„лӢӨ. Memory footprintк°Җ мӨ„м—ҲлӢӨкі  latencyк°Җ мһҗлҸҷмңјлЎң мӨ„м§ҖлҠ” м•ҠлҠ”лӢӨ.

    мҶҚлҸ„к°Җ мўӢм•„м§Җл Өл©ҙ ліҙнҶө лӢӨмқҢ мЎ°кұҙмқҙ н•„мҡ”н•ҳлӢӨ.

    • quantized formatмқ„ м§Ғм ‘ мІҳлҰ¬н•ҳлҠ” kernelмқҙ мһҲлӢӨ.
    • dequantization overheadк°Җ memory traffic к°җмҶҢліҙлӢӨ мһ‘лӢӨ.
    • batch sizeмҷҖ sequence lengthк°Җ н•ҙлӢ№ kernelмқҙ мһҳ м“°мқҙлҠ” мҳҒм—ӯм—җ мһҲлӢӨ.
    • GPU architectureк°Җ к·ё precisionмқ„ л№ лҘҙкІҢ мІҳлҰ¬н•ңлӢӨ.
    • runtimeмқҙ н•ҙлӢ№ model architectureмҷҖ quantization formatмқ„ м•Ҳм •м ҒмңјлЎң м§Җмӣҗн•ңлӢӨ.

    мқҙ мЎ°кұҙмқҙ к№Ём§Җл©ҙ modelмқҖ мһ‘м•„мЎҢм§Җл§Ң лҠҗл Өм§Ҳ мҲҳ мһҲлӢӨ. лҳҗлҠ” нҸүк·  throughputмқҖ мўӢм•„мЎҢм§Җл§Ң p95 latencyк°Җ м•…нҷ”лҗ  мҲҳ мһҲлӢӨ.

    Weight-only quantization: modelмқ„ мҳ¬лҰ¬кё° мң„н•ң мІ« лҸ„кө¬¶

    Weight-only quantizationмқҖ model weightsлҘј лӮ®мқҖ precisionмңјлЎң м ҖмһҘн•ҳкі  мӢӨн–үн•ңлӢӨ. AWQ, GPTQ, BitsAndBytes 4-bit к°ҷмқҖ мқҙлҰ„мқ„ м—¬кё°м„ң мһҗмЈј л§ҢлӮңлӢӨ.

    vLLM quantization л¬ём„ңлҠ” quantizationмқҙ model precisionмқ„ лӮ®м¶° memory footprintлҘј мӨ„мқҙкі  лҚ” лӢӨм–‘н•ң мһҘм№ҳм—җм„ң нҒ° modelмқ„ мӢӨн–үн•ҳкІҢ н•ңлӢӨкі  м„ӨлӘ…н•ңлӢӨ. лҳҗн•ң vLLMмқҖ AutoAWQ, BitsAndBytes, GPTQModel, LLM Compressor, NVIDIA Model Optimizer, Quantized KV Cache л“ұ м—¬лҹ¬ quantization кІҪлЎңлҘј м§Җмӣҗн•ңлӢӨ.

    Weight-only quantizationмқҙ мң мҡ©н•ң кІҪмҡ°лҠ” лӘ…нҷ•н•ҳлӢӨ.

    • BF16/FP16 weightк°Җ GPUм—җ мҳ¬лқјк°Җм§Җ м•ҠлҠ”лӢӨ.
    • лҚ” мһ‘мқҖ GPUм—җм„ң model serving baselineмқ„ л§Ңл“Өкі  мӢ¶лӢӨ.
    • weight memoryлҘј мӨ„м—¬ KV cache м—¬мң лҘј мЎ°кёҲмқҙлқјлҸ„ нҷ•ліҙн•ҳкі  мӢ¶лӢӨ.
    • throughputліҙлӢӨ memory fitмқҙ мІ« лІҲм§ё л¬ём ңлӢӨ.

    н•ҳм§Җл§Ң мң„н—ҳлҸ„ мһҲлӢӨ.

    • нҠ№м • GPUм—җм„ңлҠ” н•ҙлӢ№ quantization kernelмқҙ л№ лҘҙм§Җ м•Ҡмқ„ мҲҳ мһҲлӢӨ.
    • мқјл¶Җ model architectureлӮҳ layerк°Җ м§Җмӣҗлҗҳм§Җ м•Ҡмқ„ мҲҳ мһҲлӢӨ.
    • calibration лҳҗлҠ” quantization л°©мӢқм—җ л”°лқј лӢөліҖ н’Ҳм§Ҳмқҙ л–Ём–ҙм§Ҳ мҲҳ мһҲлӢӨ.
    • к°ҷмқҖ "4-bit"лқјлҸ„ AWQ, GPTQ, NF4, GGUF quant typeмқҖ к°ҷмқҖ кІғмқҙ м•„лӢҲлӢӨ.

    Hugging Face Transformers л¬ём„ңлҸ„ quantization methodл§ҲлӢӨ мһҘлӢЁм җмқҙ мһҲкі , calibrationмқҙ н•„мҡ”н•ң л°©лІ•кіј on-the-fly л°©мӢқмқҙ лӢӨлҘҙлӢӨкі  м„ӨлӘ…н•ңлӢӨ. л”°лқјм„ң "4-bit model"мқҙлқјлҠ” мқҙлҰ„л§ҢмңјлЎң production м Ғн•©м„ұмқ„ нҢҗлӢЁн•ҳл©ҙ м•Ҳ лҗңлӢӨ.

    Activation quantization: kernelкіј hardware мқҳмЎҙм„ұмқҙ лҚ” нҒ¬лӢӨ¶

    Activation quantizationмқҖ layer мӨ‘к°„ кі„мӮ° к°’мқ„ лӮ®мқҖ precisionмңјлЎң мІҳлҰ¬н•ңлӢӨ. FP8 W8A8мІҳлҹј weightмҷҖ activationмқ„ н•Ёк»ҳ лӮ®м¶”лҠ” л°©мӢқмқҙ м—¬кё°м—җ к°Җк№Ңмҡҙ мҡҙмҳҒ мқҙмҠҲлҘј л§Ңл“ лӢӨ.

    Activation quantizationмқҖ memory trafficкіј compute pathлҘј мӨ„мқј мҲҳ мһҲм§Җл§Ң, hardwareмҷҖ kernel supportм—җ лҚ” лҜјк°җн•ҳлӢӨ. vLLMмқҳ supported hardware н‘ңл§Ң лҙҗлҸ„ quantization implementationл§ҲлӢӨ Volta, Turing, Ampere, Ada, Hopper, AMD GPU, Intel GPU, CPU м§Җмӣҗ м—¬л¶Җк°Җ лӢӨлҘҙлӢӨ. мҳҲлҘј л“Өм–ҙ FP8 кі„м—ҙмқҖ GPU м„ёлҢҖмҷҖ backendм—җ к°•н•ҳкІҢ л¬¶мқёлӢӨ.

    л”°лқјм„ң activation quantizationмқҖ лӢӨмқҢ м§Ҳл¬ёмқ„ нҶөкіјн•ҙм•ј н•ңлӢӨ.

    • лӮҙ GPU architectureк°Җ мқҙ precisionмқ„ л№ лҘҙкІҢ мІҳлҰ¬н•ҳлҠ”к°Җ?
    • runtimeмқҙ мқҙ model architectureм—җм„ң н•ҙлӢ№ kernelмқ„ м“°лҠ”к°Җ?
    • batch/concurrencyк°Җ kernel нҡЁмңЁмқҙ лӮҳмҳӨлҠ” мҳҒм—ӯмқёк°Җ?
    • accuracy regressionмқ„ ліј eval setмқҙ мһҲлҠ”к°Җ?
    • fallback pathлЎң мЎ°мҡ©нһҲ лҠҗлҰ° kernelмқ„ нғҖм§Җ м•ҠлҠ”к°Җ?

    Activation quantizationмқҖ мһҳ л§һмңјл©ҙ к°•л Ҙн•ҳм§Җл§Ң, "мҳөм…ҳмқ„ мј°лӢӨ"мҷҖ "л№ лҘё kernelмқ„ нғ”лӢӨ"лҠ” лӢӨлҘё л§җмқҙлӢӨ.

    KV cache quantization: long contextмҷҖ concurrencyлҘј мң„н•ң лҸ„кө¬¶

    5мһҘм—җм„ң ліё кІғмІҳлҹј KV cacheлҠ” active tokensм—җ л”°лқј м»Өм§„лӢӨ. 6мһҘм—җм„ң мІ« serverлҘј лқ„мӣ кі , 7мһҘм—җм„ң TTFTмҷҖ p95лҘј лҙӨлӢӨл©ҙ, мқҙм ң KV cache quantizationмқҙ мҷң мӨ‘мҡ”н•ңм§Җ лӘ…нҷ•н•ҙм§„лӢӨ.

    vLLMмқҳ Quantized KV Cache л¬ём„ңлҠ” FP8 KV cacheк°Җ KV cache memory footprintлҘј мӨ„м—¬ лҚ” л§ҺмқҖ tokenмқ„ memoryм—җ м ҖмһҘн•ҳкі , throughputкіј longer context window м§Җмӣҗм—җ лҸ„мӣҖмқҙ лҗ  мҲҳ мһҲлӢӨкі  м„ӨлӘ…н•ңлӢӨ. лҳҗн•ң vLLMмқҖ FP8 KV cacheм—җ лҢҖн•ҙ per-tensorмҷҖ per-attention-head quantization м „лһөмқ„ м„ӨлӘ…н•ҳкі , scale calibration л°©мӢқмқ„ м„ё к°Җм§ҖлЎң лӮҳлҲҲлӢӨ.

    KV cache quantizationмқҖ мқҙлҹ° мғҒнҷ©м—җм„ң нӣ„ліҙк°Җ лҗңлӢӨ.

    • кёҙ promptлӮҳ кёҙ context workloadм—җм„ң KV cache usageк°Җ лҶ’лӢӨ.
    • concurrencyлҘј мҳ¬лҰ¬л©ҙ p95 latencyк°Җ кёүкІ©нһҲ лӮҳл№ м§„лӢӨ.
    • model weightлҠ” мҳ¬лқјк°Җм§Җл§Ң active sequence мҲҳк°Җ л¶ҖмЎұн•ҳлӢӨ.
    • long RAG, agent, code assistantмІҳлҹј contextк°Җ кёёкі  л°ҳліөлҗңлӢӨ.

    н•ҳм§Җл§Ң мң„н—ҳлҸ„ л¶„лӘ…н•ҳлӢӨ.

    • KV cacheлҠ” decode мӨ‘ attention кІ°кіјм—җ кі„мҶҚ мҳҒн–Ҙмқ„ мӨҖлӢӨ.
    • scale calibrationмқҙ л¶Җм •нҷ•н•ҳл©ҙ long-context н’Ҳм§Ҳмқҙ нқ”л“ӨлҰҙ мҲҳ мһҲлӢӨ.
    • мқјл¶Җ backendм—җм„ңлҠ” quantized cacheлҘј м“°лҚ”лқјлҸ„ attention compute pathк°Җ м ңн•ңлҗ  мҲҳ мһҲлӢӨ.
    • FP8 cacheк°Җ memoryлҠ” мӨ„м—¬лҸ„ TPOT лҳҗлҠ” p95к°Җ кё°лҢҖл§ҢнҒј мўӢм•„м§Җм§Җ м•Ҡмқ„ мҲҳ мһҲлӢӨ.

    vLLM л¬ём„ңлҠ” FP8 KV cache scaleмқ„ кё°ліёк°’, random token calibration, dataset calibration л°©мӢқмңјлЎң кі„мӮ°н•  мҲҳ мһҲлӢӨкі  м„ӨлӘ…н•ҳкі , dataset кё°л°ҳ calibrationмқ„ н’Ҳм§Ҳ кҙҖм җм—җм„ң к¶ҢмһҘн•ңлӢӨ. мҡҙмҳҒ кҙҖм җм—җм„ңлҠ” мқҙ л§җмқҙ мӨ‘мҡ”н•ҳлӢӨ. KV cache quantizationмқҖ лӢЁмҲң runtime flagк°Җ м•„лӢҲлқј calibration artifactк№Ңм§Җ нҸ¬н•Ён•ң release decisionмқҙлӢӨ.

    Quantization formatмқҖ release artifactлӢӨ¶

    Quantized modelмқҖ мӣҗліё modelкіј к°ҷмқҖ artifactк°Җ м•„лӢҲлӢӨ. мөңмҶҢн•ң лӢӨмқҢ м •ліҙк°Җ release recordм—җ л“Өм–ҙк°Җм•ј н•ңлӢӨ.

    quantization method AWQ, GPTQ, BitsAndBytes, FP8, GGUF л“ұ

    мқҙ кё°лЎқмқҙ м—Ҷмңјл©ҙ лӮҳмӨ‘м—җ "4-bitлЎң л°”кҝЁлҚ”лӢҲ мўӢм•„мЎҢлӢӨ" лҳҗлҠ” "FP8лЎң л°”кҝЁлҚ”лӢҲ н’Ҳм§Ҳмқҙ л–Ём–ҙмЎҢлӢӨ"лҘј мһ¬нҳ„н•  мҲҳ м—ҶлӢӨ. QuantizationмқҖ model registryмқҳ л¶Җк°Җ м •ліҙк°Җ м•„лӢҲлқј release artifactмқҳ н•өмӢ¬ н•„л“ңлӢӨ.

    мёЎм • мҲңм„ң: baselineмқ„ лЁјм Җ л§Ңл“ лӢӨ¶

    Quantization мӢӨн—ҳмқҖ н•ӯмғҒ BF16/FP16 baselineм—җм„ң мӢңмһ‘н•ңлӢӨ. мқҙлҜё 6мһҘкіј 7мһҘм—җм„ң л§Ңл“  мёЎм • нқҗлҰ„мқ„ к·ёлҢҖлЎң м“ҙлӢӨ.

    1. Baseline serverлҘј лқ„мҡҙлӢӨ.

    2. model id, runtime version, GPU, dtype, max model length, commandлҘј кё°лЎқн•ңлӢӨ.

    3. WorkloadлҘј кі м •н•ңлӢӨ.

    4. short chat, long prompt, high concurrencyлҘј л¶„лҰ¬н•ңлӢӨ.

    5. Baseline metricsлҘј м ҖмһҘн•ңлӢӨ.

    6. TTFT p50/p95, TPOT p50/p95, e2e p95, output tok/s, queue time, KV cache usage, GPU memoryлҘј кё°лЎқн•ңлӢӨ.

    7. Quantized variantлҘј н•ҳлӮҳл§Ң л°”кҫјлӢӨ.

    8. weight quantization лҳҗлҠ” KV cache quantization мӨ‘ н•ҳлӮҳл§Ң лЁјм Җ л°”кҫјлӢӨ.

    9. к°ҷмқҖ workloadлҘј лӢӨмӢң лҸҢлҰ°лӢӨ.

    10. request payloadмҷҖ benchmark мЎ°кұҙмқ„ л°”кҫём§Җ м•ҠлҠ”лӢӨ.

    11. Quality evalмқ„ к°ҷмқҙ ліёлӢӨ.

    12. latencyк°Җ мўӢм•„м ёлҸ„ hallucination, instruction following, tool call format, retrieval groundingмқҙ к№Ём§Җл©ҙ мӢӨнҢЁлӢӨ.

    13. мҡҙмҳҒ кІ°м •мқ„ м ҒлҠ”лӢӨ.

    14. мұ„нғқ, ліҙлҘҳ, нҠ№м • workloadм—җл§Ң мӮ¬мҡ©, rollback мӨ‘ н•ҳлӮҳлЎң кІ°лЎ мқ„ лӮёлӢӨ.

    м–ҙл–Ө м§Җн‘ңк°Җ мўӢм•„м ём•ј м„ұкіөмқёк°Җ¶

    мӢӨнҢЁ нҢЁн„ҙ¶

    Quantization мӢӨн—ҳмқҖ лӢӨмқҢ нҢЁн„ҙмңјлЎң мӢӨнҢЁн•ңлӢӨ.

    ModelмқҖ мһ‘м•„мЎҢм§Җл§Ң TPOTк°Җ лӮҳл№ м§„лӢӨ¶

    Dequantization overheadк°Җ нҒ¬кұ°лӮҳ, fast kernelмқ„ нғҖм§Җ лӘ»н–Ҳмқ„ к°ҖлҠҘм„ұмқҙ мһҲлӢӨ. GPU architectureмҷҖ runtime logлҘј нҷ•мқён•ңлӢӨ.

    нҸүк·  throughputмқҖ мўӢм•„мЎҢм§Җл§Ң p95к°Җ лӮҳл№ м§„лӢӨ¶

    Batchingкіј scheduler pressureк°Җ л°”лҖҢм—Ҳмқ„ мҲҳ мһҲлӢӨ. 7мһҘм—җм„ң ліё кІғмІҳлҹј averageмҷҖ tailмқ„ л¶„лҰ¬н•ҙм•ј н•ңлӢӨ.

    Short chatмқҖ кҙңм°®м§Җл§Ң long promptк°Җ лӮҳл№ м§„лӢӨ¶

    KV cache, long-context attention, calibration л¬ём ңк°Җ м„һмқј мҲҳ мһҲлӢӨ. long prompt workloadлҘј лі„лҸ„лЎң ліёлӢӨ.

    мқјл°ҳ QAлҠ” кҙңм°®м§Җл§Ң tool call formatмқҙ к№Ём§„лӢӨ¶
