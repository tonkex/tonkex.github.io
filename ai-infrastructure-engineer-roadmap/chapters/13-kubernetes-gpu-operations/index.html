    13мһҘ. Kubernetes GPU мҡҙмҳҒ: device plugin, node pool, scheduling¶

    Draft v0.1 Updated: 2026-07-10 Part 5 - Multi-GPUмҷҖ cluster inference

    12мһҘм—җм„ң мҡ°лҰ¬лҠ” н•ң GPUлҘј л„ҳмқ„ л•Ң parallelismкіј NCCL нҶөмӢ мқҙ latency м•ҲмңјлЎң л“Өм–ҙмҳЁлӢӨлҠ” м җмқ„ лҙӨлӢӨ. мқҙм ң м§Ҳл¬ёмқҙ cluster мҡҙмҳҒмңјлЎң л°”лҖҗлӢӨ.

    Kubernetesм—җ GPU nodeлҘј л¶ҷмқҙл©ҙ LLM serving podлҠ” мһҗлҸҷмңјлЎң мҳ¬л°”лҘё GPUм—җ л°°м№ҳлҗҳлҠ”к°Җ?

    лӢөмқҖ м•„лӢҲлӢӨ. Kubernetesк°Җ GPUлҘј мқёмӢқн•ҳкІҢ л§Ңл“ңлҠ” кІғкіј, inference workloadлҘј мҳ¬л°”лҘё GPU nodeм—җ л°°м№ҳн•ҳкі , лӢӨлҘё workloadмҷҖ кІ©лҰ¬н•ҳкі , мһҘм• мҷҖ capacityлҘј мҡҙмҳҒн•ҳлҠ” кІғмқҖ лӢӨлҘё л¬ём ңлӢӨ.

    мқҙ мһҘмқҳ лҸ…мһҗ м§Ҳл¬ёмқҖ лӢӨмқҢкіј к°ҷлӢӨ.

    Kubernetesм—җм„ң GPUлҠ” м–ҙл–»кІҢ resourceлЎң л…ём¶ңлҗҳкі , AI мқён”„лқј м—”м§ҖлӢҲм–ҙлҠ” node pool, label, taint, toleration, affinityлҘј м–ҙл–»кІҢ м„Өкі„н•ҙм•ј н•ҳлҠ”к°Җ?

    н•өмӢ¬мқҖ н•ҳлӮҳлӢӨ. GPU schedulingмқҖ nvidia.com/gpu: 1 н•ң мӨ„мқҙ м•„лӢҲлқј, hardware inventoryмҷҖ placement policyлҘј н•Ёк»ҳ м„Өкі„н•ҳлҠ” мқјмқҙлӢӨ.

    KubernetesлҠ” GPUлҘј кё°ліё CPUмІҳлҹј м•Ңм§Җ лӘ»н•ңлӢӨ¶

    KubernetesлҠ” CPUмҷҖ memoryлҘј кё°ліё resourceлЎң лӢӨлЈ¬лӢӨ. GPU к°ҷмқҖ specialized hardwareлҠ” device pluginмқ„ нҶөн•ҙ nodeмқҳ allocatable resourceлЎң л…ём¶ңлҗңлӢӨ. Kubernetes кіөмӢқ л¬ём„ңлҠ” GPU кҙҖлҰ¬лҘј stable кё°лҠҘмңјлЎң м„ӨлӘ…н•ҳл©ҙм„ң, AMD/NVIDIA GPUлҘј device pluginмқ„ нҶөн•ҙ cluster nodeм—җм„ң мӮ¬мҡ©н•  мҲҳ мһҲлӢӨкі  м„ӨлӘ…н•ңлӢӨ.

    NVIDIA нҷҳкІҪм—җм„ң мқјл°ҳм Ғмқё нқҗлҰ„мқҖ лӢӨмқҢкіј к°ҷлӢӨ.

    кі„мёө м—ӯн•  нҷ•мқён•  кІғ
    GPU driver hostк°Җ GPUлҘј мӮ¬мҡ©н•  мҲҳ мһҲкІҢ н•ңлӢӨ nvidia-smiк°Җ nodeм—җм„ң лҸҷмһ‘н•ҳлҠ”к°Җ
    container runtime container м•Ҳм—җм„ң GPU deviceмҷҖ libraryлҘј мӮ¬мҡ©н•  мҲҳ мһҲкІҢ н•ңлӢӨ NVIDIA Container Toolkit, runtime config
    device plugin GPUлҘј Kubernetes extended resourceлЎң л…ём¶ңн•ңлӢӨ DaemonSet health, node allocatable
    kubelet node capacityмҷҖ allocatable resourceлҘј API serverм—җ ліҙкі н•ңлӢӨ kubectl describe node
    scheduler Podмқҳ GPU limitкіј scheduling policyлҘј ліҙкі  nodeлҘј м„ нғқн•ңлӢӨ pending reason, events
    workload Pod resources.limits.nvidia.com/gpuлҘј мҡ”мІӯн•ңлӢӨ limits, tolerations, affinity

    мқҙ нқҗлҰ„ мӨ‘ н•ҳлӮҳлқјлҸ„ л№ м§Җл©ҙ л¬ём ңлҠ” лӢӨлҘҙкІҢ ліҙмқёлӢӨ. Driverк°Җ м—Ҷмңјл©ҙ node мһҗмІҙк°Җ GPUлҘј лӘ» м“ҙлӢӨ. Runtime м„Өм •мқҙ мһҳлӘ»лҗҳл©ҙ PodлҠ” лңЁм§Җл§Ң container м•Ҳм—җм„ң GPUлҘј лӘ» ліёлӢӨ. Device pluginмқҙ м—Ҷмңјл©ҙ Kubernetes schedulerк°Җ GPU capacityлҘј лӘЁлҘёлӢӨ. Scheduling policyк°Җ м—Ҷмңјл©ҙ GPUлҠ” ліҙмқҙм§Җл§Ң мӣҗн•ҳм§Җ м•ҠлҠ” nodeм—җ л°°м№ҳлҗ  мҲҳ мһҲлӢӨ.

    Device pluginмқҖ GPUлҘј ліҙмқҙкІҢ н•  лҝҗмқҙлӢӨ¶

    NVIDIA k8s-device-pluginмқҖ DaemonSetмңјлЎң л°°нҸ¬лҗҳм–ҙ к°Ғ GPU nodeм—җм„ң GPU к°ңмҲҳмҷҖ healthлҘј Kubernetesм—җ л…ём¶ңн•ңлӢӨ. кіөмӢқ READMEлҠ” мқҙ pluginмқҙ cluster nodeмқҳ GPU мҲҳлҘј л…ём¶ңн•ҳкі , GPU healthлҘј м¶”м Ғн•ҳл©°, GPU enabled containerлҘј мӢӨн–үн•  мҲҳ мһҲкІҢ н•ңлӢӨкі  м„ӨлӘ…н•ңлӢӨ.

    кё°ліё Pod specмқҖ лӢӨмқҢ нҳ•нғңлӢӨ.

    apiVersion: v1
kind: Pod
metadata:
  name: llm-serving
spec:
  containers:
    - name: server
      image: registry.example.com/llm-server:v1
      resources:
        limits:
          nvidia.com/gpu: 1

    Kubernetes GPU resourceм—җлҠ” мӨ‘мҡ”н•ң м ңм•Ҫмқҙ мһҲлӢӨ. кіөмӢқ л¬ём„ң кё°мӨҖмңјлЎң GPUлҠ” limitsм—җ м§Җм •н•ҙм•ј н•ҳл©°, requestsмҷҖ limitsлҘј л‘ҳ лӢӨ м§Җм •н•ңлӢӨл©ҙ к°’мқҙ к°ҷм•„м•ј н•ңлӢӨ. requestsл§Ң м§Җм •н•ҳлҠ” л°©мӢқмқҖ мӮ¬мҡ©н•  мҲҳ м—ҶлӢӨ.

    мқҙ м ңм•ҪмқҖ мҡҙмҳҒм ҒмңјлЎң мӨ‘мҡ”н•ҳлӢӨ. CPUмІҳлҹј "requestлҠ” лӮ®кІҢ, limitмқҖ лҶ’кІҢ" мһЎлҠ” мӮ¬кі л°©мӢқмқ„ GPUм—җ к·ёлҢҖлЎң м Ғмҡ©н•  мҲҳ м—ҶлӢӨ. GPU 1к°ңлҘј мҡ”мІӯн•ҳл©ҙ schedulerлҠ” к·ё Podк°Җ GPU 1к°ңлҘј м җмң н•ңлӢӨкі  ліёлӢӨ.

    Node poolмқ„ лЁјм Җ лӮҳлҲҲлӢӨ¶

    GPU cluster мҡҙмҳҒм—җм„ң мІ« лІҲм§ё м„Өкі„ лӢЁмң„лҠ” node poolмқҙлӢӨ. лӘЁл“  GPU nodeлҘј н•ҳлӮҳмқҳ poolмІҳлҹј ліҙл©ҙ placementк°Җ л¶ҲлӘ…нҷ•н•ҙм§„лӢӨ. LLM servingмқҖ GPU model, VRAM, NVLink topology, local SSD, network fabric, node image, driver versionм—җ лҜјк°җн•ҳлӢӨ.

    мҳҲлҘј л“Өм–ҙ лӢӨмқҢ node poolмқҖ м„ңлЎң лӢӨлҘё мҡҙмҳҒ лӢЁмң„лӢӨ.

    Node pool мҳҲ мЈјлҗң мҡ©лҸ„
    gpu-a100-80g A100 80GB, NVLink large model serving, tensor parallel
    gpu-h100 H100, high bandwidth fabric latency-sensitive production serving
    gpu-l40s L40S, PCIe cost-sensitive serving, embedding, batch
    gpu-mig MIG partitioned GPU small model, notebook, isolated low-SLO workloads
    cpu-system CPU-only gateway, controller, observability

    Node poolмқ„ лӮҳлҲ„лҠ” мқҙмң лҠ” schedulerк°Җ "GPU 1к°ң"л§Ң ліҙкі лҠ” GPUмқҳ н’Ҳм§Ҳкіј topologyлҘј мқҙн•ҙн•ҳм§Җ лӘ»н•ҳкё° л•Ңл¬ёмқҙлӢӨ. nvidia.com/gpu: 1мқҖ A100 80GB 1к°ңмҷҖ L40S 1к°ңлҘј кө¬л¶„н•ҳм§Җ м•ҠлҠ”лӢӨ. кө¬л¶„мқҖ labels, node affinity, taints/tolerations, admission policy, platform conventionмңјлЎң н•ҙм•ј н•ңлӢӨ.

    LabelмқҖ м„ нғқ кё°мӨҖмқҙкі  taintлҠ” ліҙнҳё мһҘм№ҳлӢӨ¶

    GPU nodeм—җлҠ” workloadк°Җ м„ нғқн•  мҲҳ мһҲлҠ” labelмқҙ н•„мҡ”н•ҳлӢӨ.

    kubectl label node gpu-node-1 accelerator=nvidia-a100-80g
kubectl label node gpu-node-1 serving.ai/node-pool=gpu-a100-80g
kubectl label node gpu-node-1 topology.ai/nvlink=true

    к°ҖмһҘ лӢЁмҲңн•ң Pod л°°м№ҳлҠ” nodeSelectorлЎң мӢңмһ‘н•  мҲҳ мһҲлӢӨ.

    spec:
  nodeSelector:
    serving.ai/node-pool: gpu-a100-80g

    лҚ” ліөмһЎн•ң мЎ°кұҙмқҖ node affinityлЎң н‘ңнҳ„н•ңлӢӨ. Kubernetes л¬ём„ңлҠ” nodeSelectorлҘј к°ҖмһҘ лӢЁмҲңн•ң node selection л°©мӢқмңјлЎң м„ӨлӘ…н•ҳкі , node affinityлҠ” лҚ” н‘ңнҳ„л Ҙмқҙ лҶ’кі  hard ruleкіј soft preferenceлҘј кө¬л¶„н•  мҲҳ мһҲлӢӨкі  м„ӨлӘ…н•ңлӢӨ.

    spec:
  affinity:
    nodeAffinity:
      requiredDuringSchedulingIgnoredDuringExecution:
        nodeSelectorTerms:
          - matchExpressions:
              - key: serving.ai/node-pool
                operator: In
                values:
                  - gpu-a100-80g
              - key: topology.ai/nvlink
                operator: In
                values:
                  - "true"

    Labelмқҙ "мқҙ nodeлҘј м„ нғқн•ҳлқј"лҠ” мӢ нҳёлқјл©ҙ, taintлҠ” "м•„л¬ҙ PodлӮҳ л“Өм–ҙмҳӨм§Җ лӘ»н•ҳкІҢ н•ҳлқј"лҠ” ліҙнҳё мһҘм№ҳлӢӨ.

    kubectl taint nodes gpu-node-1 nvidia.com/gpu=true:NoSchedule

    GPU workload Podм—җлҠ” matching tolerationмқ„ л„ЈлҠ”лӢӨ.

    spec:
  tolerations:
    - key: nvidia.com/gpu
      operator: Exists
      effect: NoSchedule

    Kubernetes л¬ём„ңлҠ” taintк°Җ nodeк°Җ PodлҘј л°Җм–ҙлӮҙлҠ” л°©мӢқмқҙкі , tolerationмқҖ Podк°Җ н•ҙлӢ№ taintлҘј кІ¬л”ң мҲҳ мһҲкІҢ н•ңлӢӨкі  м„ӨлӘ…н•ңлӢӨ. лӢЁ, tolerationмқҖ schedulingмқ„ н—Ҳмҡ©н•  лҝҗ ліҙмһҘн•ҳм§Җ м•ҠлҠ”лӢӨ. SchedulerлҠ” м—¬м „нһҲ resource, affinity, spread, priority к°ҷмқҖ лӢӨлҘё мЎ°кұҙмқ„ н•Ёк»ҳ нҸүк°Җн•ңлӢӨ.

    GPU Pod specмқҖ resourceмҷҖ placementлҘј н•Ёк»ҳ мқҪм–ҙм•ј н•ңлӢӨ¶

    LLM serving Podм—җм„ң лӢӨмқҢ л„Ө к°Җм§ҖлҠ” к°ҷмқҙ лҙҗм•ј н•ңлӢӨ.

    apiVersion: apps/v1
kind: Deployment
metadata:
  name: llm-server
spec:
  replicas: 2
  selector:
    matchLabels:
      app: llm-server
  template:
    metadata:
      labels:
        app: llm-server
    spec:
      tolerations:
        - key: nvidia.com/gpu
          operator: Exists
          effect: NoSchedule
      affinity:
        nodeAffinity:
          requiredDuringSchedulingIgnoredDuringExecution:
            nodeSelectorTerms:
              - matchExpressions:
                  - key: serving.ai/node-pool
                    operator: In
                    values:
                      - gpu-a100-80g
      containers:
        - name: server
          image: registry.example.com/llm-server:v1
          resources:
            limits:
              nvidia.com/gpu: 1
    н•ӯлӘ© мқҳлҜё
    resources.limits.nvidia.com/gpu GPU лӘҮ к°ңлҘј м җмң н• м§Җ
    nodeAffinity лҳҗлҠ” nodeSelector м–ҙл–Ө GPU node poolм—җ к°Ҳм§Җ
    tolerations GPU м „мҡ© node taintлҘј нҶөкіјн•  мҲҳ мһҲлҠ”м§Җ
    replicas data parallel serving replica мҲҳ

    мқҙ л„Ө к°Җм§Җ мӨ‘ н•ҳлӮҳлқјлҸ„ л№ м§Җл©ҙ мҡҙмҳҒ мқҳлҸ„к°Җ нқҗл Өм§„лӢӨ. GPU limitл§Ң мһҲмңјл©ҙ м•„л¬ҙ GPU nodeлӮҳ к°Ҳ мҲҳ мһҲлӢӨ. Affinityл§Ң мһҲкі  GPU limitмқҙ м—Ҷмңјл©ҙ GPU nodeм—җ мҳ¬лқјк°ҖлҸ„ GPUлҘј м җмң н•ҳм§Җ м•Ҡмқ„ мҲҳ мһҲлӢӨ. Tolerationмқҙ м—Ҷмңјл©ҙ GPU nodeк°Җ taintлҗҳм–ҙ мһҲмқ„ л•Ң Pending мғҒнғңк°Җ лҗңлӢӨ.

    Pending PodлҠ” schedulerмқҳ м„ӨлӘ…мқ„ мқҪлҠ”лӢӨ¶

    GPU workloadк°Җ Pendingмқҙл©ҙ лЁјм Җ eventлҘј ліёлӢӨ.

    kubectl describe pod llm-server-xxxxx

    мһҗмЈј ліҙлҠ” мӣҗмқёмқҖ лӢӨмқҢкіј к°ҷлӢӨ.

    Event / мҰқмғҒ мқҳлҜё нҷ•мқё
    Insufficient nvidia.com/gpu allocatable GPUк°Җ л¶ҖмЎұн•ҳлӢӨ kubectl describe node
    node affinity mismatch label мЎ°кұҙмқ„ л§ҢмЎұн•ҳлҠ” nodeк°Җ м—ҶлӢӨ node labels
    untolerated taint Podк°Җ GPU node taintлҘј tolerateн•ҳм§Җ лӘ»н•ңлӢӨ taints/tolerations
    image pull delay GPU schedulingмқҖ лҗҗм§Җл§Ң imageлҘј лӘ» л°ӣлҠ”лӢӨ registry, image size
    runtime error after scheduled PodлҠ” л–ҙм§Җл§Ң containerк°Җ GPUлҘј лӘ» ліёлӢӨ container runtime, driver mount

    AI мқён”„лқј м—”м§ҖлӢҲм–ҙлҠ” Pendingмқ„ "Kubernetes л¬ём ң"лқјкі  лӯүлҡұк·ёлҰ¬л©ҙ м•Ҳ лҗңлӢӨ. Schedulerк°Җ лӘ» мҳ¬лҰ° кІғмқём§Җ, kubeletмқҙ containerлҘј лӘ» мӢӨн–үн•ң кІғмқём§Җ, runtime м•Ҳм—җм„ң CUDAк°Җ мӢӨнҢЁн•ң кІғмқём§Җ л¶„лҰ¬н•ҙм•ј н•ңлӢӨ.

    Topology-aware placementк°Җ н•„мҡ”н•ҙм§ҖлҠ” мҲңк°„¶

    12мһҘм—җм„ң ліё tensor parallelismмқҖ GPU мӮ¬мқҙ topologyм—җ лҜјк°җн•ҳлӢӨ. Kubernetes schedulerмқҳ кё°ліё resource modelмқҖ nvidia.com/gpu: 4к°Җ к°ҷмқҖ nodeмқҳ м–ҙл–Ө л„Ө GPUмқём§Җ, к·ё GPUл“Өмқҙ NVLinkлЎң л¬¶мҳҖлҠ”м§Җк№Ңм§Җ м„ёл°Җн•ҳкІҢ м•Ңм§Җ лӘ»н•ңлӢӨ.

    л”°лқјм„ң production LLM servingм—җм„ңлҠ” лӢӨмқҢ мӨ‘ н•ҳлӮҳк°Җ н•„мҡ”н•ҙм§Ҳ мҲҳ мһҲлӢӨ.

    • н•ң Podк°Җ н•ң nodeмқҳ лӘЁл“  GPUлҘј м җмң н•ҳлҸ„лЎқ м„Өкі„н•ңлӢӨ.
    • Node pool мһҗмІҙлҘј лҸҷмқј topology мһҘл№„лЎңл§Ң кө¬м„ұн•ңлӢӨ.
    • GPU feature labelsлЎң NVLink, memory, MIG profile, GPU productлҘј л…ём¶ңн•ңлӢӨ.
    • Runtimeмқҙ visible device orderingмқ„ м–ҙл–»кІҢ л°ӣлҠ”м§Җ кІҖмҰқн•ңлӢӨ.
    • Multi-node TP/PPлҠ” Kubernetes кё°ліё schedulerл§Ң лҜҝм§Җ л§җкі  Ray, MPI operator, custom scheduler, gang scheduling, placement group к°ҷмқҖ кі„мёөмқ„ кІҖнҶ н•ңлӢӨ.

    мқҙ мһҘм—җм„ңлҠ” custom schedulerк№Ңм§Җ к№ҠкІҢ л“Өм–ҙк°Җм§Җ м•ҠлҠ”лӢӨ. к·ёлҹ¬лӮҳ кё°м–өн•ҙм•ј н•  мӣҗм№ҷмқҖ л¶„лӘ…н•ҳлӢӨ. Kubernetesмқҳ кё°ліё GPU resourceлҠ” "к°ңмҲҳ"лҘј мһҳ лӢӨлЈЁм§Җл§Ң, LLM servingм—җм„ң мӨ‘мҡ”н•ң "GPU мӮ¬мқҙ кҙҖкі„"лҠ” лі„лҸ„ м •мұ…мңјлЎң ліҙк°•н•ҙм•ј н•ңлӢӨ.

    MIGмҷҖ time-slicingмқҖ capacityк°Җ м•„лӢҲлқј isolation м •мұ…мқҙлӢӨ¶

    NVIDIA GPU мҡҙмҳҒм—җм„ңлҠ” MIGлӮҳ time-slicingмқ„ л§ҢлӮҳкІҢ лҗңлӢӨ. л‘ҳ лӢӨ GPU кіөмң мҷҖ кҙҖл Ё мһҲм§Җл§Ң мқҳлҜёк°Җ лӢӨлҘҙлӢӨ.

    MIGлҠ” м§Җмӣҗ GPUлҘј н•ҳл“ңмӣЁм–ҙ partitionмңјлЎң лӮҳлҲ„м–ҙ лҚ” мһ‘мқҖ GPU instanceмІҳлҹј мӮ¬мҡ©н•ҳлҠ” л°©мӢқмқҙлӢӨ. мһ‘мқҖ model serving, notebook, batch inferenceмІҳлҹј isolationмқҙ мӨ‘мҡ”н•ң workloadм—җ мң мҡ©н•  мҲҳ мһҲлӢӨ. н•ҳм§Җл§Ң н•ң large model replicaк°Җ GPU м „мІҙ VRAMкіј bandwidthлҘј н•„мҡ”лЎң н•ңлӢӨл©ҙ MIGлҠ” мҳӨнһҲл Ө м ңм•Ҫмқҙ лҗңлӢӨ.

    Time-slicingмқҖ м—¬лҹ¬ workloadк°Җ GPUлҘј мӢңк°„м ҒмңјлЎң лӮҳлҲ  м“°кІҢ н•ҳлҠ” л°©мӢқмқҙлӢӨ. GPU utilizationмқҙ лӮ®мқҖ мһ‘мқҖ workloadм—җлҠ” мң мҡ©н•  мҲҳ мһҲм§Җл§Ң, latency-sensitive LLM servingм—җм„ңлҠ” tail latencyк°Җ нқ”л“ӨлҰҙ мҲҳ мһҲлӢӨ.

    л”°лқјм„ң м§Ҳл¬ёмқҖ "GPUлҘј лҚ” мһҳкІҢ лӮҳлҲҢ мҲҳ мһҲлӮҳ"к°Җ м•„лӢҲлӢӨ.

    л°©мӢқ лЁјм Җ л¬јм–ҙліј м§Ҳл¬ё
    full GPU мқҙ workloadк°Җ latency SLOмҷҖ KV cacheлҘј мң„н•ҙ GPU м „мІҙк°Җ н•„мҡ”н•ңк°Җ?
    MIG мһ‘мқҖ workloadлҘј к°•н•ҳкІҢ кІ©лҰ¬н•ҳлҠ” кІғмқҙ лӘ©н‘ңмқёк°Җ?
    time-slicing latencyліҙлӢӨ нҸүк·  utilizationмқҙ лҚ” мӨ‘мҡ”н•ңк°Җ?

    Production serving node poolкіј notebook/shared experimentation node poolмқ„ м„һм§Җ м•ҠлҠ” мқҙмң лҸ„ м—¬кё°м—җ мһҲлӢӨ.

    AutoscalingмқҖ GPU к°ңмҲҳліҙлӢӨ warm capacityк°Җ м–ҙл өлӢӨ¶

    CPU serviceлҠ” replicaлҘј лҠҳлҰ¬л©ҙ л№„көҗм Ғ л№ лҘҙкІҢ capacityк°Җ лҠҳ мҲҳ мһҲлӢӨ. LLM servingмқҖ лӢӨлҘҙлӢӨ. мғҲ Podк°Җ GPU nodeм—җ л°°м№ҳлҗҳм–ҙлҸ„ model download, weight load, engine build, CUDA graph capture, tokenizer мӨҖл№„, health checkк№Ңм§Җ мӢңк°„мқҙ кұёлҰ°лӢӨ.

    л”°лқјм„ң GPU autoscalingмқҖ лӢӨмқҢ м§Ҳл¬ёмқ„ нҸ¬н•Ён•ҙм•ј н•ңлӢӨ.

    • мғҲ GPU nodeк°Җ мӨҖл№„лҗҳлҠ” лҚ° лӘҮ л¶„ кұёлҰ¬лҠ”к°Җ?
    • Model artifactлҠ” local cacheлӮҳ shared cacheм—җ мһҲлҠ”к°Җ?
    • Podк°Җ Readyк°Җ лҗҳкё° м „м—җ trafficмқҙ л“Өм–ҙк°Җм§Җ м•ҠлҸ„лЎқ readinessк°Җ м •нҷ•н•ңк°Җ?
    • Scale down л•Ң active requestмҷҖ KV cacheлҘј м–ҙл–»кІҢ drainн•ҳлҠ”к°Җ?
    • HPA кё°мӨҖмқҙ QPSмқёк°Җ, queue timeмқёк°Җ, active sequenceмқёк°Җ, GPU KV cache usageмқёк°Җ?

    GPU node autoscalingмқҖ л№„мҡ©мқ„ мӨ„мқј мҲҳ мһҲм§Җл§Ң, cold startк°Җ кёёл©ҙ incident мӨ‘м—җлҠ” лҸ„мӣҖмқҙ лҠҰкІҢ мҳЁлӢӨ. Production servingм—җлҠ” мөңмҶҢ warm replicaмҷҖ warm node capacityк°Җ н•„мҡ”н•  мҲҳ мһҲлӢӨ.

    мӢӨм ң clusterм—җм„ң лӢӨмқҢ лӘ…л №мңјлЎң мӢңмһ‘н•ңлӢӨ.

    мӢӨнҢЁ нҢЁн„ҙ¶

    GPU limitл§Ң л„Јкі  node poolмқ„ м§Җм •н•ҳм§Җ м•ҠлҠ”лӢӨ¶

    GPU мў…лҘҳк°Җ н•ҳлӮҳлҝҗмқё мһ‘мқҖ clusterм—җм„ңлҠ” мҡ°м—°нһҲ лҸҷмһ‘н•  мҲҳ мһҲлӢӨ. н•ҳм§Җл§Ң A100, L40S, MIG, spot GPUк°Җ м„һмқҙл©ҙ workloadк°Җ м—үлҡұн•ң nodeм—җ к°Ҳ мҲҳ мһҲлӢӨ.

    GPU nodeм—җ taintлҘј кұём§Җ м•ҠлҠ”лӢӨ¶

    CPU-only PodлӮҳ system workloadк°Җ GPU nodeм—җ мҳ¬лқјмҷҖ CPU, memory, disk, networkлҘј к°үм•„лЁ№мқ„ мҲҳ мһҲлӢӨ. GPU nodeлҠ” л№„мӢј nodeмқҙлҜҖлЎң workload admissionмқ„ лӘ…нҷ•нһҲ н•ҙм•ј н•ңлӢӨ.

    Tolerationмқ„ л§ҢлҠҘ scheduling ruleлЎң м°©к°Ғн•ңлӢӨ¶

    TolerationмқҖ taintлҘј нҶөкіјн•ҳкІҢ н•  лҝҗмқҙлӢӨ. мӣҗн•ҳлҠ” GPU typeмқ„ м„ нғқн•ҳл Өл©ҙ node selectorлӮҳ node affinityк°Җ н•„мҡ”н•ҳлӢӨ.

    Podк°Җ Runningмқҙл©ҙ GPU мӮ¬мҡ© к°ҖлҠҘн•ҳлӢӨкі  ліёлӢӨ¶

    Pod phaseк°Җ Runningмқҙм–ҙлҸ„ container м•Ҳм—җм„ң CUDA library, device mount, driver compatibility л¬ём ңк°Җ лӮ  мҲҳ мһҲлӢӨ. GPU test commandмҷҖ runtime metricмқ„ нҷ•мқён•ҙм•ј н•ңлӢӨ.

    Autoscalingл§Ң лҜҝкі  warm capacityлҘј л‘җм§Җ м•ҠлҠ”лӢӨ¶

    LLM server cold startлҠ” кёёлӢӨ. Model artifact downloadмҷҖ weight loadк°Җ лӘҮ л¶„ кұёлҰ¬л©ҙ HPAк°Җ л°ҳмқ‘н•ҙлҸ„ SLOлҠ” мқҙлҜё к№Ём§Ҳ мҲҳ мһҲлӢӨ.

    м •лҰ¬¶

    • KubernetesлҠ” GPUлҘј device plugin кё°л°ҳ extended resourceлЎң л…ём¶ңн•ңлӢӨ.

    • GPU resourceлҠ” limitsм—җ м§Җм •н•ҙм•ј н•ҳл©°, requestsмҷҖ н•Ёк»ҳ м“ё кІҪмҡ° л‘җ к°’мқҖ к°ҷм•„м•ј н•ңлӢӨ.
    • Node pool, labels, node affinity, taints/tolerationsлҠ” GPU workload placementмқҳ н•өмӢ¬ мҡҙмҳҒ лҸ„кө¬лӢӨ.
    • TolerationмқҖ scheduling н—Ҳмҡ© мЎ°кұҙмқј лҝҗ placement ліҙмһҘмқҖ м•„лӢҲлӢӨ. GPU type м„ нғқмқҖ labelкіј affinityк°Җ лӢҙлӢ№н•ңлӢӨ.
    • LLM servingм—җм„ңлҠ” GPU к°ңмҲҳлҝҗ м•„лӢҲлқј GPU type, VRAM, NVLink topology, local cache, network, warm capacityк№Ңм§Җ node pool м„Өкі„м—җ нҸ¬н•Ён•ҙм•ј н•ңлӢӨ.
    • Pending PodлҠ” scheduler eventлҘј мқҪкі  resource л¶ҖмЎұ, label mismatch, taint mismatch, runtime failureлҘј л¶„лҰ¬н•ҙ м§„лӢЁн•ҙм•ј н•ңлӢӨ.
