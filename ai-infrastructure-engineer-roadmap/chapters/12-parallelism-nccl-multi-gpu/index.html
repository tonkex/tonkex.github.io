    12мһҘ. Parallelismкіј NCCL: н•ң GPUлҘј л„ҳмқ„ л•Ң мғқкё°лҠ” л¬ём ң¶

    Draft v0.1 Updated: 2026-07-10 Part 5 - Multi-GPUмҷҖ cluster inference

    11мһҘк№Ңм§ҖлҠ” runtimeкіј platformмқ„ кі лҘҙлҠ” л¬ём ңлҘј лӢӨлӨҳлӢӨ. мқҙм ң лҚ” к·јліём Ғмқё м ңм•Ҫмқҙ лӮҳмҳЁлӢӨ.

    лӘЁлҚёмқҙ н•ң GPUм—җ л“Өм–ҙк°Җм§Җ м•Ҡкұ°лӮҳ, н•ң GPUлЎңлҠ” н•„мҡ”н•ң throughputмқ„ л§Ңл“Ө мҲҳ м—Ҷмңјл©ҙ л¬ҙм—Үмқҙ л°”лҖҢлҠ”к°Җ?

    GPUлҘј м—¬лҹ¬ к°ң л¶ҷмқҙл©ҙ capacityк°Җ мһҗлҸҷмңјлЎң лҠҳм–ҙлӮ  кІғмІҳлҹј ліҙмқёлӢӨ. мӢӨм ңлЎңлҠ” model weight, KV cache, activation, scheduler, network topology, collective communicationмқҙ лӘЁл‘җ к°ҷмқҖ мҡ”мІӯмқҳ latency м•ҲмңјлЎң л“Өм–ҙмҳЁлӢӨ. лӢЁмқј GPUм—җм„ңлҠ” "GPUк°Җ л№ лҘёк°Җ"лҘј мЈјлЎң л¬јм—ҲлӢӨл©ҙ, multi-GPUм—җм„ңлҠ” "GPUл“Өмқҙ м„ңлЎң м–јл§ҲлӮҳ мһҗмЈј, м–јл§ҲлӮҳ л©ҖлҰ¬, м–јл§ҲлӮҳ нҒ° лҚ°мқҙн„°лҘј мЈјкі л°ӣлҠ”к°Җ"лҘј л¬јм–ҙм•ј н•ңлӢӨ.

    мқҙ мһҘмқҳ лҸ…мһҗ м§Ҳл¬ёмқҖ лӢӨмқҢкіј к°ҷлӢӨ.

    Tensor parallelism, pipeline parallelism, data parallelismмқҖ к°Ғк°Ғ м–ҙл–Ө л¬ём ңлҘј н’Җкі , NCCL нҶөмӢ мқҖ decode latencyм—җ м–ҙл””м„ң л“Өм–ҙмҳӨлҠ”к°Җ?

    н•өмӢ¬мқҖ н•ҳлӮҳлӢӨ. Multi-GPU servingмқҖ GPU мҲҳлҘј лҠҳлҰ¬лҠ” мһ‘м—…мқҙ м•„лӢҲлқј, memory pressureмҷҖ communication costлҘј көҗнҷҳн•ҳлҠ” мһ‘м—…мқҙлӢӨ.

    лЁјм Җ мҷң н•ң GPUлҘј л„ҳлҠ”м§Җ кө¬л¶„н•ңлӢӨ¶

    Multi-GPUк°Җ н•„мҡ”н•ң мқҙмң лҠ” нҒ¬кІҢ м„ё к°Җм§ҖлӢӨ.

    мқҙмң  мҰқмғҒ лЁјм Җ мқҳмӢ¬н•  м ңм•Ҫ
    лӘЁлҚёмқҙ нҒ¬лӢӨ weightк°Җ н•ң GPU VRAMм—җ л“Өм–ҙк°Җм§Җ м•ҠлҠ”лӢӨ model weight memory
    contextк°Җ кёёлӢӨ лӘЁлҚёмқҖ мҳ¬лқјк°Җм§Җл§Ң лҸҷмӢң мҡ”мІӯмқ„ лӘ» л°ӣлҠ”лӢӨ KV cache memory
    trafficмқҙ л§ҺлӢӨ latency SLOлҘј мң м§Җн•ҳл©ҙм„ң QPSлҘј лӘ» л°ӣлҠ”лӢӨ replica мҲҳ, scheduler, routing

    мқҙ м„ё мқҙмң лҘј м„һмңјл©ҙ м„Өкі„к°Җ нқ”л“ӨлҰ°лӢӨ. мҳҲлҘј л“Өм–ҙ weightк°Җ н•ң GPUм—җ л“Өм–ҙк°Җм§Җ м•ҠлҠ” л¬ём ңлҠ” tensor parallelismмқҙлӮҳ pipeline parallelismмңјлЎң н’Җм–ҙм•ј н•ңлӢӨ. л°ҳл©ҙ лӘЁлҚёмқҖ н•ң GPUм—җ л“Өм–ҙк°ҖлҠ”лҚ° trafficмқҙ л§ҺмқҖ л¬ём ңлҠ” data parallel replicaлҘј лҠҳлҰ¬лҠ” мӘҪмқҙ лҚ” лӢЁмҲңн•  мҲҳ мһҲлӢӨ. Contextк°Җ кёҙ л¬ём ңлҠ” GPUлҘј лҠҳл ӨлҸ„ KV cache л°°м№ҳ л°©мӢқ, max model length, batch token limitмқ„ к°ҷмқҙ ліҙм§Җ м•Ҡмңјл©ҙ н•ҙкІ°лҗҳм§Җ м•ҠлҠ”лӢӨ.

    л”°лқјм„ң multi-GPU м„Өкі„мқҳ мІ« л¬ёмһҘмқҖ лӢӨмқҢмІҳлҹј мҚЁм•ј н•ңлӢӨ.

    мҡ°лҰ¬лҠ” weight л•Ңл¬ём—җ GPUлҘј лӮҳлҲ„лҠ”к°Җ, KV cache л•Ңл¬ём—җ capacityлҘј лҠҳлҰ¬лҠ”к°Җ, м•„лӢҲл©ҙ traffic л•Ңл¬ём—җ replicaлҘј лҠҳлҰ¬лҠ”к°Җ?

    м„ё к°Җм§Җ parallelismмқ„ к°ҷмқҖ л§җлЎң л¶ҖлҘҙм§Җ м•ҠлҠ”лӢӨ¶

    LLM servingм—җм„ң мһҗмЈј л§ҢлӮҳлҠ” parallelismмқҖ data parallelism, tensor parallelism, pipeline parallelismмқҙлӢӨ.

    Data parallelismмқҖ replicaлҘј лҠҳлҰ°лӢӨ¶

    Data parallel servingмқҖ к°ҷмқҖ лӘЁлҚё replicaлҘј м—¬лҹ¬ к°ң лқ„мҡ°кі  request routerк°Җ мҡ”мІӯмқ„ лӮҳлҲ„лҠ” л°©мӢқмқҙлӢӨ. к°Ғ replicaлҠ” м „мІҙ model weightмҷҖ мһҗкё° replicaмқҳ KV cacheлҘј к°Җм§„лӢӨ.

    мһҘм җмқҖ лӢЁмҲңн•ҳлӢӨ. Replica к°„м—җ л§Ө tokenл§ҲлӢӨ лҸҷкё°нҷ”н•  н•„мҡ”к°Җ м—ҶлӢӨ. н•ң replicaк°Җ лҠҗлҰ¬кұ°лӮҳ мЈҪм–ҙлҸ„ routingкіј retryлЎң кІ©лҰ¬н•ҳкё° мүҪлӢӨ. лӢЁмқј GPUм—җм„ң лӘЁлҚёмқҙ мһҳ мҳ¬лқјк°Җкі  trafficл§Ң л¬ём ңлқјл©ҙ к°ҖмһҘ лЁјм Җ мғқк°Ғн•  мҲҳ мһҲлҠ” л°©мӢқмқҙлӢӨ.

    н•ҳм§Җл§Ң data parallelismмқҖ model memory л¬ём ңлҘј н’Җм§Җ лӘ»н•ңлӢӨ. 70B лӘЁлҚёмқҙ н•ң GPUм—җ м•Ҳ л“Өм–ҙк°„лӢӨл©ҙ replicaлҘј лҠҳлҰҙ мҲҳ м—ҶлӢӨ. лҳҗ long context мҡ”мІӯмқҙ нҠ№м • replicaм—җ лӘ°лҰ¬л©ҙ к·ё replicaмқҳ KV cacheл§Ң м••л°•мқ„ л°ӣлҠ”лӢӨ. Routerк°Җ token shapeмқ„ лӘЁлҘҙкі  лӢЁмҲң round-robinл§Ң н•ҳл©ҙ GPU мӮ¬мҡ©лҘ мқҖ лҶ’м•„ ліҙмқҙлҠ”лҚ° p95 latencyк°Җ л¬ҙл„Ҳм§Ҳ мҲҳ мһҲлӢӨ.

    Tensor parallelismмқҖ layer лӮҙл¶Җ кі„мӮ°мқ„ лӮҳлҲҲлӢӨ¶

    Tensor parallelismмқҖ н•ҳлӮҳмқҳ layer м•Ҳм—җм„ң нҒ° matrix multiplicationмқ„ м—¬лҹ¬ GPU rankлЎң лӮҳлҲҲлӢӨ. к°Ғ GPUлҠ” weight shardлҘј л“Өкі  мһҗкё° shardмқҳ кі„мӮ°мқ„ мҲҳн–үн•ңлӢӨ. к·ё лӢӨмқҢ partial resultлҘј н•©м№ҳкё° мң„н•ҙ collective communicationмқҙ н•„мҡ”н•ҳлӢӨ.

    мқҙ л°©мӢқмқҖ model weightк°Җ н•ң GPUм—җ л“Өм–ҙк°Җм§Җ м•Ҡмқ„ л•Ң к°•л Ҙн•ҳлӢӨ. 4 GPU tensor parallel groupмқ„ м“°л©ҙ к°Ғ GPUк°Җ м „мІҙ weightмқҳ мқјл¶Җл§Ң л“Ө мҲҳ мһҲлӢӨ. лҳҗн•ң attention headлӮҳ MLP projectionмІҳлҹј нҒ° tensor м—°мӮ°мқ„ лі‘л ¬нҷ”н•  мҲҳ мһҲлӢӨ.

    лҢҖк°ҖлҸ„ л¶„лӘ…н•ҳлӢӨ. л§Ө layer лҳҗлҠ” м—¬лҹ¬ layer м§Җм җм—җм„ң rank к°„ лҸҷкё°нҷ”к°Җ н•„мҡ”н•ҳлӢӨ. PrefillмІҳлҹј нҒ° matrix м—°мӮ°мқҙ л§ҺмқҖ кө¬к°„м—җм„ңлҠ” кі„мӮ° лі‘л ¬нҷ” мқҙл“қмқҙ нҒҙ мҲҳ мһҲм§Җл§Ң, decodeмІҳлҹј token н•ҳлӮҳм”© л°ҳліөлҗҳлҠ” кө¬к°„м—җм„ңлҠ” мһ‘мқҖ collective latencyк°Җ TPOTм—җ кі„мҶҚ лҲ„м ҒлҗңлӢӨ.

    Pipeline parallelismмқҖ layer кө¬к°„мқ„ лӮҳлҲҲлӢӨ¶

    Pipeline parallelismмқҖ layer 0-11мқҖ GPU 0, layer 12-23мқҖ GPU 1мІҳлҹј model depthлҘј stageлЎң лӮҳлҲҲлӢӨ. к°Ғ requestмқҳ activationмқҖ stageлҘј л”°лқј мқҙлҸҷн•ңлӢӨ.

    мқҙ л°©мӢқмқҖ GPU мӮ¬мқҙ нҶөмӢ  л№ҲлҸ„лҘј мӨ„мқј мҲҳ мһҲлӢӨ. Tensor parallelismмІҳлҹј layer лӮҙл¶Җл§ҲлӢӨ мһҗмЈј н•©м№ҳлҠ” лҢҖмӢ , stage кІҪкі„м—җм„ң activationмқ„ л„ҳкёҙлӢӨ. vLLM л¬ём„ңлҠ” лӘЁлҚёмқҙ н•ң nodeм—җ л“Өм–ҙк°Җм§Җл§Ң GPU мҲҳк°Җ лӘЁлҚё нҒ¬кё°лҘј кі лҘҙкІҢ лӮҳлҲ„м§Җ лӘ»н•ҳкұ°лӮҳ NVLinkк°Җ м—ҶлҠ” GPU нҷҳкІҪм—җм„ңлҠ” pipeline parallelismмқҙ лҚ” лӮҳмқ„ мҲҳ мһҲлӢӨкі  м„ӨлӘ…н•ңлӢӨ.

    н•ҳм§Җл§Ң pipelineм—җлҠ” bubbleмқҙ мһҲлӢӨ. Stageк°Җ лӘЁл‘җ н•ӯмғҒ л°”мҒҳл Өл©ҙ м¶©л¶„н•ң microbatch лҳҗлҠ” request concurrencyк°Җ н•„мҡ”н•ҳлӢӨ. Decode servingм—җм„ңлҠ” token лӢЁмң„ л°ҳліөкіј scheduler м •мұ… л•Ңл¬ём—җ pipeline нҡЁмңЁмқҙ кё°лҢҖліҙлӢӨ лӮ®м•„м§Ҳ мҲҳ мһҲлӢӨ.

    NCCLмқҖ GPUл“Өмқҙ н•Ёк»ҳ мӣҖм§ҒмқҙкІҢ н•ҳлҠ” нҶөмӢ  кі„мёөмқҙлӢӨ¶

    NCCLмқҖ NVIDIA GPU мӮ¬мқҙмқҳ collective communicationкіј point-to-point communicationмқ„ м ңкіөн•ҳлҠ” libraryлӢӨ. AI мқён”„лқј м—”м§ҖлӢҲм–ҙк°Җ мһҗмЈј л§ҢлӮҳлҠ” collectiveлҠ” лӢӨмқҢкіј к°ҷлӢӨ.

    Collective м§ҒкҙҖ Servingм—җм„ң ліҙмқҙлҠ” мң„м№ҳ
    all-reduce к°Ғ rankмқҳ partial resultлҘј н•©міҗ лӘЁл‘җм—җкІҢ лҸҢл ӨмӨҖлӢӨ tensor parallel layer output н•©м№ҳкё°
    all-gather rankлі„ shardлҘј лӘЁм•„ лӘЁл‘җк°Җ м „мІҙлҘј ліҙкІҢ н•ңлӢӨ sharded tensor мһ¬кө¬м„ұ
    reduce-scatter н•©м№ң кІ°кіјлҘј лӢӨмӢң rankлі„ shardлЎң лӮҳлҲҲлӢӨ distributed tensor м—°мӮ° мөңм Ғнҷ”
    broadcast н•ң rankмқҳ к°’мқ„ лӢӨлҘё rankлЎң ліҙлӮёлӢӨ parameter, metadata, sampling result кіөмң 
    all-to-all rankл§ҲлӢӨ м„ңлЎң лӢӨлҘё лҚ°мқҙн„°лҘј көҗнҷҳн•ңлӢӨ expert parallel, MoE routing

    мӨ‘мҡ”н•ң м җмқҖ NCCLмқҙ "лҠҗлҰ° кІғ"мқҙ м•„лӢҲлқјлҠ” мӮ¬мӢӨмқҙлӢӨ. NCCLмқҖ GPU нҶөмӢ мқ„ л№ лҘҙкІҢ н•ҳкё° мң„н•ң кі„мёөмқҙлӢӨ. л¬ём ңлҠ” serving workloadк°Җ нҶөмӢ мқ„ л°ҳліөм ҒмңјлЎң мҡ”кө¬н•ңлӢӨлҠ” лҚ° мһҲлӢӨ.
